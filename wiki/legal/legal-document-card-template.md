# Шаблон карточки нормативного документа

Служебная страница для авторов и агентов: при добавлении или правке нормативной карточки копируйте **блок ниже** (содержимое fenced-блока) в файл `wiki/legal/<slug>.md`, замените плейсхолдеры и удалите неприменимые разделы целиком (без пустых заголовков). Для **новых** карточек в frontmatter укажите `slug: /legal/<slug>`, где `<slug>` совпадает с именем файла без `.md`, чтобы URL оставался `/legal/<slug>`.

Один акт — **одна** каноническая карточка; варианты «кратко» и «развёрнуто» объединяйте подзаголовками или вынесите в отдельные разделы внутри одного файла, а не дублируйте файлами.

### Поля frontmatter

| Поле | Обязательно | Пример |
|------|-------------|--------|
| `title` | да | `"Федеральный закон № 44-ФЗ …"` |
| `sidebar_label` | да | `"44-ФЗ"` |
| `description` | рекомендуется | 1–2 предложения для превью и индексации (без экранирования скобок лишними слэшами) |
| `tags` | да | `[legal, procurement]` — вторая метка по теме (`budget`, `procurement`, `open-data`, `accounting`, …) |
| `last_updated` | да | `2026-05-11` |
| `slug` | да для новых карточек | `/legal/<slug>` — тот же `<slug>`, что имя файла |
| `content_type` | да | `legal` |
| `entity_type` | да | `legal-document` |
| `doc_kind` | рекомендуется | `constitution` \| `federal_law` \| `government_decree` \| `ministry_order` \| `instruction` \| `overview` |
| `source_url` | по возможности | одна основная ссылка на актуальную редакцию (Консультант, Гарант, pravo.gov.ru) |
| `official_urls` | при нескольких входах | список зеркал официального текста (YAML-массив строк) |

---
title: "<Полный заголовок документа>"
sidebar_label: "<Короткая подпись в сайдбаре>"
description: >-
  <1–2 предложения: что регулирует и зачем это странице про открытые финансовые данные.>
tags: [legal, <тема>]
last_updated: YYYY-MM-DD
slug: /legal/<slug>
content_type: legal
entity_type: legal-document
doc_kind: federal_law
# source_url: "https://..."
# official_urls:
#   - "http://pravo.gov.ru/..."
---

## Где это в данных

Перечисляйте **конкретно** под этот акт (не копируйте один и тот же список во все карточки). Дайте **минимум 2–3** элемента со ссылками вида `/data-sources/<подраздел>/<slug>`, `/information-systems/<подраздел>/<slug>`, при необходимости `/reporting/<slug>` — каждая ссылка с **одной строкой пояснения** (какой объект данных, публикация или поле смыслово вытекает из нормы).

Если прямых открытых наборов нет — напишите это явно и укажите публичные производные (портал, PDF, смежная ИС) со ссылкой на соответствующую карточку wiki.
