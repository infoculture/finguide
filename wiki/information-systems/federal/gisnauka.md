# ГИС «Наука» (gisnauka.ru)

| Признак | Значение |
| :--- | :--- |
| Краткое имя | ГИС «Наука», домен науки и инноваций Минобрнауки России |
| Оператор | Министерство науки и высшего образования Российской Федерации |
| Официальный URL | https://gisnauka.ru/ |
| Режим доступа | Смешанный: публичные разделы учёта НИОКТР и результатов интеллектуальной деятельности (РИД) и отраслевые сервисы на поддоменах; часть функций — через ЕСИА и подпись по регламентам ведомства |

## Назначение и границы

**Государственная информационная система в сфере науки** на домене `gisnauka.ru` — федеральный контур, в котором Минобрнауки России ведёт и раскрывает учёт **научно-исследовательских, опытно-конструкторских и технологических работ (НИОКТР)** и связанных с ними **результатов интеллектуальной деятельности (РИД)**, а также сводную отчётность организаций так, чтобы при наличии в источниках можно было сопоставить **объекты науки** (организация, тема, регистрационные записи НИОКТР/РИД) с **источниками финансирования** из федерального бюджета и иных мер поддержки.

Система **не** является **единым реестром госпрограмм** и паспортов программных показателей — этим занимаются **[programs.gov.ru](/information-systems/federal/programs-gov)** и смежные контуры Минэкономразвития. Она **не** заменяет **ГИИС «Электронный бюджет»** для казначейского исполнения и росписи — фактические **субсидии и соглашения** сверяйте по **[наборам budget.gov.ru](/data-sources/federal/budget-gov-ru-datasets)** и **[реестру соглашений](/data-sources/federal/subsidy)**. Учёт **НИОКТР гражданского назначения** закреплён нормативно в том числе через развитие **ЕГИСУ НИОКТР**; публичный доступ к сведениям об НИОКТР и смежным объектам следует искать в **актуальных** интерфейсах, **HTTP API открытых выгрузок** [`/api/egisu/open-data`](https://gisnauka.ru/api/egisu/open-data), **[глобальном поиске](https://gisnauka.ru/global-search)**, выгрузках **«Сформировать выгрузку»** и ответах внутреннего API на **ГИС «Наука»**, а не по устаревшим адресам отдельных реестров.

## Пользователи и типовые задачи

- Исследователь открытых данных — найти регистрационные и отчётные записи по НИОКТР/РИД и идентификаторы организаций для стыковки с бюджетными реестрами по **ИНН** и программным кодам.
- Журналист или контролёр — проследить цепочку от темы исследования до заявленного источника финансирования там, где она публикуется в открытой части.
- Разработчик интеграции — использовать **`/api/egisu/open-data`** для помесячных и годовых срезов по **`card_type`** (см. **[how-to](/howto/access/gisnauka)**), **глобальный поиск** и при необходимости снимать параметры **внутреннего API** из вкладки Network в браузере, а также **выгрузки по годам и типам данных** через интерфейс «Сформировать выгрузку»; параллельно подключать **[открытые данные «Электронного бюджета»](/data-sources/federal/budget-gov-ru-datasets)** и **[реестр соглашений](/data-sources/federal/subsidy)** для денежного контура.

## Подсистемы и разделы

- Учёт и публикация сведений о **НИОКТР** и **РИД** (в т.ч. связь темы, исполнителя, регистрационных номеров и отчётности по методике Минобрнауки).
- **Глобальный поиск** по данным контура ЕГИСУ на странице [`/global-search`](https://gisnauka.ru/global-search): загружает данные через JSON API (см. точки доступа ниже).
- **API открытых выгрузок** [`/api/egisu/open-data`](https://gisnauka.ru/api/egisu/open-data) — помесячные и годовые файлы по **`card_type`** (см. **[how-to](/howto/access/gisnauka)**).
- На ряде страниц сайта доступна интерактивная ссылка **«Сформировать выгрузку»**: формирование **дампа** по выбранным **году** и **типу данных** (конкретный перечень типов задаётся интерфейсом на дату обращения).
- Сервисы на **поддоменах** домена `gisnauka.ru` (например отраслевые реестры и кабинеты; состав разделов меняется релизами ведомства).
- Интеграция с **единой цифровой платформой** «ГосТех» и **ЕСИА** для сервисов, переведённых на общий контур входа (по сообщениям ведомства и подведомственных организаций — сверяйте инструкции на сайте оператора сервиса).

## Точки доступа

| Тип доступа | URL | Формат |
| --- | --- | --- |
| Корень ГИС «Наука» | https://gisnauka.ru/ | HTML |
| Глобальный поиск (ЕГИСУ) | https://gisnauka.ru/global-search | HTML (SPA; запросы к API см. ниже) |
| Внутренний JSON API поиска (вызывается со страницы глобального поиска; **нет** открытой спецификации) | https://gisnauka.ru/api/egisu/base/search | JSON (метод, заголовки и тело запроса — как в DevTools браузера) |
| API открытых выгрузок по периоду и типу карточки (`year`, `month`, `card_type`; за год — `month=all_months`, только **завершённые** периоды) | https://gisnauka.ru/api/egisu/open-data | файл/поток (см. `Content-Type` на дату запроса) |

**Выгрузки:** пункт **«Сформировать выгрузку»** на страницах портала открывает сценарий массовой выгрузки по **году** и **типу данных**; итог обычно загружается как файл или выдаётся по ссылке после генерации — зависит от раздела и версии интерфейса. Параллельно для **регулярных** машинных срезов по типам сведений используйте **`/api/egisu/open-data`** (параметры — в **[how-to](/howto/access/gisnauka)**).

Доступ с некоторых сред автоматизации к `gisnauka.ru` может упираться в **политики TLS** или фильтрацию; для массовых выгрузок ориентируйтесь на **`open-data`** с корректной цепочкой доверия, сценарий **«Сформировать выгрузку»** и сохранённые **прямые URL** файлов из браузера после генерации. Для **воссоздания** вызовов **`/api/egisu/base/search`** копируйте запрос из инструментов разработчика (**Network**) при работе с [глобальным поиском](https://gisnauka.ru/global-search): публичного описания контракта нет, параметры и авторизация могут меняться без предупреждения.

## Данные: объекты и связь с источниками

Типовые объекты публичного контура: **карточки НИОКТР**, **РИД** (охранные документы, отчётность по результатам), **организации исполнителей** с **ИНН/ОГРН**, при наличии в публикации — **ссылки на гранты, субсидии, программы** или номера государственных контрактов. Связка «наука — бюджет» в открытых данных обычно **неполная**: часть полей остаётся в служебных подсистемах, а денежный факт исполнения подтверждается в **[ГИИС «Электронный бюджет»](/information-systems/federal/giis-eb)**.

- **[Доступ к данным ГИС «Наука» (how-to)](/howto/access/gisnauka)** — параметры **`/api/egisu/open-data`**, типы **`card_type`**, ограничения по завершённым периодам.
- **[ГИС «Наука»: открытые данные НИОКТР, РИД и бюджетное финансирование](/data-sources/federal/gisnauka-niokr-rid-budget)** — как искать ведомственные наборы и что считать ключами стыковки с бюджетом.
- **[Наборы данных ГИИС «Электронный бюджет»](/data-sources/federal/budget-gov-ru-datasets)** — реестр соглашений, исполнение и участники для соединения с ИНН организации из научных реестров.
- **[Соглашения (субсидии и субвенции)](/data-sources/federal/subsidy)** — логика реестра соглашений и типичные поля для сверки с грантами и субсидиями на НИОКТР.

Для расшифровки поддержки «на конкурсной основе» см. **[Гранты](/glossary/grants)** и отличие от **субсидий** по **[КБК](/glossary/kbk)** в выгрузках бюджета.

## Связанные информационные системы

- **[ГИИС «Электронный бюджет»](/information-systems/federal/giis-eb)** — операционный и открытый бюджетный контур для сумм и реестров соглашений; дополняет научный учёт фактическим исполнением.
- **[programs.gov.ru — Государственные программы РФ](/information-systems/federal/programs-gov)** — паспорта и показатели программ, с которыми часто связывают ведомственные отчёты по науке на качественном уровне.

## Нормативная основа

Учёт НИОКТР гражданского назначения и открытость сведений опираются на **Постановление Правительства РФ от 12.04.2013 № 327** о единой государственной информационной системе учёта НИОКТР гражданского назначения и последующие акты о развитии системы; положения **Федерального закона «О науке и государственной научно-технической политике»** задают рамки отчётности и гражданского контроля. Детали применения к конкретному сервису на `gisnauka.ru` сверяйте с **официальными текстами** НПА на дату задачи.

## Ограничения и актуальность

- Состав публичных разделов и поддоменов **меняется** при переводе сервисов на **ГосТех** и обновлении регламентов; фиксируйте дату снимка и URL карточки набора.
- **Один ИНН** может фигурировать в научном учёте и в бюджетных реестрах с **разной полнотой** полей — склейка только по ИНН без периода и вида документа даёт ложные совпадения.
- Машиночитаемые выгрузки по линии Минобрнауки нужно искать в **интерфейсе** **ГИС «Наука»** (поиск, выгрузки, API), а не предполагать единый стабильный каталог файлов на корне `gisnauka.ru`.
- **`/api/egisu/open-data`** — публичный **GET** с параметрами периода и **`card_type`**, но **годовые** срезы (`month=all_months`) и **помесячные** файлы ожидайте только за **уже завершённые** календарные периоды; детали — в **[how-to](/howto/access/gisnauka)**.
- Эндпоинт **`/api/egisu/base/search`** и сценарий **«Сформировать выгрузку»** для внешних интеграций **менее предсказуемы**: у **`base/search`** нет опубликованной схемы, возможны **403/429**, смена URL и обязательные заголовки из фронтенда; закладывайте обслуживание пайплайна при обновлениях сайта.

## Частые вопросы

**В:** Где увидеть связь темы НИОКТР с суммой из федерального бюджета?

**О:** В **[наборах по ГИС «Наука»](/data-sources/federal/gisnauka-niokr-rid-budget)** и в файлах, полученных через **`open-data`**, **«Сформировать выгрузку»** или ответы поискового API — если поле есть в выгрузке; для подтверждения оплаты и лимитов используйте **[реестр соглашений](/data-sources/federal/subsidy)** и **[открытые данные «Электронного бюджета»](/data-sources/federal/budget-gov-ru-datasets)** по **ИНН** и отчётному году.

**В:** Где искать сведения об НИОКТР, если отдельный публичный реестр на прежнем домене недоступен?

**О:** Ориентируйтесь на **интерфейсы и справочники внутри** **ГИС «Наука»** (`gisnauka.ru`): API **`/api/egisu/open-data`**, [глобальный поиск](https://gisnauka.ru/global-search), выгрузки **«Сформировать выгрузку»** и сценарии на поддоменах; конкретные разделы и форматы меняются релизами ведомства.
