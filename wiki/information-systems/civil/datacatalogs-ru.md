# DataCatalogs.ru — реестр каталогов открытых данных

| Признак | Значение |
| :--- | :--- |
| Краткое имя | DataCatalogs.ru |
| Оператор | НП «Информационная культура» |
| Официальный URL | https://datacatalogs.ru/ |
| Режим доступа | Открытый: веб-каталог без регистрации |

## Назначение и границы

**DataCatalogs.ru** — гражданский **метакаталог**: карточки **порталов и реестров открытых данных** (федеральный портал ОД, региональные и муниципальные порталы, отраслевые витрины, международные источники и др.), с кратким описанием, ссылками и заметками об актуальности и качестве публикации. Сервис помогает **найти точку входа** в открытые данные по ведомству или территории, не подменяя собой **первичные каталоги** операторов.

Система **не** хранит и **не** нормирует сами **наборы строк** бюджета, закупок или отчётности: за состав полей, API и лицензии на датасеты отвечают **исходные порталы**. Для машинной выгрузки конкретных таблиц по госфинансам обычно идут в **[открытые данные «Электронного бюджета»](/data-sources/federal/budget-gov-ru-datasets)**, **[Минфина](/data-sources/federal/minfin-opendata)**, **[ЕИС](/data-sources/federal/procurement)** и региональные витрины, а DataCatalogs.ru используют как **навигатор по каталогам**.

## Пользователи и типовые задачи

- Аналитик открытых данных — сопоставить, какие **ведомственные и территориальные порталы ОД** существуют и куда смотреть за бюджетной и смежной тематикой.
- Исследователь или журналист — быстро получить **обзорную справку** о портале (охват, форматы, API — по мере отражения в карточке) до перехода на официальный сайт.
- Разработчик интеграции — выбрать **канонический URL каталога** поставщика данных, не полагаясь только на поисковую выдачу.

## Подсистемы и разделы

- **Перечень каталогов** — тематические и территориальные записи со ссылками на операторов.
- **Описания и метаданные** — текстовые аннотации к порталам (в т.ч. машиночитаемые срезы на стороне проекта — по мере поддержки функционала сайта).
- **Оценка актуальности и качества** — заметки о полноте, форматах, типичных ограничениях порталов (ориентир для читателя, не официальный аудит).

## Данные: объекты и связь с источниками

В терминах учёта и аналитики открытых данных здесь в первую очередь фигурируют **объекты-каталоги** (портал ОД, реестр наборов, тематическая витрина) и их **метаданные** (URL, краткое описание, тематика), а не строки **исполнения бюджета**, **контрактов** или **субсидий**. Чтобы перейти от «нашли портал» к «скачали таблицу», нужно следовать на **официальный источник** и сверяться с паспортами наборов на стороне оператора.

Связь с разделом источников данных в wiki:

- **[Открытые данные Минфина России](/data-sources/federal/minfin-opendata)** — федеральные справочники и срезы (КБК, роспись, долг и др.): типичный пример наборов, которые ищут через федеральный контур ОД после ориентира в каталогах.
- **[Открытые данные портала «Электронный бюджет»](/data-sources/federal/budget-gov-ru-datasets)** — машиночитаемые выгрузки по бюджету и исполнению на `budget.gov.ru`; DataCatalogs.ru может указывать на **федеральный портал ОД РФ** как на вход, но детальные поля и API описаны в карточке источника.
- **[Региональные порталы «Открытый бюджет»](/data-sources/regional/regional-portals)** — перечень официальных региональных витрин исполнения и открытости бюджета; полезно сочетать с записями DataCatalogs.ru о **региональных порталах ОД** для выбора именно бюджетной ветки.

Для понимания классификации и полей в выгрузках после выбора портала имеет смысл заглянуть в глоссарий, например **[КБК](/glossary/kbk)** и **[ГРБС](/glossary/grbs)**.

## Связанные информационные системы

- **[HubOfData](/information-systems/civil/hubofdata)** — каталог **датасетов** (в т.ч. по госфинансам) на базе CKAN с API того же оператора; дополняет DataCatalogs.ru, который ориентирован на **каталоги-порталы**, а не на единичные наборы.
- **[ГИИС «Электронный бюджет»](/information-systems/federal/giis-eb)** — официальный портал бюджетной системы и публикации открытых данных исполнения; первичный контур для сверки с **гражданским навигатором** по каталогам.

## Ограничения и актуальность

Карточки порталов и оценки обновляются **неравномерно**: даты в описаниях могут отставать от текущего состояния сайта оператора. Состав наборов, версии API и URL разделов на **сторонних порталах** меняются без уведомления DataCatalogs.ru — перед интеграцией всегда проверяйте **официальный каталог** поставщика. Проект **не** является юридически значимым реестром открытых данных государства.

## Частые вопросы

**В:** Чем DataCatalogs.ru отличается от федерального портала открытых данных?

**О:** Федеральный портал ОД — **операторская витрина** наборов в установленном законом контуре; DataCatalogs.ru — **независимый обзорный реестр** множества каталогов (в т.ч. региональных и отраслевых) с навигацией и комментариями, без замены официальных метаданных и паспортов наборов.

**В:** Можно ли отсюда скачать исполнение бюджета или реестр контрактов?

**О:** **Нет** как из единого API этой ИС: здесь описаны **входы в чужие каталоги**. Сами таблицы берут с порталов ведомств и из специализированных источников в wiki, например **[бюджетные датасеты](/data-sources/federal/budget-gov-ru-datasets)** и **[закупки](/data-sources/federal/procurement)**.

**В:** Когда удобнее HubOfData, а когда DataCatalogs.ru?

**О:** **HubOfData** — когда уже нужен **конкретный датасет** и CKAN API по агрегированным выгрузкам. **DataCatalogs.ru** — когда нужно **сопоставить и выбрать портал-каталог** (федеральный, региональный, муниципальный или отраслевой) до углубления в файлы.
