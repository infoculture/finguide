# Шаблон карточки информационной системы

Служебная страница для авторов и агентов: при добавлении или правке карточки копируйте **блок ниже** (содержимое fenced-блока) в файл `wiki/information-systems/<подраздел>/<slug>.md`, замените плейсхолдеры и удалите неприменимые разделы целиком (без пустых заголовков).

Подраздел файла (`federal`, `civil`, `regional`) должен соответствовать типу системы. Slug страницы совпадает с именем файла без `.md` (как в ссылках Docusaurus: `/information-systems/federal/zakupki`).

### Поля frontmatter

| Поле | Обязательно | Пример / значения |
|------|-------------|-------------------|
| `title` | да | `"ЕИС в сфере закупок"` |
| `sidebar_label` | да | `"ЕИС Закупки"` |
| `tags` | да | `[information-system, federal]` — вторая метка: `federal` \| `civil` \| `regional`; при необходимости добавьте тему (`procurement`, `budget`, `tax`, …) |
| `last_updated` | да | `2026-05-10` |
| `source_url` | по возможности | канонический URL портала или страницы об ИС |
| `system_kind` | рекомендуется | `government_is` — ведомственная ИС (часто закрытая); `public_portal` — официальная публичная витрина/портал; `civil_aggregator` — гражданский агрегатор/зеркало; `registry` — реестр; `statistics_portal` — статистика; `parliament_control` — парламент/контроль |
| `access` | рекомендуется | `open` \| `restricted` \| `mixed` |
| `primary_topics` | по желанию | `["procurement", "budget_execution"]` — короткие ярлыки темы для связи с другими разделами wiki |
| `operator_type` | по желанию | `federal_body` \| `regional_body` \| `ngo` \| `community` |

## Данные: объекты и связь с источниками

Заполняй **конкретно** под систему. Не ограничивайтесь отсылкой «см. источники данных»:

- Перечислите **типы объектов и показателей**, которые относятся к этой ИС (контракты, извещения, отчёты исполнения, справочники и т.д.).
- Дайте **минимум 2–3 ссылки** на страницы `wiki/data-sources/` с пояснением одной строкой, что там описано относительно этой системы. Если подходящих страниц ещё нет — явно напишите это и укажите характерные поля/сущности текстом.
- При необходимости добавьте 1–2 ссылки на термины глоссария: `/glossary/<slug>` с пояснением связи.

Если система **не** публикует пользовательские наборы (закрытая ИС), опишите это явно и перечислите **публичные производные** — другие карточки в `information-systems/` или разделы `data-sources/`.
