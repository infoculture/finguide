# Региональные информационные системы

## Назначение раздела

Здесь собраны карточки **региональных** информационных систем пилота, через которые публикуются бюджет и исполнение. Первая волна охватывает шесть моделей доступа; остальные субъекты пилота представлены карточками финансовых органов и источников без искусственного выделения «ИС» из обычной страницы документов.

## Системы пилота

- [Открытый бюджет Москвы](/information-systems/regional/moscow-open-budget) — интерактивная городская витрина.
- [Открытый бюджет Санкт-Петербурга](/information-systems/regional/saint-petersburg-open-budget) — показатели и связь с Комитетом финансов.
- [Открытый бюджет Татарстана](/information-systems/regional/tatarstan-open-budget) — документно-вебовый раздел Минфина.
- [Открытый бюджет Свердловской области](/information-systems/regional/sverdlovsk-open-budget) — сервис `smart.mfural.ru`.
- [Открытый бюджет Дагестана](/information-systems/regional/dagestan-open-budget) — отдельный региональный портал.
- [Открытый бюджет Югры](/information-systems/regional/yugra-open-budget) — контрастный документный пример без подтверждённого массового API.

## Типовые вопросы

**Чем региональная ИС отличается от регионального источника в `data-sources`?** ИС — операторская среда; **источник** — конкретные наборы, API и ограничения выгрузки. Карточки ИС и источников должны перекрёстно ссылаться друг на друга, когда связь однозначна.
