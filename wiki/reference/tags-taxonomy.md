# Таксономия тегов в frontmatter

В карточках wiki поле `tags` — массив строк. **Первый тег** задаёт тип раздела (например `glossary`, `data-source`, `legal`, `intro`). **Второй тег** — тематика; его стоит выбирать из списка ниже, чтобы материалы было проще группировать и находить (в том числе при индексации для ИИ).

## Рекомендуемые вторые теги

| Тег | Когда использовать |
| --- | --- |
| `budget` | Бюджетный процесс, доходы/расходы, исполнение, консолидация |
| `classification` | КБК, КОСГУ, КВР, целевые статьи, НПА по классификации |
| `procurement` | Закупки, контракты, ЕИС, 44-ФЗ / 223-ФЗ |
| `transfers` | Субсидии, гранты, межбюджетные трансферты |
| `reporting` | Формы бюджетной отчётности, статистика сектора |
| `opendata` | Открытые данные, порталы, машиночитаемые форматы |
| `treasury` | Казначейство, ЛБО, кассовое исполнение |
| `tax` | Налоги, ФНС, таможня (как слой доходов) |
| `legal` | Уже первый тег в разделе legal; вторая метка — тема акта (`procurement`, `classification`, …) |
| `meta` | Шаблоны, служебные страницы для авторов |
| `navigation` | Оглавления, карты разделов, how-to по сайту |
| `rag` | Страницы, явно ориентированные на поиск/чанкинг (можно комбинировать с темой) |

Если подходящего тега нет, добавьте узкий новый (например `debt`, `nko`) и при необходимости допишите его в эту таблицу.
