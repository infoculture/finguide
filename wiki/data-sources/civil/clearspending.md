# Clearspending.ru — открытые данные закупок

| Признак | Значение |
| :--- | :--- |
| Уровень и охват | Гражданский агрегатор; **контуры 44-ФЗ и 223-ФЗ** поверх публикаций **[ЕИС Закупки](/information-systems/federal/zakupki)**; отдельный поддомен по **субсидиям** |
| Тип доступа | **Смешанный**: веб-поиск и карточки, **REST API** закупочного контура (описание — на странице открытых данных проекта) |
| Основные форматы | **JSON** (API), **HTML** (витрины **clearspending.ru** и **sub.clearspending.ru**) |
| Оператор | Комитет гражданских инициатив, АНО «Инфокультура» и сообщество проекта («Госзатраты») |
| Режим доступа | Публичный веб и API; условия переработки и атрибуции — в материалах на сайте проекта |

Коротко: **нормализованный контрактный след** и поиск по **контрактам**, **извещениям**, **заказчикам** и **поставщикам** без сборки **сырых XML-архивов ЕИС**; удобная отправная точка для мониторинга и скриптов, если юридически значимая копия документа не нужна.

## Описание

Карточка фиксирует **публичные машиночитаемые и веб-входы** проекта **Clearspending.ru**: закупочная часть на основном домене и связанная витрина по **субсидиям** на **sub.clearspending.ru**. Сервис **не** дублирует обязанности оператора **ЕИС** и **не** заменяет **[официальные реестры субсидий](/data-sources/federal/subsidy)** для юридически значимой сверки.

Сюда **не** входят внутренние кабинеты (если появятся) и любые каналы, не описанные на публичных страницах проекта.

## Оператор

**Комитет гражданских инициатив**, **АНО «Инфокультура»** и сообщество проекта — каноническая витрина: https://clearspending.ru/

- Карточка ИС: **[Clearspending.ru](/information-systems/civil/clearspending)** — назначение подсистем, типовые задачи и ограничения относительно **ЕИС**.

## URL и точки доступа

| Тип доступа | URL | Формат |
| --- | --- | --- |
| Веб-портал (закупки) | https://clearspending.ru/ | HTML |
| REST API (открытые данные закупочного контура) | https://clearspending.ru/opendata/ | JSON (см. документацию на странице) |
| Веб-портал (субсидии) | https://sub.clearspending.ru/ | HTML |

## Объекты данных и показатели

- **Строки выдачи** — как правило, **контракт** или **извещение/процедура** в нормализованном виде, плюс агрегаты в аналитических разделах (набор зависит от версии сайта и API).
- **Справочные сущности** — **заказчики** и **поставщики** с реквизитами для связки (в т.ч. **ИНН**, **ОГРН**).
- **Субсидии** — отдельная логика на **sub.clearspending.ru**; склейка с бюджетным контуром требует сверки с **[соглашениями и реестрами](/data-sources/federal/subsidy)**.

Типичные **ключи связи** с официальными контурами: **ИНН/ОГРН**, реестровые идентификаторы контракта/извещения в логике **ЕИС**, коды **ОКПД2** и **КБК** там, где они проходят в индекс. **44-ФЗ** и **223-ФЗ** смешаны в одном потоке публикаций **ЕИС** — перед суммированием нужен явный отбор по режиму (см. **[сведения о закупках](/data-sources/federal/procurement)** и термины **[44-ФЗ](/glossary/procurement-44fz)** / **[223-ФЗ](/glossary/procurement-223fz)**).

## Периодичность обновления

Фиксированного **SLA** у гражданского агрегатора нет: полнота и **задержка относительно ЕИС** зависят от конвейера индексации. Ориентиры и оговорки — в разделе **«Ограничения и актуальность»** карточки **[Clearspending.ru](/information-systems/civil/clearspending)**; для даты и состава юридически значимого документа используйте **[ЕИС](/information-systems/federal/zakupki)**.

## Ограничения и особенности

- **Не первичный реестр**: юридически значимое размещение закупочных документов — в **ЕИС**; витрина Clearspending может **отставать**, **фильтровать поля** или иначе отличаться от сырого XML.
- **Программный доступ**: часть URL (в т.ч. раздел **opendata** и префиксы API) может отдавать **страницу проверки Anubis** при запросах без полноценного браузера или без cookie после прохождения challenge — закладывайте в архитектуру **браузерный клиент**, **ручную сессию** или **официальные выгрузки ЕИС**, если нужен стабильный headless-канал.
- **Схема API и лимиты** меняются — перед продакшеном сверяйте актуальную документацию на https://clearspending.ru/opendata/ и практическую инструкцию **[REST API Clearspending](/howto/access/clearspending)**.
- **Лицензия**: условия повторного использования и указания источника — в сопроводительных текстах проекта на сайте.
