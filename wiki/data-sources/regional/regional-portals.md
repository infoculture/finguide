# Региональные порталы «Открытый бюджет»

| Признак | Значение |
| :--- | :--- |
| Уровень и охват | **Субъекты РФ**: отдельные публичные витрины прозрачности бюджета (часто с названием «Открытый бюджет» или аналогом), на которые ссылается или которые дополняет **единый портал** бюджетной системы |
| Тип доступа | Смешанный: веб-порталы регионов (HTML, вложения), иногда разделы открытых данных и табличные выгрузки; **единого API «все регионы»** нет |
| Основные форматы | Преимущественно HTML и офисные вложения (XLS/XLSX, PDF); машиночитаемые наборы там, где регион выложил CSV/JSON на своей стороне |
| Оператор | Финансовые органы и иные органы исполнительной власти **субъектов РФ**; федеральная навигация и контекст — **[ГИИС «Электронный бюджет»](/information-systems/federal/giis-eb)** (`budget.gov.ru`) |
| Режим доступа | Открытый для опубликованных разделов (типичный сценарий без регистрации для чтения) |

Коротко: это **совокупность официальных региональных сайтов** с материалами открытости бюджета (показатели, законы о бюджете, исполнение, иногда «бюджет для граждан»). Карточка задаёт **как искать вход** с федерального портала и чем дополнять выгрузку **федеральными** наборами `budget.gov.ru`, не подменяя региональный сайт.

## Описание

Инициатива «Открытый бюджет» на уровне субъектов реализуется **на сайтах регионов** и может называться по-разному. Состав разделов (план, факт, госпрограммы, долг, субсидии) и глубина раскрытия **заметно отличаются**. Федеральный портал **не заменяет** полный комплект документов на стороне субъекта, но помогает **ориентироваться** в структуре ГИИС и связанных открытых данных.

Карточка **не** описывает каждый региональный URL по отдельности (список меняется); фиксируются **проверенные точки входа** на `budget.gov.ru` и типичные типы объектов на региональных витринах.

## Оператор

**Органы государственной власти субъектов РФ** (как правило финансовый блок) — первичная публикация региональной «открытости» бюджета на собственных доменах.

Федеральный контекст и машиночитаемые слои единого портала: **Минфин России** и **Федеральное казначейство** — канонический вход https://budget.gov.ru/

- Карточка ИС: **[ГИИС «Электронный бюджет»](/information-systems/federal/giis-eb)** — единый портал бюджетной системы; публичные разделы по субъектам и открытые данные портала согласованы с этой системой, но **не исчерпывают** все региональные публикации.

## URL и точки доступа

| Тип доступа | URL | Формат |
| --- | --- | --- |
| Единый портал бюджетной системы (главная, навигация к разделам) | https://budget.gov.ru/ | HTML |
| Раздел «Регионы» | https://budget.gov.ru/регионы | HTML |
| Паспорт региона (типовой сценарий отбора субъекта) | https://budget.gov.ru/регионы/паспорт-региона | HTML |
| Открытые данные портала (федеральные наборы; не путать с сайтом конкретного субъекта) | https://budget.gov.ru/открытые-данные | HTML; см. каталог наборов в связанных источниках |

Проверка ссылок (HTTP-заголовки, **2026-05-11**): перечисленные выше пути отвечают **200 OK**. Исторический короткий путь вида `https://budget.gov.ru/Развитие/Открытый-бюджет` (раньше использовался как «перечень региональных порталов») по той же проверке отдаёт **404** — на него ориентироваться не следует; используйте главную страницу и раздел **«Регионы»**, а также официальный сайт выбранного субъекта.

Устаревшие URL вида `/Развитие/…` на стороне портала в целом ненадёжны (см. замечание в **[наборы данных ГИИС «Электронный бюджет»](/data-sources/federal/budget-gov-ru-datasets)**): для открытых данных портала надёжнее путь **`/открытые-данные`** или технические адреса каталога из карточки наборов.

## Объекты данных и показатели

Заполняйте содержательно под **конкретный** региональный сайт; типично на витринах встречаются:

- агрегированные **доходы и расходы**, показатели **дефицита и источников** покрытия;
- **закон о бюджете субъекта**, изменения, пояснительные материалы;
- срезы **исполнения** и отчётность в представлении региона (не смешивать без сверки методики с **[казначейскими формами](/data-sources/federal/roskazna-reports)**);
- **государственные программы** субъекта, реестры расходных обязательств — если регион выкладывает их в открытой части.

Ключи связки с другими источниками: код **субъекта РФ**, **[ОКТМО](/glossary/oktmo)** при территориальных таблицах, **[КБК](/glossary/kbk)** и **[КОСГУ](/glossary/kosgu)** в машиночитаемых выгрузках, **[ИНН](/glossary/inn)** участников в реестрах.

Пробелы: нет единой схемы полей по всем регионам; часть данных может быть только в PDF; динамические страницы без стабильных выгрузок.

## Периодичность обновления

Задаётся **регионом** (цикл подготовки и утверждения бюджета, ежемесячное или квартальное обновление показателей исполнения — по сайту субъекта). Общие ограничения и смена URL на федеральной витрине — в разделе **«Ограничения и актуальность»** карточки **[ГИИС «Электронный бюджет»](/information-systems/federal/giis-eb)**.

## Ограничения и особенности

- **Лицензия и условия переработки** задаются на стороне каждого регионального портала и его файлов открытых данных (если выделены отдельно).
- Нет одного endpoint для загрузки всех региональных витрин; автоматизация строится **по спискам URL** или парсингу HTML/API конкретного региона (см. **[Работа с региональными порталами открытых данных](/howto/access/regional-portals)**).
- Показатели на федеральном портале и на сайте субъекта могут **расходиться** по методике и периоду — сверяйте метаданные и документ-источник.

## Примеры доступа

Проверка доступности раздела «Регионы» с командной строки:

```bash
curl -sI -L "https://budget.gov.ru/%D1%80%D0%B5%D0%B3%D0%B8%D0%BE%D0%BD%D1%8B" | head -n 5
```

Ожидается ответ сервера с кодом **200** для публичной страницы (при сетевых ограничениях возможны иные коды).
