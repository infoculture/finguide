# Город Москва: портал открытых данных Правительства Москвы (data.mos.ru)

| Признак | Значение |
| :--- | :--- |
| Уровень и охват | **Город Москва** (субъект РФ): единая витрина **открытых наборов** правительства |
| Тип доступа | **Каталог** наборов с карточками, метаданными, API и файловыми выгрузками (по набору) |
| Основные форматы | **CSV**, **JSON**, **XLSX** (зависят от набора), веб-интерфейс **HTML** |
| Оператор | **Правительство Москвы** (портал открытых данных) |
| Режим доступа | Открытая публикация; условия использования — на портале и в паспортах наборов |

Коротко: **точка входа** для **машинной** работы с опубликованными городскими данными: поиск набора по теме, выгрузка и API. **Полный бюджет** в разрезе КБК и форм исполнения сверяйте с **[«Открытым бюджетом»](/data-sources/regional/moscow-open-budget)** и **[наборами «Электронного бюджета»](/data-sources/federal/budget-gov-ru-datasets)**.

## Описание

Портал **data.mos.ru** агрегирует **машиночитаемые** наборы разных департаментов. Для **бюджетного контура** используйте фильтры каталога по ключевым словам («бюджет», «расход», «контракт», «субсидия» и т.п.) и перекрёстно проверяйте с **[budget.mos.ru](/data-sources/regional/moscow-open-budget)** и **[департаментом финансов](/data-sources/regional/moscow-findep)**.

## Оператор

**Правительство Москвы** — корень каталога: https://data.mos.ru/

## Объекты данных и показатели

- **Единица публикации** — **набор открытых данных** с собственным идентификатором, паспортом и историей версий.
- **Ключи связки** зависят от набора: ОКТМО, коды муниципальных услуг, ИНН контрагента, год и т.д. — указываются в структуре полей.
- **Пробелы**: не каждый показатель бюджетной классификации выведен в open data; для **сводной отчётности** используйте **[консолидированные бюджеты](/data-sources/regional/consolidated-budgets)** и **ГИИС «Электронный бюджет»**.

## Периодичность обновления

**По каждому набору отдельно** (частота в паспорте на data.mos.ru). Портал как целое обновляется непрерывно по мере публикаций ведомств.

## Ограничения и особенности

- Качество и полнота **различаются по владельцам** наборов: перед построением рядов проверяйте **дату последнего изменения** и **описание полей**.
- Не смешивайте **город Москва** с **Московской областью** — это разные субъекты РФ и разные каталоги open data.
