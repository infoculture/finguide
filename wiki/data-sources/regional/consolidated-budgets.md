# Консолидированные бюджеты субъектов РФ

| Признак | Значение |
| :--- | :--- |
| Уровень и охват | **Консолидированный бюджет субъекта РФ** — публичные отчёты об исполнении по всем субъектам в одном разделе сайта Казначейства (оператор федеральный, объект данных — региональный свод) |
| Тип доступа | Веб-раздел с перечнем субъектов и **файлами по ссылкам** (типовая форма **0503317** и сопутствующие материалы периода) |
| Основные форматы | HTML (оглавление), XLS/XLSX, PDF — по конкретной публикации и году |
| Оператор | Федеральное казначейство |
| Режим доступа | Открытый (публичные страницы без регистрации) |

Коротко: **единая витрина** Федерального казначейства для скачивания **исполнения консолидированных бюджетов субъектов РФ** (в первую очередь форма **0503317**) — удобный вход для **межрегиональных сравнений** и построения рядов по **КБК** на методике казначейской отчётности, без обхода только федерального контура «Исполнение бюджетов».

## Описание

Раздел **`roskazna.gov.ru`** «**Консолидированные бюджеты субъектов**» агрегирует ссылки на **ежемесячные и годовые** материалы исполнения по каждому субъекту РФ в логике **[консолидированного бюджета](/glossary/consolidated-budget)** (в свод входят бюджет субъекта, **ТФОМС** и **местные бюджеты** по правилам консолидации — см. **[форму 0503317](/reporting/0503317)**).

**Сознательно не входит**: внутренние казначейские кабинеты, построчные платёжные документы, плановые и утверждённые показатели **[региональных бюджетов](/data-sources/federal/regionalnye-byudzhety)** на `budget.gov.ru` и сайтах субъектов — это **другой контур** (план / иная подача), его нельзя подменять файлами исполнения без методики.

## Оператор

**[Федеральное казначейство](/organizations/federal-treasury)** — канонический URL раздела: https://roskazna.gov.ru/ispolnenie-byudzhetov/konsolidirovannye-byudzhety-subektov-rossijskoj-federacii

- Карточка ИС: **[Портал Федерального казначейства](/information-systems/federal/roskazna-portal)** — витрина оператора; ограничения сайта, смена URL и сопоставление с «Электронным бюджетом» — в разделе **«Ограничения и актуальность»** той карточки.

## Объекты данных и показатели

- **Строка формы исполнения** — агрегаты по **[КБК](/glossary/kbk)** (доходы, расходы, источники финансирования дефицита, долг — по структуре конкретной версии **[0503317](/reporting/0503317)** и отчётного периода).
- **Единица выборки** — как правило **субъект РФ × период (месяц/год)** в файле отчёта; детализации до контрагента или контракта в типовой публичной подаче **нет**.
- **Ключи связки** — **КБК**, **финансовый год**, **месяц** (или годовой срез), **код/наименование субъекта** в метаданных файла или имени вложения; для трансфертов и источников финансирования — совместно с **[0503387](/reporting/0503387)** по методике периода.

**Пробелы**: прямые URL на конкретные файлы **нестабильны** — ориентируйтесь на актуальное оглавление раздела и дату публикации; состав столбцов и редакция формы меняются — сверяйте с пояснениями оператора к отчёту.

## Периодичность обновления

По смыслу отчётности — **ежемесячно** (оперативное исполнение) и **за год** (годовой комплект), в том же регламенте, что и прочие формы раздела «Исполнение бюджетов». Точные **сроки выхода** конкретной формы по субъекту и изменения перечня публикаций сверяйте с **актуальным оглавлением** на странице оператора.

Дополнительно про типичные задержки витрины и смену структуры меню — см. **«Ограничения и актуальность»** в **[Портал Федерального казначейства](/information-systems/federal/roskazna-portal)**.

## Ограничения и особенности

- Условия использования и переработки — в сопроводительном тексте к форме или в карточках наборов **[открытых данных Казначейства](https://roskazna.gov.ru/opendata/)**; обычно требуется **указание источника**.
- Показатели **не тождественны** построчно срезам **[ГИИС «Электронный бюджет»](/information-systems/federal/giis-eb)** без согласования **периода, показателя и методики** (касса / план / иной срез).
- Для автоматизации загрузок полезен обзорный howto **[Скачивание отчётов Казначейства](/howto/access/roskazna-reports)** — шаблон обхода веб-витрины; конкретные селекторы и URL проверяйте на дату запуска.

## Не путать с

- **[Региональные бюджеты](/data-sources/federal/regionalnye-byudzhety)** — прежде всего **плановый** и нормативный контур (законы о бюджете, открытые данные портала бюджетной системы); **0503317** — **фактическое исполнение** консолидированного свода по методике Казначейства.
- **[Муниципальные бюджеты (портал)](/data-sources/regional/municipal-budgets)** — объект **отдельного МО** на `budget.gov.ru`; **0503317** даёт **региональную консолидацию**, включающую местные бюджеты субъекта, но не полную открытую детализацию по каждому МО.
- **[Отчёты Федерального казначейства](/data-sources/federal/roskazna-reports)** — эта карточка описывает **узкий подраздел** только про консолидированные бюджеты субъектов; общий перечень форм и форматов — там.

## Частые вопросы

**В:** Почему суммы в **0503317** не совпадают с показателями «закона о бюджете субъекта» на региональном сайте?

**О:** Чаще всего сравнивают **разные объекты**: консолидированный свод **шире** бюджета субъекта (включает ТФОМС и местные бюджеты по методике), плюс возможны различия **периода** (месяц vs год) и **показателя** (план vs кассовый факт). Сверяйте определения в **[форме 0503317](/reporting/0503317)** и в **[консолидированном бюджете](/glossary/consolidated-budget)**.

**В:** Можно ли стабильно скачивать все субъекты одним API-запросом?

**О:** Публичный контур — **HTML-оглавление и файлы по ссылкам**; универсального документированного API «все 0503317» здесь не фиксируется. Для регулярной автоматизации комбинируйте разбор оглавления (см. **[howto](/howto/access/roskazna-reports)**), **[datamarts](/data-sources/federal/roskazna-datamarts)** и **[opendata](https://roskazna.gov.ru/opendata/)**, проверяя метаданные наборов.
