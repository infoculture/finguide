Коротко: это **тематическая** карточка источника: машиночитаемые слои **федеральной бюджетной росписи** разнесены по двум основным витринам — **[открытым данным Минфина](/data-sources/federal/minfin-opendata)** и **[наборам портала budget.gov.ru](/data-sources/federal/budget-gov-ru-datasets)**. Используйте её, когда нужно связать строки росписи с **[КБК](/glossary/kbk)**, **[ЛБО](/glossary/lbo)** и исполнением по кассе (**не подменяя план фактом**).

## Описание

**Бюджетная роспись** федерального бюджета — плановый документ (совокупность сведений), фиксирующий распределение бюджетных ассигнований, предельных объёмов финансирования и показателей по расходам и источникам финансирования дефицита по главным распорядителям, распорядителям и получателям в разрезе кодов бюджетной классификации. В открытых данных строки росписи встречаются как **отдельные именованные наборы** и как **колонки в смежных таблицах** исполнения и справочников; состав полей и название набора меняются по годам и редакции портала.

Карточка **не** описывает внутренние кабинеты участников бюджетного процесса и не заменяет юридически значимый текст закона о бюджете в приложениях PDF/XLS — для построчного чтения утверждённого закона см. материалы **[федеральных бюджетов](/data-sources/federal/minfin-opendata)** и разделы портала ГИИС.

## Оператор

**Минфин России** и **Федеральное казначейство** обеспечивают публичный контур **[ГИИС «Электронный бюджет»](/information-systems/federal/giis-eb)** (`budget.gov.ru`), включая открытые данные и API. Отдельный федеральный реестр машиночитаемых наборов по бюджетной тематике ведётся на сайте Минфина — см. **[открытые данные Минфина России](/data-sources/federal/minfin-opendata)**.

- Карточка ИС: [ГИИС «Электронный бюджет»](/information-systems/federal/giis-eb) — операционная и публичная витрина бюджетной росписи, исполнения и смежных реестров в модели единого портала.

## URL и точки доступа

Канонического одного URL «только роспись» нет — выбирайте набор по названию в каталоге или используйте API/API-документацию портала.

| Тип доступа | URL | Формат |
| --- | --- | --- |
| Единый портал бюджетной системы | https://budget.gov.ru/ | HTML; переход к разделам данных и открытым наборам |
| Открытые данные портала (раздел «Развитие») | https://budget.gov.ru/Развитие/Открытые-данные | CSV, JSON и др. (по паспортам наборов) |
| API единого портала | https://budget.gov.ru/Развитие/API | JSON |
| Реестр открытых данных Минфина | https://minfin.gov.ru/ru/opendata/ | HTML; карточки наборов со ссылками на CSV/XML |
| Сводный перечень наборов Минфина | https://minfin.gov.ru/opendata/list.csv | CSV (строка набора: идентификатор, название, ссылка на `meta.csv`) |

## Объекты данных и показатели

- **Строка выгрузки** обычно означает сочетание **финансового года**, **кодов бюджетной классификации** ([КБК](/glossary/kbk)), **административных кодов** главного распорядителя / распорядителя / получателя (в зависимости от набора), **суммы ассигнований или предельного объёма финансирования** и иных плановых показателей по методике набора.
- **Связки**: коды **ГРБС** и подведомственности — для сопоставления с **[госпрограммами и ФЦП](/data-sources/federal/fcp)** и **[субсидиями](/data-sources/federal/subsidy)**; для межбюджетных строк могут понадобиться территориальные коды (см. справочники в тех же витринах).
- **Пробелы**: детализация до **конкретного учреждения** и внутренние **ЛБО** по цепочке доведения в полном виде в открытой выгрузке **не гарантированы** — см. [ЛБО](/glossary/lbo); факт кассы смотрите в **[отчётах Казначейства](/data-sources/federal/roskazna-reports)** и **[исполнении бюджета](/data-sources/federal/otchety-ob-ispolnenii-byudzhetov)**, а не в одной только росписи.

## Периодичность обновления

Периодичность и дата среза задаются **паспортом конкретного набора** в реестре Минфина и на страницах открытых данных `budget.gov.ru`; при автоматизации опирайтесь на поля метаданных набора, а не на фиксированный календарь.

Общие ограничения смены состава наборов и URL описаны в карточке ИС — см. раздел **«Ограничения и актуальность»** в [ГИИС «Электронный бюджет»](/information-systems/federal/giis-eb).

## Ограничения и особенности

- **План и факт**: роспись отражает **плановые** распределения на выбранную дату; кассовое исполнение и отчётность распорядителей следует брать из казначейских и иных источников исполнения.
- **Уровень агрегации** в разных наборах может отличаться (ведомство, функциональный разрез, код получателя) — перед сводкой проверьте методику в паспорте набора.
- **Лицензия**: открытые государственные данные — как правило, допускают переработку с указанием источника; уточняйте в карточке набора на дату скачивания.
- **Стабильность URL**: прямые ссылки на файлы могут меняться при обновлении реестра; для пайплайнов предпочтительны API и каталоги с версией набора.

## Примеры доступа

- Через каталог: откройте [реестр открытых данных Минфина](https://minfin.gov.ru/ru/opendata/), найдите набор по словам «бюджетная роспись» в названии или описании и используйте ссылки на файлы из карточки набора.
- Через API портала `budget.gov.ru`: см. пошаговые примеры в [Руководстве: API budget.gov.ru](/howto/access/budget-gov-api).

## Частые вопросы

**В:** Почему суммы в выгрузке росписи не совпадают с кассовым отчётом за месяц?  
**О:** Роспись задаёт **плановые** лимиты и распределение на выбранный срез; кассовое исполнение зависит от платежей через казначейство и методики отчёта. Сопоставляйте одинаковые **показатели**, **год** и **уровень детализации**.

**В:** Где бюджетная роспись главного распорядителя как документ ведомства?  
**О:** В открытых данных чаще доступны **агрегированные** федеральные таблицы; полнота ведомственной детализации зависит от набора. Для связки плана ведомства с отчётностью распорядителя см. **[консолидированную отчётность ГРБС](/reporting/grbs-consolidated-reporting)**; **не путайте** роспись ГРБС с **[бюджетной сметой казённого учреждения](/glossary/budget-estimates)** — это разные уровни плановых документов.
