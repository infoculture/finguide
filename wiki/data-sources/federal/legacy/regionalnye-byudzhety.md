# Региональные бюджеты (сайты и витрины субъектов РФ)

| Признак | Значение |
| :--- | :--- |
| Уровень и охват | Бюджеты **субъектов Российской Федерации**: законы, проекты, пояснения и отчётность, опубликованные **на региональных сайтах** и дублируемые или дополняемые **федеральными витринами** (`budget.gov.ru`, открытые данные Минфина) |
| Тип доступа | Смешанный: официальные сайты органов субъекта (HTML и файлы), навигация к материалам субъектов на `budget.gov.ru` (в т.ч. раздел «Регионы»), каталоги открытых наборов и API портала |
| Основные форматы | HTML, PDF, DOC/DOCX, XLS/XLSX на сайтах субъектов; CSV/JSON в контуре «Электронного бюджета» — по паспортам конкретных наборов |
| Оператор | Органы государственной власти субъекта РФ (финансовый блок, правительство); федеральная агрегация — **Минфин России** и **Федеральное казначейство** в рамках **[ГИИС «Электронный бюджет»](/information-systems/federal/giis-eb)** |
| Режим доступа | Открытый для размещённых публично материалов и опубликованных наборов |

Коротко: ориентиры для поиска **полного пакета** регионального бюджетного цикла (проект → закон → исполнение) на **сайте субъекта**, с опорой на **единый портал** и выгрузки, когда нужны **машиночитаемые** срезы или сверка с **казначейским исполнением** и **[консолидированным бюджетом](/reporting/0503317)**.

## Описание

Бюджет субъекта РФ по-прежнему оформляется **нормативно** (закон о бюджете, изменения) и сопровождается таблицами в PDF и табличных форматах. Параллельно действует требование раскрывать бюджетную информацию в **[ГИИС «Электронный бюджет»](/information-systems/federal/giis-eb)** (`budget.gov.ru`): паспорт региона, агрегированные показатели, разделы открытых данных и конструктор наборов. На практике аналитику полезно смотреть **оба контура**: на региональном сайте часто полнее **пояснительные записки**, решения законодательного органа и локальные акты; на портале — **единые коды**, справочники и выгрузки для стыковки с другими федеральными источниками.

Типичные места публикации на стороне субъекта:

- региональный **портал открытого бюджета** или **портал открытых данных**;
- при отсутствии отдельной витрины — сайт **департамента (комитета, министерства) финансов** или **правительства (администрации)** региона.

Часто выкладывают:

- основные характеристики и показатели бюджета;
- проект закона о бюджете субъекта РФ;
- законы о бюджете и об изменениях к нему;
- «бюджет для граждан» и иные разъяснительные материалы;
- нормативные акты о внесении изменений в бюджет.

На `budget.gov.ru` и в **[открытых данных Минфина](/data-sources/federal/minfin-opendata)** удобно брать **сводные** ряды по доходам, расходам, источникам финансирования дефицита и долгу; **детальные реестры региональных субсидий** и адресная поддержка зачастую требуют **региональных** разделов или **[реестра соглашений](/data-sources/federal/subsidy)** в каталоге портала — проверяйте метаданные набора.

**Исполнение** по кассе и регламентированные формы консолидированного бюджета субъекта публикует **Федеральное казначейство** — см. **[отчёты Казначейства](/data-sources/federal/roskazna-reports)** и **[консолидированные бюджеты субъектов РФ](/data-sources/regional/consolidated-budgets)**. **Краткие макро-своды** исполнения консолидированных бюджетов **всех субъектов** в **XLSX** выкладывает **Минфин** — **[страница subbud/execute](/data-sources/federal/minfin-subbud-execute)** (не замена формам Казначейства).

Карточка **не** подменяет описание каталога наборов `budget.gov.ru` — для идентификаторов наборов и API см. **[наборы данных ГИИС «Электронный бюджет»](/data-sources/federal/budget-gov-ru-datasets)**.

## Оператор

**Органы государственной власти субъекта РФ** (финансовый блок, высший исполнительный орган) — первичная публикация закона о бюджете и региональных документов; конкретный базовый URL у каждого субъекта свой.

Федеральная витрина и машиночитаемые слои: **Минфин России** и **Федеральное казначейство** — см. канонический вход https://budget.gov.ru/

- Карточка ИС: **[ГИИС «Электронный бюджет»](/information-systems/federal/giis-eb)** — единый портал бюджетной системы; открытые данные и подсистемы, куда должны попадать сведения о бюджетах субъектов при соблюдении норм раскрытия.

## URL и точки доступа

| Тип доступа | URL | Формат |
| --- | --- | --- |
| Единый портал бюджетной системы (в т.ч. паспорт субъекта, разделы данных) | https://budget.gov.ru/ | HTML; переход к открытым наборам и сервисам |
| Ориентиры по региональным витринам «Открытый бюджет» | см. **[Региональные порталы «Открытый бюджет»](/data-sources/regional/regional-portals)** — канонический вход https://budget.gov.ru/, раздел «Регионы»; короткий путь `…/Развитие/Открытый-бюджет` по проверке **404** | HTML |
| Каталог открытых наборов портала | https://budget.gov.ru/epbs/opendata/list.json | JSON |
| Открытые данные Минфина России (агрегированные наборы, в т.ч. по регионам) | https://minfin.gov.ru/ru/opendata/ | HTML; CSV/XML по карточкам наборов |
| Исполнение бюджетов (в т.ч. консолидированные бюджеты субъектов) | https://roskazna.gov.ru/ispolnenie-byudzhetov/ | HTML; вложения XLS/XLSX/PDF |

Базовый URL официального сайта конкретного субъекта в одной строке не фиксируется: используйте **[перечень порталов](/data-sources/regional/regional-portals)** на портале Электронного бюджета или поиск по наименованию региона.

## Объекты данных и показатели

- **Документ законодательства о бюджете** — текст закона, приложения с таблицами по **[КБК](/glossary/kbk)**, распорядителям, источникам финансирования дефицита;
- **Плановые показатели** — проект бюджета, пояснительная записка, «бюджет для граждан»;
- **Агрегированные показатели на портале** — доходы, расходы, долг, межбюджетные операции — в зависимости от набора и метаданных;
- **Исполнение** — регламентированные формы казначейства и консолидированный бюджет субъекта, не смешивайте без проверки методики **план** (закон, роспись) и **факт** (отчётность Казначейства).

Ключи связки: код **субъекта РФ**, **[ОКТМО](/glossary/oktmo)** при территориальных таблицах, **[КБК](/glossary/kbk)**, **[ИНН](/glossary/inn)** участников в реестрах и соглашениях.

Пробелы: неполная загрузка в ГИИС по сравнению с сайтом субъекта; разные структуры меню региональных сайтов; детальная разбивка субсидий может быть только в региональных реестрах, а не в сводных открытых наборах портала.

## Периодичность обновления

На стороне субъекта обновление привязано к **бюджетному циклу** (проект, три чтения, утверждение, корректировки, отчёт об исполнении). Периодичность выгрузок на `budget.gov.ru` и в каталоге открытых данных задаётся **паспортом набора**; общие ограничения витрины — в разделе **«Ограничения и актуальность»** карточки **[ГИИС «Электронный бюджет»](/information-systems/federal/giis-eb)**.

Публикации исполнения на сайте Казначейства — преимущественно **ежемесячно** и **ежегодно** для регламентированных форм; уточняйте по оглавлению раздела **[исполнение бюджетов](/data-sources/federal/roskazna-reports)**.

## Ограничения и особенности

- Нет единого API «все бюджеты субъектов одним запросом»: машиночитаемый доступ идёт **через каталог наборов** и документацию API портала; региональные сайты чаще HTML и файлы по ссылкам.
- Сводные показатели на федеральной витрине **не заменяют** текст закона и полный комплект приложений на сайте органа субъекта.
- Для сопоставления с налоговыми и иными федеральными рядами учитывайте методику распределения доходов между уровнями бюджетной системы — см. **[уровни бюджета](/glossary/budget-levels)**.

## Примеры доступа

Каталог открытых наборов `budget.gov.ru` можно сохранить локально и отфильтровать записи по ключевым словам в названии (например, «бюджет», «субъект», «консолидированный» — по задаче).

```bash
curl -sS "https://budget.gov.ru/epbs/opendata/list.json" -o opendata-catalog.json
```

Пошаговые вызовы API портала — в **[руководстве по API budget.gov.ru](/howto/access/budget-gov-api)**.
