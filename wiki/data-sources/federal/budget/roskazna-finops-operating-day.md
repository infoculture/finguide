# Казначейство: операционный день (материалы раздела «Финансовые операции»)

| Признак | Значение |
| :--- | :--- |
| Уровень и охват | **Федеральный уровень**: страница **операционного дня** в разделе **[«Финансовые операции»](https://roskazna.gov.ru/finansovye-operacii)** Казначейства |
| Тип доступа | Интерактивная **HTML-страница** с выбором **периода дат** (компонент `operation-day` на портале); выдача таблиц и вложений зависит от выбранного интервала |
| Основные форматы | HTML; при формировании выдачи — типично вложения в тех же форматах, что и в соседних ветках (**DOCX**, **XML** по шаблону `storage/operation-day-files/`), если оператор публикует файлы за выбранные дни |
| Оператор | Федеральное казначейство |
| Режим доступа | Открытый |

Коротко: **календарный интерфейс** для просмотра материалов **операционного дня** казначейской системы в публичном разделе финансовых операций; для **стабильных безусловных выгрузок** без выбора дат используйте **[листовые страницы инструментов](/data-sources/federal/roskazna-finops-treasury-account-placements)**.

## Описание

Страница https://roskazna.gov.ru/finansovye-operacii/operacionnyj-den реализована как **Livewire-компонент** с полями «дата с / дата по» и кнопками фильтрации: содержимое таблицы **привязано к выбранному диапазону** и может подгружаться после действий пользователя в браузере.

**Не входит**: автоматическое API без сессии браузера (в карточке не фиксируется); замена **[оперативной отчётности исполнения бюджета](/data-sources/federal/roskazna-reports)**.

- Карточка ИС: **[Портал Федерального казначейства](/information-systems/federal/roskazna-portal)**.

## Объекты данных и показатели

- **Своды операционного дня** за выбранный пользователем период — состав колонок и вложений задаётся текущей версией страницы.
- **Ключи связки** — **календарная дата** (операционный день); интерпретация полей — по заголовкам таблицы и сопроводительному тексту на сайте.

**Пробелы**: машинная выгрузка «всех дней» одним запросом **не описана** как стабильный публичный контракт; возможны **403** без браузерного `User-Agent`.

## Периодичность обновления

Определяется появлением новых записей за соответствующие операционные даты после завершения операционного цикла и публикации на портале; точный регламент — в материалах оператора на странице раздела.

## Ограничения и особенности

- Для **регулярной автоматизации** чаще используют **[каталог открытых данных](/data-sources/federal/roskazna-reports)** (`/opendata/`), **[datamarts](/data-sources/federal/roskazna-datamarts)** или **файловые архивы листовых страниц** с постоянными URL, если они покрывают нужный показатель.
- Не смешивайте «операционный день» в этом разделе с **[кассовым фактом исполнения бюджета](/glossary/treasury-execution)** в **[формах раздела «Исполнение бюджетов»](/data-sources/federal/roskazna-reports)** без методики.
