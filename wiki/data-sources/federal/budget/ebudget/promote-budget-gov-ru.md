# Единая площадка мер финансовой государственной поддержки (promote.budget.gov.ru)

| Признак | Значение |
| :--- | :--- |
| Уровень и охват | Федеральный контур **мер финансовой государственной поддержки** (в т.ч. отборы получателей **субсидий** для бизнеса и иных форм поддержки), о которых ведомства ведут сведения в связке с **ГИИС «Электронный бюджет»** |
| Тип доступа | Веб-площадка: каталог мер, условия, документы отборов; подача заявок и работа в кабинетах — через **Госуслуги** и/или **квалифицированную электронную подпись** (по ролям) |
| Основные форматы | HTML (интерфейс площадки); PDF и иные вложения в карточках мер и регламентах отборов (по конкретной публикации) |
| Оператор | **Минфин России** (официальное описание и методические материалы — на сайте Минфина; площадка — поддомен `budget.gov.ru`) |
| Режим доступа | Публичные описания мер и нормативные ссылки чаще доступны **без входа**; участие в отборе и персональные кабинеты — **по учётной записи** (Госуслуги, ЭП для должностных лиц) |

Коротко: официальная **точка входа для заявителей** по отборам на меры финансовой поддержки из бюджета: условия, календарь и подача заявок в электронном виде. Для **машиночитаемого** учёта уже заключённых соглашений и сумм по **КБК** ориентируйтесь на **[реестр соглашений](/data-sources/federal/subsidy)** и **[наборы budget.gov.ru](/data-sources/federal/budget-gov-ru-datasets)**, а не на HTML площадки отбора.

## Описание

Карточка описывает **публичный веб-доступ** к **единой площадке предоставления мер финансовой государственной поддержки** по адресу https://promote.budget.gov.ru/: сведения о мерах поддержки, типовые сценарии отбора получателей и переходы к подаче заявок. По смыслу разъяснений Минфина площадка предназначена для **юридических лиц**, **индивидуальных предпринимателей** и **физических лиц — производителей** (см. страницу Минфина по ссылке ниже).

Сюда **не** входят: внутренние регламенты ведомств без публикации на площадке, **полный** казначейский факт исполнения, **конкурсный контур президентских грантов** (отдельный оператор), а также **закупочные контракты** в **[ЕИС](/information-systems/federal/zakupki)**.

## Оператор

**Министерство финансов Российской Федерации** — организационно-методический контур площадки; канонический URL площадки: https://promote.budget.gov.ru/

- Карточка ИС: **[ГИИС «Электронный бюджет»](/information-systems/federal/giis-eb)** — экосистема единого портала бюджетной системы (`budget.gov.ru`), к которой относится поддомен `promote`; публичные **файловые наборы** и **API** по-прежнему нужно брать из **[каталога открытых данных](/data-sources/federal/budget-gov-ru-datasets)** по конкретным идентификаторам наборов.

Страница Минфина с описанием площадки, документами и обучающими материалами: https://minfin.gov.ru/ru/ismf/electronic_budget/cost_management/promote/

## Объекты данных и показатели

Для аналитики открытых финансовых данных площадку удобно трактовать как **источник нормативно-содержательного и процессуального слоя**:

- **Мера поддержки / конкурс отбора** — название, ведомство-оператор отбора, условия участия, сроки, ссылки на НПА и шаблоны заявок (в интерфейсе; состав полей **не стабилизирован** для выгрузки как единого открытого набора в этой карточке).
- **Заявитель / получатель** — идентификация в рамках отбора через **Госуслуги** и ЭП; для сквозной склейки с бюджетными реестрами после заключения соглашения используйте **[ИНН](/glossary/inn)**, **[ОГРН](/glossary/ogrn)** и реквизиты **соглашения** из **[реестра](/data-sources/federal/subsidy)**.
- **Связь с бюджетным планом и исполнением** — через коды и соглашения в **[ГИИС «Электронный бюджет»](/information-systems/federal/giis-eb)**; сама площадка отбора **не заменяет** выгрузки `list.json` и паспортов наборов.

**Пробелы:** отдельного публичного **REST API каталога мер** с документацией уровня `budget.gov.ru/epbs/opendata` для этой карточки **не фиксируем**; при автоматизации ориентируйтесь на **[наборы данных портала](/data-sources/federal/budget-gov-ru-datasets)** и **[реестр соглашений](/data-sources/federal/subsidy)** как на машиночитаемые опоры.

## Периодичность обновления

Обновление карточек мер, документов отборов и статусов конкурсов привязано к **регламентам ведомств** и календарю отборов; единого поля «версия каталога для скачивания» в открытом виде, аналогичного `meta.json` наборов `budget.gov.ru`, для площадки **promote** здесь не приводится. Общие ограничения витрин **Электронного бюджета** — в разделе **«Ограничения и актуальность»** карточки **[ГИИС «Электронный бюджет»](/information-systems/federal/giis-eb)**.

## Ограничения и особенности

- **Машиночитаемость:** основной сценарий — **человеко-читаемый** интерфейс и документы; для конвейеров ETL приоритетнее **[каталог открытых данных](/data-sources/federal/budget-gov-ru-datasets)** и **[реестр соглашений](/data-sources/federal/subsidy)**.
- **Доступ роботов:** публичные запросы без браузерного контекста могут получать **ошибки доступа** (например, 403) — не закладывайте несогласованный скрейпинг как стабильный канал.
- **Правовая квалификация:** формулировка «субсидия» на витрине поддержки **не заменяет** проверку института по **[субсидиям в смысле БК РФ](/glossary/subsidies)** и тексту конкретного соглашения.

## Не путать с

- **[Реестр соглашений](/data-sources/federal/subsidy)** — **договорной** и регистрационный слой в ГИИС; площадка **promote** отвечает за **процедуру отбора** и работу заявителя до/вокруг заключения соглашения.
- **[Президентские гранты (pgrants)](/data-sources/civil/pgrants)** — отдельная витрина и открытые данные Фонда президентских грантов.
- **[ЕИС в сфере закупок](/information-systems/federal/zakupki)** — контракты закупок товаров, работ, услуг; не является площадкой мер финансовой поддержки по смыслу карточки.
