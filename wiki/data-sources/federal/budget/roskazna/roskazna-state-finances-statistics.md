# Казначейство: статистика государственных финансов РФ (СГФ по методологии МВФ)

| Признак | Значение |
| :--- | :--- |
| Уровень и охват | Федеральный; **статистика государственных финансов (СГФ)** России в редакции Федерального казначейства по **методологии МВФ (РСГФ 2014 / GFSM)**: сектора государственного управления, единицы — млрд руб., метод учёта — кассовый |
| Тип доступа | Веб-страница с фильтром (Livewire) и прямыми ссылками на файлы; API нет |
| Основные форматы | `.xls` (BIFF, основная масса), `.xlsx` (годовые и часть месячных 2024+), `.pdf`; архивы `.zip` (годовые 2009–2013, 2020, 2022), `.7z` (месячные 2017–2021), `.rar` (отдельные месяцы 2017) |
| Оператор | [Федеральное казначейство](/organizations/federal-treasury) |
| Режим доступа | Открытый (браузерный `User-Agent` обязателен — без него сервер отдаёт **HTTP 403**; сертификат сайта — российский УЦ, `curl` по умолчанию падает на TLS — нужен `-k` или сертификат Минцифры) |

Коротко: **официальные таблицы СГФ** Казначейства в **методологии МВФ (GFSM/РСГФ 2014)** — доходы, расходы, источники и использование денежных средств, баланс активов и обязательств по секторам госуправления (ЦП / ССО / регионы / местные / консолидация СГУ). Это **не** то же, что казначейское исполнение бюджетов по БК: методики и периметр консолидации различаются, прямая сверка невозможна без методического моста. Для статистики Росстата/ЕМИСС и сводных таблиц Минфина используйте карточки [Росстата](/data-sources/federal/rosstat-finances-of-russia) и [Минфина](/data-sources/federal/minfin-fedbud-execute) — это иные операторы и иная рамка наблюдения.

## Описание

Карточка описывает подраздел **«Статистика государственных финансов РФ»** раздела [«Исполнение бюджетов»](https://roskazna.gov.ru/ispolnenie-byudzhetov/) портала Казначейства: https://roskazna.gov.ru/ispolnenie-byudzhetov/statistika-gosudarstvennyh-finansov-rf/. Это **статистические** публикации по госфинансам в **методологии МВФ** — единая анкета СГФ (РСГФ 2014 / *Government Finance Statistics Manual*), которую Казначейство готовит и одновременно направляет в МВФ (на это указывает код страны `922` и шаблоны `922GYQ14…`, `gfsyq14r…`, `IFS REPORT FORM` в метаданных файлов).

Витрина фильтрует файлы двумя признаками через Livewire-компонент `files-list-with-month-filter`:

- **Вид отчёта** (`filter_type`): `48` Ежемесячный, `50` Ежеквартальный, `49` Годовой;
- **Год** (`filter_year`): диапазон зависит от вида — месячные 2013–2026, квартальные 2016–2026, годовые 2009–2024.

В объём карточки **не** входят казначейские формы исполнения бюджетов по БК (это карточка [Федеральный бюджет на портале Казначейства](/data-sources/federal/roskazna-federal-budget-execution)), отчёты об исполнении бюджетов БО / региональные своды ([отдельная ветка](/data-sources/federal/otchety-ob-ispolnenii-byudzhetov)) и витрины данных Казначейства для исторических рядов ([datamarts](/data-sources/federal/roskazna-datamarts)).

**Ключевые разрывы и особенности архива:**

- **Месячные**: нет файла за **сентябрь 2014** (между «Август» и «Октябрь» 2014); 2013–2014 гг. выложены «пачкой» 29.12.2014 с шаблонными датами публикации (`1 апреля 2013/2014`); два разных файла имеют одинаковое имя `mvf_aprel.xls` (2013 и 2014) — различаются только путём `/b61/…` и `/189/…` в URL, поэтому качать надо по полному `href`, а не по имени.
- **Квартальные**: **отсутствуют 4 кв 2016 и 4 кв 2017** (за эти годы есть только 1–3 кв); с 2018 г. — полный набор 1–4 кв.
- **Имена версий** указывают на пересмотры: `_с`/`_c` (латиницей), `_изм` (изменённый), `_уточ` (уточнённый), `_обн` (обновлённый), `_нов` (новый), `_iiS`. О смысле `_с` см. «Структура файла».

## Объекты данных и показатели

**Строка набора** — связка **«код показателя × сектор государственного управления × период»**: значение показателя (млрд руб.) для заданного сектора в заданном месяце/квартале/году.

| Элемент | Тип | Описание |
| --- | --- | --- |
| `period` | месяц/квартал/год | Из заголовка `922 - РОССИЯ (MM.ГГГГ)` (месячные) или листа `Sources & Uses of Cash` колонки «Январь-Март…Октябрь-Декабрь» (квартальные), или CoverPage «Год: ГГГГ» (годовые). Месячные и квартальные значения — **потоки за период**, не нарастающим итогом |
| `code` | строка/число | Код показателя по РСГФ 2014 (см. таблицу кодов ниже): `1`, `11`…`14`, `2`, `21…28`, `CIO`, `31`, `31.1`…, `CSD`, `32x`, `3202…3208`, `33`, `3302…3308`, `331`, `332`, `NFB`, `NCB`, `CSDz`, `2M`, `63.0`, `6М3…6202.0`. В квартальных/годовых анкетах те же показатели детализированы (`11` → `111…116` → `1111…1113` и т. д.) |
| `sector` | код/имя | Сектор госуправления по GFSM (см. таблицу секторов) |
| `value` | float | Значение в **млрд рублей**. Допускаются **отрицательные** значения (например, профицит/дефицит `CSD`, чистое приобретение финактивов `32x`, социальные фонды с консолидацией); в выборках встречаются аномальные/preliminary значения — см. «Ограничения и особенности» |
| `units` | константа | Млрд рублей (метод учёта — **кассовый**, `к`/`C` в строке «Метод учета:») |
| `territory` | константа | `922` — Россия (код МВФ) |

**Ключи связки с другими наборами wiki:** **год/квартал/месяц** и **сектор**. Прямого ключа по КБК/ОКТМО/ИНН у СГФ **нет** — это агрегированная статистика, а не первичный учёт. Соотнесение с казначейским исполнением бюджетов возможно **только через методический мост**: СГФ ≠ исполнение по БК (разные определения доходов/расходов, периметр консолидации, валовые/чистые показатели). Примеры точек сравнения — [Федеральный бюджет на портале Казначейства](/data-sources/federal/roskazna-federal-budget-execution) и [Сводный бюджет РФ (Минфин)](/data-sources/federal/minfin-fedbud-execute).

**Известные пробелы:**

- Нет непрерывного помесячного ряда по единой схеме секторов: состав колонок менялся (8 колонок в 2013, 9 колонок с 2015/2017 — см. «Структура файла»).
- Нет машиночитаемого словаря кодов и секторов в самих файлах — расшифровка берётся из заголовков строк/колонок и Руководства МВФ.
- Квартальные и годовые анкеты содержат не только денежные потоки, но и балансовые показатели (сток на конец периода) — смешивать поток/сток нельзя без указателя «природа данных».
- `_с`-версии и `_изм`/`_уточ`-версии могут заменять предыдущие — для длинного ряда нужно брать **последнюю** версию каждого периода.

## Структура файла

Файлы трёх видов устроены по-разному — это **три разных шаблона IMF-анкеты**.

### 1. Месячный файл (единый лист «отчет на русcком.»)

**Формат и кодировка.** `.xls` (BIFF8), cp1251, автор в метаданных — «2107» / Финансовое казначейство. Часть файлов с расширением `.xls` реально являются **`.xlsx`** (например, `Октябрь_2025.xls`, `Sentyabr_2024_s.xlsx`) — `xlrd` их не откроет, нужен `openpyxl`/`pandas`. Для чтения BIFF используйте **`xlrd`** (не `openpyxl` — он не поддерживает старый формат).

**Шапка и сектора** (многострочная, на 2026 г. — 9 секторов):

```
922 -  РОССИЯ  (MM.ГГГГ)                       ← заголовок (территория 922 = Россия, месяц/год)
Отчетность по статистике государственных финансов…
(в миллиардах рублей)
┌─────────────────────────────────────────────┬───────────────────────────┐
│         Центральное правительство           │   Сектор гос. управления   │
│ Бюджетные │ Внебюдж. │ Консол. │ Центр.  │  │ Консол. │  GG             │
│ единицы   │ единицы  │  ЦП     │ пр-во a/│…│  СГУ    │                 │
│  (1)   │  (2)   │  (3)  │  (4)   │ (5) │ (6) │ (7) │ (8) │ (9)  │
Метод учета:  к      к      к     к     к    к    к    к    к
```

Имена секторов и количество колонок **эволюционировали** (проверено на файлах за разные годы):

| Период | Колонки | Расположение секторов |
| --- | --- | --- |
| **2013** | 8 | `(1)` Бюджетные ед. ЦП, `(2)` **Фонды соцобеспечения внутри ЦП**, `(3)` консолидация ЦП, `(4)` Центральное пр-во, `(5)` Региональные, `(6)` Местные, `(7)` консолидация, `(8)` СГУ |
| **2015–2016** | 9 | `(1)` Бюджетные ед. ЦП, `(2)` Внебюджетные ед. ЦП, `(3)` **Фонды соцобеспечения**, `(4)` консолидация ЦП, `(5)` Центральное пр-во, `(6)` Региональные, `(7)` Местные, `(8)` консолидация, `(9)` СГУ |
| **с 2017 г. (актуальный)** | 9 | `(1)` Бюджетные ед. ЦП, `(2)` Внебюджетные ед. ЦП, `(3)` консолидация ЦП, `(4)` Центральное пр-во a/, `(5)` **Фонды соцобеспечения** (параллельно ЦП), `(6)` Региональные, `(7)` Местные, `(8)` консолидация СГУ, `(9)` Сектор гос. управления |

Каноническое сопоставление колонок (по годовому шаблону GFSM 2014): `(1)` BCG, `(2)` EBG, `(3)` CGC, `(4)` CG, `(5)` SSG, `(6)` RG, `(7)` LG, `(8)` GGC, `(9)` GG; параллельная колонка `CG01` (справочно). **`a/`** — сноска к колонке «Центральное пр-во» (текст сноски в файлах не расшифрован; по методологии МВФ уточняет периметр ЦП), **`b/`** — сноска к СГУ.

**Заголовки и служебные строки.** До таблицы данных идут 4–6 строк заголовков; строка нумерации `(1)…(9)` и строка «Метод учета:» лежат на разных позициях в разные годы (`numrow = 6` в 2022, `7` в 2020/2023–2026, `8` в 2013–2019). Программно искать начало данных лучше по **первой строке с кодом `1`** в col 0, а колонки секторов — по строке, где col 2 = `(1)`. Единицы — «в миллиардах рублей» (в 2026 г. по-русски, до 2025 г. — «in billions of rubles»). Версии **без** `_с` дополнительно содержат колонки «Горизонтальные проверки» (`=(1)+(2)+(3)`, `=(4)+…+(8)`) и строку «Вертикальная проверка» в хвосте — это служебные столбцы самоконтроля.

**Смысл суффикса `_с`** (проверено сравнением `Март_2026.xls` и `Май_2026_с.xls` и серии за 2020–2026): официальной расшифровки на сайте **нет**; наблюдаемая разница структурная — файл **с `_с`** это «чистая» публикационная таблица (**11 колонок**, без столбцов проверок), файл **без `_с`** — рабочая версия с теми же данными **плюс 2–3 служебные колонки** (`Горизонтальные проверки`, `Вертикальная проверка`) и лишние пустые строки. Исторически та же буква встречалась как латиницей (`mvf_iyun_2016_c.xls`, `mvf_mart_2017_c.xls`), так и кириллицей (`_с`). Для парсинга ориентируйтесь на **детект числа колонок/наличия «Горизонтальные проверки»**, а не на суффикс.

**Реальный фрагмент** `Май_2026_с.xls` (60 строк × 11 колонок; лист «отчет на русcком.»):

### 2. Квартальный файл (IMF-анкета High Frequency Template, 7 листов)

Метаданные (`Title: High Freqency Template_Nov21`) и структура — анкета МВФ; файлы за разные годы однотипны (`otchet-za-1-kv-2017.xls` и `Отчет за 1 кв 2026.xls` совпадают по набору листов). Листы:

| Лист | Что содержит |
| --- | --- |
| `Instructions` | Инструкции по заполнению |
| `CoverPage` | Страна (Россия), код `922`, год, валюта (млрд руб.), отчётный период |
| `Balance Sheet` | Баланс активов и обязательств — **стоки** на конец квартала (колонки «Январь-Март», «Январь-Июнь», «Январь-Сентябрь», «Январь-Декабрь» — нарастающие) |
| `Stmt of Govt Operations` | Отчёт об операциях госуправления — доходы/расходы (потоки за квартал) |
| `Sources & Uses of Cash` | **Основной лист**: «Отчёт об источниках и использовании денежных средств» — потоки за квартал; колонки «Кв.1…Кв.4», в файле за квартал *K* заполнены колонки **1…K** (остальные `0`/пусто). Коды РСГФ 2014 с детализацией (`11`→`111…116`, `3301` СДР и т. д.) |
| `Control` | Технический лист (имена диапазонов) |
| `Report Form` | Служебный |

Значения в квартальном файле — **потоки за отдельный квартал**, не нарастающим итогом (проверено на `Таблицы СГФ за 4 кв 2025.xls`: receipts `1.0` = 18545 / 16507 / 21772 / 22747 по кварталам — колеблются, не растут). Для построения ряда за год берите **последний** файл квартала за нужный год (он содержит все четыре колонки, возможно с пересмотром ранних кварталов).

### 3. Годовой файл (анкета GFSM 2014 — GYQ14)

Полный годовой вопросник МВФ (`GFSM2014_V1.7`): **24 листа** — `Coverpage`, `StatementI–IV`, `Table1–9`, `Annex1–2`, листы проверок (`Consolidation Checks`, `Cash Data Checks Report`), `Report Form`. `Table1` (ДОХОДЫ) содержит тот же 9-секторный layout, что и месячные файлы, с полным кодированием GFSM (`GR_G14`, `GRT_G14`, `GRTI_G14…`). **Внимание на единицы**: ярлык `In Million of` в шапке `Table1` — **артефакт шаблона**, реальный масштаб задан на `Coverpage` («Масштаб: Billion») и подтверждается порядком значений (ДОХОДЫ GG за 2024 ≈ 79 748 = ~79,7 трлн руб.). Годовые файлы 2009–2013 — старая анкета (GFSM 1986/2001) с 30 листами, включая листы `…TS` (time series); 2009–2013 упакованы в `YYYY_rus.zip` с пояснительной запиской `.doc`.

С 2019/2020 г. публикуются **две годовые серии**: **«СГУ»** (сектор государственного управления / *general government*, GG — полный вопросник `922GYQ14…`) и **«ГС»** (государственный сектор / *public sector* — отдельный файл `SGF_GS_<год>.xls` только с балансом). «2022 СГУ» на сайте ссылается на английскую версию для МВФ (`922GYQ14_en_2022-imf.xlsx`).

## Периодичность обновления

**Месяц → квартал → год** по мере подготовки отчётности:

- **Месячный** файл за месяц *M* — около 20-го числа *M+1* (фактически: «Май 2026» → 13.07.2026, «Март 2026» → 20.05.2026, «Февраль 2026» → 10.04.2026).
- **Квартальный** — в течение ~30–90 дней после конца квартала (1 кв 2026 → 24.06.2026, 4 кв 2024 → 12.05.2025).
- **Годовой** — в декабре следующего года (за 2024 → 13.12.2025; за 2023 → 02.12.2024).

Прямые URL файлов стабильны и не меняются при появлении новых периодов (старые не удаляются). Сервер отдаёт `Last-Modified`, но у мигрированных файлов оно равно дате миграции (`08.10.2025`), поэтому надёжный признак свежести — текст карточки на странице.

## Ограничения и особенности

- **Доступ для скриптов**: без браузерного `User-Agent` — **HTTP 403** (проверено: `curl` с дефолтным UA, `python-requests`, пустой и короткий `Mozilla/5.0` — все 403). Нужен полноценный браузерный UA. Сертификат сайта — российского УЦ (Минцифры), `curl`/`openssl` с системным хранилищем macOS падают на TLS (`unable to get local issuer certificate`) — используйте `-k` или установите корневые сертификаты Минцифры.
- **Жёсткий рейт-лимит WAF**: интенсивный обход (десятки запросов за пару минут) даёт временный бан «Forbidden / Transaction ID» на **3–5 минут** — ограничивайте параллелизм и делайте паузы.
- **Несопоставимость с казначейским исполнением по БК**: СГФ строится по методологии МВФ (валовые/чистые, кассовый/начисления, периметр консолидации), исполнение бюджетов — по БК РФ. Ряды **не совпадают** между собой и с агрегатами Росстата/ЕМИСС без методического моста (см. [статистику Росстата](/data-sources/federal/rosstat-finances-of-russia)).
- **Расхождения между месячными и квартальными публикациями**: сумма месячных потоков за квартал **не равна** квартальному значению (проверено на Q1 2026: сумма месячных `1.0` GG = 4652.2 + 2025.4 + 10700.4 = 19 378 vs квартальный `1.0` = 17 642.6) — разные периметр консолидации и моменты ревизии. Для длинных рядов используйте **один** вид публикации.
- **Аномальные значения в месячных файлах**: предварительные версии могут содержать экстремальные/отрицательные числа (например, в `Февраль_2026_с.xls` «Налоги» `(1)` BCG = −60.5, «Поступления» `(1)` BCG = 132.7 — нетипично против соседних месяцев); сверяйтесь с квартальным/годовым значением того же периода.
- **Консолидационные колонки не суммируются напрямую**: сумма `(1)+(2)+(3)+(4)` ≠ `(9)` из-за взаимного исключения трансфертов между секторами; колонки `(3)` и `(8)` — **консолидационные** (со знаком минус для исключения двойного счёта), и СГУ `(9)` может быть меньше суммы компонентов.
- **Эволюция схемы секторов**: ряд по секторам 2013–2026 непрерывен только с оговорками (8 колонок → 9; переезд Фондов соцобеспечения). Архив 2017–2021 дополнительно упакован в `.7z`/`.zip` с пояснительной запиской — для извлечения нужна библиотека 7-Zip и декодирование имён файлов в архивах ZIP (cp866→utf-8).
- **`_с`- и `_изм`/`_уточ`-версии**: в инвентаре сосуществуют несколько версий одного периода; для финального ряда берите последнюю по дате публикации.

### б) Чтение месячного `.xls` в tidy-таблицу (Python, `xlrd`)

Для `.xls` (BIFF) — **только `xlrd`** (`openpyxl` старый формат не поддерживает). Кириллическое имя листа и многострочная шапка учтены; код конца алгоритма — детект строки нумерации `(1)` и первой строки данных с кодом `1`.

```python
import xlrd, csv, sys

FILE = 'Май_2026_с.xls'
# Сектора (9-колонный layout, действует с 2017 г.); col-индекс -> сектор
SECTORS = {
    2: 'BCG — бюджетные единицы ЦП',
    3: 'EBG — внебюджетные единицы ЦП',
    4: 'консолидация ЦП',
    5: 'CG — центральное правительство',
    6: 'SSG — фонды соц. обеспечения',
    7: 'RG — региональные органы',
    8: 'LG — местные органы',
    9: 'консолидация СГУ',
    10: 'GG — сектор гос. управления',
}

wb = xlrd.open_workbook(FILE)
sh = wb.sheet_by_index(0)                    # единственный лист «отчет на русcком.»
numrow = next(r for r in range(sh.nrows)
              if str(sh.cell_value(r, 2)).strip() == '(1)')   # строка нумерации (1)..(9)

rows = []
for r in range(numrow + 2, sh.nrows):
    code = str(sh.cell_value(r, 0)).strip()
    label = str(sh.cell_value(r, 1)).strip()
    if not code or not label:
        continue
    for col, sector in SECTORS.items():
        if col < sh.ncols and sh.cell_type(r, col) == 2:       # только числовые значения
            rows.append({'code': code, 'indicator': label,
                         'sector': sector, 'value': sh.cell_value(r, col)})

w = csv.DictWriter(sys.stdout, fieldnames=['code', 'indicator', 'sector', 'value'])
w.writeheader()
for row in rows[:10]:
    w.writerow(row)
```

## Не путать с

- **[Финансы России (Росстат)](/data-sources/federal/rosstat-finances-of-russia)** — официальный статистический орган; сборники Росстата по госфинансам строятся по собственной наблюдательной рамке и не совпадают с казначейской СГФ.
- **[Федеральный бюджет на портале Казначейства](/data-sources/federal/roskazna-federal-budget-execution)** — формы **исполнения** федерального бюджета по БК РФ (оперативные данные, КБК-разрез), а не статистика по методологии МВФ.
- **Годовая анкета «СГУ» vs «ГС»**: на одной странице это **две разные** публикации за один и тот же год — `922GYQ14…` (сектор госуправления, полный вопросник) и `SGF_GS_…` (государственный сектор, только баланс).

## Частые вопросы

**В:** Чем СГФ Казначейства отличается от отчётов об исполнении бюджетов?

**О:** Методологией и периметром. СГФ строится по **Руководству МВФ по статистике госфинансов (РСГФ 2014)** — доходы/расходы/финансирование по секторам госуправления (ЦП, ССО, регионы, местные, консолидация СГУ), кассовый метод, млрд руб. Исполнение бюджетов — по **БК РФ** и формам бюджетной отчётности (КБК-разрез, оперативные данные, разные определения доходов/расходов). Прямой свалки показателей не получится: ряды систематически расходятся (см. «Ограничения и особенности»).

**В:** Как получить ряд за несколько лет?

**О:** Для длинных рядов берите **годовые** анкеты (Table1, сектор GG) — они консистентнее и детальнее (`2009_rus.zip`…`922GYQ14_2024_рус.xlsx`). Для оперативной динамики внутри года — квартальные файлы (файл за последний квартал года содержит все четыре квартальные колонки). Месячные файлы полезны для самой свежей оперативки, но у них менялся состав колонок секторов (8 → 9) и есть пересмотры (`_с`/`_изм`/`_уточ`) — берите последнюю версию периода по дате публикации на странице.

**В:** Почему в таблицах встречаются отрицательные значения?

**О:** Это нормально для СГФ: отрицательные значения бывают у **консолидационных** колонок `(3)` и `(8)` (исключение двойного счёта между секторами), у профицита/дефицита `CSD`, у чистого приобретения финансовых активов `32x` и чистого принятия обязательств `33`. Кроме того, в предварительных месячных версиях встречаются **аномальные** значения (например, отрицательные «Налоги» в `Февраль_2026_с.xls`) — такие лучше сверять с квартальной/годовой публикацией.

**В:** Что входит в охват сектора «государственного управления» (GG)?

**О:** По GFSM 2014 — **центральное правительство** (бюджетные + внебюджетные единицы ЦП + фонды соцобеспечения, консолидированные), **региональные** и **местные органы управления**, консолидированные между собой. Государственный сектор (ГС, «public sector») шире — включает ещё госкорпорации и предприятия с госучастием; он публикуется отдельно (`SGF_GS_…`) и только балансом. Колонка `(9)` GG — это не сумма `(1)+(2)+(3)+(4)+(5)+(6)+(7)+(8)`, а результат после исключения внутри-секторных трансфертов через консолидационные колонки.
