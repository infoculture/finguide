# Единый реестр субъектов МСП — публичный портал (rmsp.nalog.ru)

| Признак | Значение |
| :--- | :--- |
| Уровень и охват | Федеральный; интерактивное представление сведений о субъектах МСП — ЮЛ и ИП — и агрегированной статистики реестра |
| Тип доступа | HTML-витрина; браузерные POST-обработчики JSON поиска и статистики; POST-выгрузки XLSX |
| Основные форматы | HTML `text/html;charset=utf-8`; JSON `application/json;charset=UTF-8`; XLSX `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet` |
| Оператор | [ФНС России](/organizations/fns) |
| Режим доступа | Публичный: тестовые HTTP-запросы без авторизации получили `200`; сервер выдаёт cookie `JSESSIONID` |

Коротко: **rmsp.nalog.ru** — официальная интерактивная витрина того же реестра, который ФНС публикует файловым набором [7707329152-rsmp](/data-sources/federal/7707329152-rsmp). Витрина удобна для единичного или фильтрованного поиска, выбора исторической даты и статистики; файловый набор — для массовой загрузки XML. JSON-обработчики ниже реально работают на дату проверки, но портал не объявляет их стабильным публичным API.

## Описание

В карточке описан именно **канал веб-витрины**: обычный и расширенный поиск, JSON, XLSX и статистика. XML-поля и правила формирования файла не дублируются: они документированы в [7707329152-rsmp](/data-sources/federal/7707329152-rsmp).

В объём не входят учётные роли портала и другой реестр — получатели поддержки на `rmsp-pp.nalog.ru`. Принадлежность к МСП и факт получения поддержки — разные данные.

[Федеральная налоговая служба](/organizations/fns) — оператор и публикатор витрины: https://rmsp.nalog.ru/

### HTTP-заголовки ответов

Значения сняты `curl` 2026-07-19. `Content-Length` у динамических и XLSX-ответов может зависеть от тела; «нет» означает отсутствие заголовка в конкретном ответе, а не обещание сервера.

| URL и проверочный запрос | Статус и `Content-Type` | `Content-Length` / передача | `Last-Modified`, сжатие, cookie |
| --- | --- | --- | --- |
| `GET /` | `200`; `text/html;charset=utf-8` | `Content-Length` нет; `Transfer-Encoding: chunked` | `Last-Modified` и `Content-Encoding` нет; `Set-Cookie: JSESSIONID=…; Path=/; HttpOnly` |
| `POST /search-proc.json`, `mode=quick`, `query=4707048043` | `200`; `application/json;charset=UTF-8` | `Content-Length: 1361` | `Last-Modified` и `Content-Encoding` нет; тот же `JSESSIONID` |
| `POST /report.xlsx` с теми же полями | `200`; `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet` | `Content-Length` нет; `Transfer-Encoding: chunked`; `content-disposition: attachment; filename=rsmp_19.07.2026.xlsx` | `Last-Modified` и `Content-Encoding` нет; `JSESSIONID`; пустые `Pragma`, `Cache-Control`, `Expires` |
| `POST /statistics-proc.json`, `statDate=10.07.2026` | `200`; `application/json;charset=UTF-8` | `Content-Length` нет; `Transfer-Encoding: chunked` | `Last-Modified` и `Content-Encoding` нет; `JSESSIONID` |
| `GET /statistics.xlsx?statDate=10.07.2026` | `200`; XLSX MIME | `Content-Length` нет; `Transfer-Encoding: chunked`; `content-disposition: attachment; filename=rsmp_statistics_19.07.2026.xlsx` | `Last-Modified` и `Content-Encoding` нет; `JSESSIONID`; пустые `Pragma`, `Cache-Control`, `Expires` |
| `GET /developers.html` и `GET /razrabotchikam.html` | `200`; `text/html;charset=utf-8` | `Content-Length` нет; `Transfer-Encoding: chunked` | `Last-Modified` и `Content-Encoding` нет; `JSESSIONID` |

Cookie выставляется даже на главной и JSON-ответах. При этом проверочный `POST /search-proc.json` и `POST /statistics-proc.json` от `curl` без заранее переданной cookie и без специального `User-Agent` получили корректный `200` и JSON. Это не доказывает отсутствие будущих ограничений.

## Структура данных и объекты

### A. Параметры поиска

Обычная форма передаёт `mode=quick`, `query`, `page`, `sortField`, `sort`. Расширенная форма имеет `mode=extended` и следующие имена полей. Значения, коды и обязательность сняты с HTML/клиентского JS формы; прямой сетевой запрос со всеми фильтрами сразу **не** выполнялся, поэтому поведение редких комбинаций помечено отдельно.

| Параметр (HTTP) | Где передаётся | Тип | Обяз. | Семантика | Соответствие полю XML `7707329152-rsmp` |
| --- | --- | --- | --- | --- | --- |
| `mode` | body `application/x-www-form-urlencoded` | строка | да | `quick` или `extended`; определяет вариант формы | Нет поля: технический параметр витрины |
| `query` | body | строка | для обычного поиска да | ИНН, ОГРН, ОГРНИП, наименование ЮЛ или ФИО ИП | Поиск по `ИННЮЛ`/`ИННФЛ`, `ОГРН`/`ОГРНИП`, наименованию/ФИО; не отдельное XML-поле |
| `npType` | body, повторяемый checkbox | enum `UL` / `IP` | нет | Тип субъекта: юридическое лицо или ИП | `Документ/@ВидСубМСП`: `1` / `2`; глава КФХ — отдельный XML-код `3` |
| `vidIp` | body | enum `2` / `1` | нет | В UI при выборе ИП: «является главой КФХ», да / нет | `Документ/@ВидСубМСП`; прямое соответствие кодов **ТРЕБУЕТ ПРОВЕРКИ** |
| `category` | body, повторяемый checkbox | enum `1` / `2` / `3` | нет | Микро / малое / среднее предприятие | `Документ/@КатСубМСП` |
| `dtCategory` | body | дата `ДД.ММ.ГГГГ` | **да** в расширенной форме | Дата среза, по которой определяется категория МСП; в селекторе доступны исторические даты | `Документ/@ДатаСост` |
| `isNew` | body | enum `1` / `0` | нет | Вновь созданный субъект: да / нет | `Документ/@ПризНовМСП` (в XML «да»=`1`, «нет»=`2`; коды витрины отличаются) |
| `isSocial` | body | enum `1` / `0` | нет | Социальное предприятие: да / нет | `Документ/@СведСоцПред` (аналогично кодировка UI отличается) |
| `dtRegistry` | body | дата | нет | Дата включения в реестр | `Документ/@ДатаВклМСП` |
| `region` | body | строка кода субъекта РФ | нет | Регион | `Документ/СведМН/@КодРегион` |
| `district`, `city`, `locality` | body | строковые коды | нет | Последовательно доступные район, город, населённый пункт | Адресные части `СведМН`; точное соответствие кодов **ТРЕБУЕТ ПРОВЕРКИ** |
| `hasLicenses` | body | флаг `1` | нет | Только субъекты с лицензиями | Наличие `Документ/СвЛиценз` |
| `isHitech` | body | флаг `1` | нет | Инновационная/высокотехнологичная продукция | Связано с `СвПрод/@ПрОтнПрод`; точное правило фильтра **ТРЕБУЕТ ПРОВЕРКИ** |
| `isPartnership` | body | флаг `1` | нет | Участие в программах партнёрства | Наличие `Документ/СвПрогПарт` |
| `okved1` | body | строка кода | нет | Вид деятельности ОКВЭД; виджет допускает до 1 000 значений | `СвОКВЭД/СвОКВЭДОсн/@КодОКВЭД` |
| `tru` | body | строковый код | нет | Виды продукции, работ, услуг | `СвПрод/@КодПрод` |
| `slvd` | body | строковый код | нет | Виды лицензируемой деятельности | Поле XML-группы `СвЛиценз`; точная проекция **ТРЕБУЕТ ПРОВЕРКИ** |
| `page` | body | непрозрачный строковый токен | нет | Текущая страница или выбранный размер страницы; берётся из `pageNav` ответа | Нет поля: техническая пагинация |
| `sortField` | body | строка | нет | В проверенном запросе `NAME_EX` | Нет поля: техническая сортировка витрины |
| `sort` | body | `ASC` / `DESC` | нет | Направление сортировки | Нет поля: техническая сортировка витрины |

Каскадный адресный виджет в коде страницы использует `POST /kladr-proc.json` с `parentLevel` (`REGION`, `DISTRICT`, `CITY`, `LOCALITY`) и `parentCode`; его JSON-ответ в этой проверке отдельно не снимался. **ТРЕБУЕТ ПРОВЕРКИ:** фактическая схема ответа и поведение всех уровней.

```json
  "pageNav": {
    "pages": [["1", null]],
    "pageSizes": [["5", "<opaque token>"], ["10", null], ["25", "<opaque token>"], ["50", "<opaque token>"], ["100", "<opaque token>"]]

```

В последней строке примера токены размеров намеренно сокращены: они являются техническими значениями текущей сессии и не нужны для воспроизводимого первого запроса. Для следующей страницы или размера берите токен из **своего** свежего `pageNav`, передавая его как `page`; не конструируйте токен.

### C. XLSX-выгрузка

Кнопка «Экспорт в Excel» переводит текущую HTML-форму на `POST /report.xlsx`. Поля формы в результате поиска: `mode`, `query`, `page`, `sortField`, `sort`; формат body — `application/x-www-form-urlencoded`.

Однако реальная проверка выявила важное текущее ограничение. После обычного UI-поиска по `4707048043` экран показывал одну запись, но `query` в форме результата был пустым. Нажатие кнопки скачало `rsmp_19.07.2026.xlsx` с одним листом `Лист1`, размером **1 × 1**: ячейка `A1` — строка `Единый реестр субъектов малого и среднего предпринимательства по состоянию на 19.07.2026`. Табличного заголовка и строк данных в этом файле не было.

| Колонка (заголовок) | Порядок | Тип ячейки | Соответствует полю JSON/XML |
| --- | ---: | --- | --- |
| Титульная строка `Единый реестр субъектов малого и среднего предпринимательства по состоянию на 19.07.2026` | `A1` | строка | Дата — текущая дата формирования файла, не поле записи |
| Столбцы данных результата поиска | — | — | **ТРЕБУЕТ ПРОВЕРКИ:** в фактически скачанном XLSX их нет, поэтому нельзя достоверно назвать порядок или типы |

Не смешивайте это с HTML-таблицей результата: у неё подтверждены пять видимых колонок — «Наименование субъекта», «Категория», «Регион», «Дата включения в реестр», «Дата исключения из реестра». Они **не** подтверждают схему XLSX.

**ТРЕБУЕТ ПРОВЕРКИ:** максимально допустимое число строк XLSX. В UI подтверждены варианты размера одной страницы `5`, `10`, `25`, `50`, `100`, но в короткой выдаче сообщения о лимите экспорта не было, а выгрузка не содержала строк.

### Статистика

Раздел «Статистика» позволяет выбрать `statDate` из того же наблюдаемого ряда дат. UI отображает число субъектов, работников и продукции; таблицу РФ → федеральных округов → регионов; распределения по ЮЛ/ИП и категориям микро/малое/среднее; социальные и вновь созданные микро-предприятия; диаграммы динамики, численности работников и топ-10 ОКВЭД. Статистика отделена от построчного поиска.

Проверенный программный запрос:

```text
POST https://rmsp.nalog.ru/statistics-proc.json
Content-Type: application/x-www-form-urlencoded

statDate=10.07.2026
```

возвращает JSON-массив. Первая строка РФ содержала, в частности, `stat_date`, `cnt_level: 2`, `cnt_name: "Российская Федерация"`, `cnt_total: 6612445`, `cnt_ul_total: 2022689`, `cnt_ip_total: 4589756`, `cnt_worker_total: 15093124`; у строк округов есть `focode`/`foname`, у регионов — `ssrf` и `name`. Это пригодно для программного извлечения агрегатов. В клиентском коде также есть запросы диаграмм с `level`, `fo`, `region`, `type=chartstat` или `chartokvedstat` и запрос по продукции `?type=tru`; их ответы в этой проверке отдельно не сняты — **ТРЕБУЕТ ПРОВЕРКИ** перед автоматизацией.

## Периодичность обновления

На 2026-07-19 витрина показывала текущую дату `10.07.2026`; в обоих селекторах были видны даты срезов по месяцам, начиная с `01.08.2016`. Для расширенного поиска дата передаётся в `dtCategory`, для статистики — в `statDate`, обе в виде `ДД.ММ.ГГГГ`.

Это наблюдаемая публикационная последовательность, а не заявленное расписание. Для массовой исторической обработки сверяйте и скачивайте соседние ZIP-срезы [7707329152-rsmp](/data-sources/federal/7707329152-rsmp): там дата среза находится в самом файле и в архиве, а не только в контроле UI.

## Ограничения и особенности

- Пагинация выдачи основана на непрозрачных токенах `pageNav`. В UI доступны `5`, `10`, `25`, `50`, `100` строк на странице. В проверенном поиске «Ромашка» `rowCount` был `937`, на первой странице — 10 объектов; следующий переход требует токена из ответа.
- Лимит строк `report.xlsx` и полный перечень XLSX-колонок **ТРЕБУЮТ ПРОВЕРКИ**: UI-экспорт фактической однострочной выборки выдал только заголовок реестра. Не строить массовый процесс на этом экспорте.
- Сервер постоянно устанавливает `JSESSIONID`; поиск и статистика с `curl` работали без заранее сохранённой сессии и без специального `User-Agent`. Требование сохранять cookie для XLSX, лимиты запросов и допустимая частота **не опубликованы** на `developers.html` и **ТРЕБУЮТ ПРОВЕРКИ** нагрузочным тестом/официальным разъяснением.
- CAPTCHA или отдельный anti-bot-челлендж не появились в проверенных браузерных действиях и HTTP-запросах. Это наблюдение, а не гарантия отсутствия защиты для других объёмов или IP.
- Для снятых HTML и JSON-ответов кодировка — UTF-8; заголовка `Content-Encoding` не было. XLSX — бинарный Office Open XML; `Content-Length` у проверенных XLSX не передан, использован `Transfer-Encoding: chunked`.
- В подтверждённой JSON-строке нет КПП, ОКТМО и полного почтового адреса: есть только городские компоненты. Глобальное отсутствие этих полей в недокументированной JSON-схеме **ТРЕБУЕТ ПРОВЕРКИ**; в XML-наборе [7707329152-rsmp](/data-sources/federal/7707329152-rsmp) КПП, ОКТМО и полный адрес также не публикуются.
- Не используйте счётчик главной, `rowCount` результата или предполагаемый Excel как контроль точного объёма полного реестра. Для сверки считайте XML самостоятельно. Численное расхождение счётчиков портала и файлового набора на одну дату в этой проверке **не установлено**.
- Не подменяйте принадлежность к МСП фактом получения поддержки: [7707329152-rsmppp](/data-sources/federal/7707329152-rsmppp) и [rmsp-pp.nalog.ru](/information-systems/federal/rmsp-pp-nalog) описывают меры поддержки, а не критерий включения в реестр МСП.

### Что говорит страница «Разработчикам»

Проверенная `developers.html` не публикует HTTP-спецификацию `search-proc.json`, лимит запросов, обязательный `User-Agent`, список заголовков или правила anti-bot. На ней указаны:

- сервис СМЭВ 3.X «Предоставление сведений из реестра малого и среднего предпринимательства…», зарегистрированный в продуктивной среде 25.08.2016;
- ссылка на каталог открытых данных ФНС `https://www.nalog.gov.ru/opendata/`;
- три архива описаний форматов файлового реестра: `fns-rrmsp406.zip`, `VO_RRMSPSV_2_213_04_04_06.zip`, `VO_SVRRMSPSV_2_213_02_04_08.zip` на `data.nalog.ru`.

Следовательно, это полезный маршрут к официальным файловым контрактам и СМЭВ, но **не** документ для безопасной интеграции с внутренними JSON-обработчиками витрины.

## Примеры доступа

Команды и код ниже исполнены 2026-07-19. Поиск выполняется по публичной записи ЮЛ с ИНН `4707048043`; он выбран для воспроизводимости, а не как рекомендация использовать этот субъект в анализе.

Проверка доступности главной без авторизации:

```bash
curl -sI 'https://rmsp.nalog.ru/' | head -n 7
```

Проверенный признак — `HTTP/1.1 200`, `Content-Type: text/html;charset=utf-8`, `Set-Cookie: JSESSIONID=…` и `Transfer-Encoding: chunked`.

Программный поиск. `curl` сам задаёт `Content-Type: application/x-www-form-urlencoded` при `--data-urlencode`:

```bash
curl -sS -X POST 'https://rmsp.nalog.ru/search-proc.json' \
  --data-urlencode 'mode=quick' \
  --data-urlencode 'query=4707048043' \
  --data-urlencode 'sortField=NAME_EX' \
  --data-urlencode 'sort=ASC'
```

Сокращённый фактический результат: `rowCount: 1`, `data[0].inn: "4707048043"`, `data[0].ogrn: "1224700016750"`, `data[0].category: 1`, `data[0].regioncode: "47"`, `data[0].okved1: "35.22"`; полный пример тела приведён выше. Не передавайте старые `pageNav`-токены в новый запрос.

Потоковый разбор массива `data` и плоский CSV ключевых полей (Python 3.12 standard library; код исполнен против ответа выше):

```python
import csv
import json
import sys
from urllib.parse import urlencode
from urllib.request import Request, urlopen

payload = urlencode({
    "mode": "quick",
    "query": "4707048043",
    "sortField": "NAME_EX",
    "sort": "ASC",
}).encode("utf-8")
request = Request(
    "https://rmsp.nalog.ru/search-proc.json",
    data=payload,
    headers={"Content-Type": "application/x-www-form-urlencoded"},
    method="POST",
)
fields = ["inn", "ogrn", "name_ex", "category", "regioncode", "dtregistry", "okved1"]
writer = csv.DictWriter(sys.stdout, fieldnames=fields)
writer.writeheader()
json_decoder = json.JSONDecoder()
marker = '"data":['
buffer = ""
inside_data = False
with urlopen(request, timeout=30) as response:
    while chunk := response.read(8192):
        buffer += chunk.decode("utf-8")
        if not inside_data:
            marker_at = buffer.find(marker)
            if marker_at < 0:
                buffer = buffer[-(len(marker) - 1):]
                continue
            buffer = buffer[marker_at + len(marker):]
            inside_data = True
        while inside_data:
            buffer = buffer.lstrip()
            if buffer.startswith("]"):
                inside_data = False
                break
            try:
                row, parsed_to = json_decoder.raw_decode(buffer)
            except json.JSONDecodeError:
                break
            writer.writerow({field: row.get(field) for field in fields})
            buffer = buffer[parsed_to:].lstrip()
            if buffer.startswith(","):
                buffer = buffer[1:]
            elif buffer.startswith("]"):
                inside_data = False
                break
            else:
                break
```

Вывод проверки:

```text
inn,ogrn,name_ex,category,regioncode,dtregistry,okved1
4707048043,1224700016750,"ПОТРЕБИТЕЛЬСКИЙ КООПЕРАТИВ ""РОМАШКА-ГАЗ"" ПО ОБСЛУЖИВАНИЮ ЖИЛЫХ ДОМОВ В СНТ ""РОМАШКА""",1,47,10.10.2022 00:00:00,35.22
```

Проверка XLSX, уже скачанного браузером. Код использует `openpyxl==3.1.5`, исполняется и безопасно сообщает о фактическом «титульном» файле; если портал начнёт отдавать таблицу, он нормализует даты в ISO, идентификаторы оставляет строками и создаёт `pandas.DataFrame`.

```python
import sys
from datetime import date, datetime
from pathlib import Path

import pandas as pd                    # pandas==2.2.3
from openpyxl import load_workbook      # openpyxl==3.1.5

path = Path(sys.argv[1])
workbook = load_workbook(path, read_only=True, data_only=True)
sheet = workbook[workbook.sheetnames[0]]
rows = list(sheet.iter_rows(values_only=True))

if len(rows) == 1 and len(rows[0]) == 1:
    raise SystemExit(f"XLSX содержит только титул, без строк данных: {rows[0][0]!r}")

def normalize(value):
    if isinstance(value, datetime):
        return value.isoformat(sep=" ")
    if isinstance(value, date):
        return value.isoformat()
    return value if value is None or isinstance(value, (int, float)) else str(value).strip()

headers = [normalize(value) for value in rows[1]]
records = [dict(zip(headers, map(normalize, row))) for row in rows[2:]]
frame = pd.DataFrame.from_records(records)
print(frame.dtypes)
print(frame.head(3).to_csv(index=False))
```

Запуск против фактически полученного файла завершается сообщением `XLSX содержит только титул, без строк данных: 'Единый реестр субъектов малого и среднего предпринимательства по состоянию на 19.07.2026'`. Это намеренный fail-fast: не интерпретировать титульную строку как набор данных.

Склейка результата поиска с внешней таблицей по ИНН (Python 3.12, `pandas==2.2.3`). Скрипт принимает два CSV: сначала результаты поиска, затем внешние данные; проверяет дубли ИНН в справочнике и использует `validate="m:1"`.

```python
import sys

import pandas as pd

subjects = pd.read_csv(sys.argv[1], dtype={"inn": "string"})
external = pd.read_csv(sys.argv[2], dtype={"inn": "string"})

for name, frame in {"subjects": subjects, "external": external}.items():
    if "inn" not in frame:
        raise ValueError(f"{name}: нет колонки inn")
    frame["inn"] = frame["inn"].str.strip()
    if frame["inn"].isna().any() or (~frame["inn"].str.fullmatch(r"\d{10}|\d{12}")).any():
        raise ValueError(f"{name}: некорректный ИНН")

if subjects["inn"].duplicated().any():
    raise ValueError("subjects: дубли ИНН; сначала выберите один срез/одну запись на субъект")

joined = external.merge(
    subjects[["inn", "ogrn", "name_ex", "category", "regioncode", "okved1"]],
    on="inn",
    how="left",
    validate="m:1",
)
joined.to_csv(sys.stdout, index=False)
```

## Частые вопросы

**В:** Чем эта витрина отличается от `7707329152-rsmp`?

**О:** Это один реестр, но разные каналы. `rmsp.nalog.ru` даёт UI-поиск, исторический выбор даты и агрегаты; `7707329152-rsmp` — полный файловый XML-срез с XSD, пригодный для массовой обработки.

**В:** Как выбрать исторический срез?

**О:** В расширенной форме выберите дату состояния категории МСП: браузер отправляет её как `dtCategory` в формате `ДД.ММ.ГГГГ`. В статистике используется отдельный параметр `statDate`. На дату проверки селекторы содержали даты от `01.08.2016` до `10.07.2026`.

**В:** Можно ли программно выгрузить весь реестр из этой витрины в Excel?

**О:** Не следует рассчитывать на это. Публично подтверждён стабильный JSON-поиск по выборке, но не полный репликационный API; проверенный UI-XLSX даже для одной найденной записи отдал только титул без данных. Для полного реестра берите [7707329152-rsmp](/data-sources/federal/7707329152-rsmp).

**В:** Какой ключ брать для склейки?

**О:** Основной ключ — строковый ИНН (`10` цифр для ЮЛ, `12` для ИП), с проверкой типа субъекта. ОГРН полезен как дополнительный ключ только для ЮЛ; код региона не является ОКТМО.
