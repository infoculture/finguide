Коротко: **О структуре начисления по налогу на добавленную стоимость (Форма № 1-НДС)** — опубликованные ФНС территориальные агрегаты показателей НДС. Для загрузки каждой версии нужны оба файла из её паспорта: `data-*.csv` со значениями и `structure-*.csv` с русскими описаниями и форматами колонок.

## Описание

Карточка относится к каталогу [открытых данных ФНС](https://www.nalog.gov.ru/rn77/opendata/) (идентификатор **7707329152-1nds**). Официальный [паспорт набора](https://www.nalog.gov.ru/opendata/7707329152-1nds/) — единственная точка, с которой следует начинать новый сбор: он публикует текущие URL и список предыдущих выпусков. Датированный URL — ссылка на конкретный выпуск, а не стабильный URL «latest».

В паспорте, проверенном 2026-07-18, указаны текущий выпуск `20260201`, первое опубликование 2017-09-04 и 39 предыдущих ссылок на выпуски. Это не устанавливает периодичность: расписание публикации в проверенном паспорте не раскрыто. В объём карточки не входят внутренние сервисы администрирования, индивидуальные декларации или выписки из реестров.

## Объекты данных и показатели

- Одна строка проверенного выпуска `20260201` — набор показателей формы № 1-НДС для значения `GA`; русское описание `GA` в структуре — «Субъект РФ». Это территориальный агрегат, а не запись налогоплательщика или декларация.
- `GA` имеет формат `text` и содержит названия субъектов. В схеме нет подтверждённого машиночитаемого кода субъекта РФ, а также нет ИНН, ОКВЭД или идентификатора отдельной декларации. Поэтому CSV нельзя трактовать как реестр налогоплательщиков и нельзя автоматически соединять с внешними данными по названию региона.
- `GB` и `G1`–`G207` имеют опубликованный формат `integer`, но смысл, единица и временной охват берутся **для каждого кода** из русской колонки `russian description` в парном `structure-*.csv`. Английская колонка вспомогательная.
- В выпуске `20260201` `GA` уникален среди 85 строк. Это естественный идентификатор территориальной записи только в этом выпуске, не универсальный код региона. Рекомендуемый технический ключ: `(release_version, GA)`; `release_version` храните из имени/даты проверенного выпуска, поскольку его нет среди колонок данных.

## Проверка согласованности выпуска `20260201`

- В `structure-20260201.csv` после строки заголовка 209 полей; в заголовке `data-20260201-structure-20260201.csv` тоже 209. Их коды и порядок совпадают полностью: `GA`, `GB`, затем `G1`…`G207`.
- Все 85 строк данных содержат по 209 значений. Среди 17 680 значений 208 числовых колонок нет пустых и нечисловых строк; все проходят проверку знакового целого `-?\\d+`.
- `GA` заполнен и уникален во всех 85 строках. Это подтверждает естественный идентификатор записи только для данного релиза, но не подтверждает его как долговременный или машиночитаемый код субъекта.
- Контрольные суммы и параметры диалекта приведены в разделе «Контракт CSV». При повторной загрузке проверяйте их для зафиксированной версии; для нового выпуска фиксируйте новые значения после чтения паспорта.

## Безопасное преобразование wide CSV в long-формат

Храните технический код, русское описание и формат рядом со значением. Код ниже рассчитан на проверенную пару `20260201`, использует штатный модуль `csv`, отвергает несогласованный заголовок и не преобразует отрицательные значения в ноль. В long-таблицу не включается `GA` как метрика: он остаётся географическим текстовым измерением.

```python
import csv
import re
from pathlib import Path

DATA = Path("data-20260201-structure-20260201.csv")
STRUCTURE = Path("structure-20260201.csv")
RELEASE_VERSION = "20260201"


def read_csv_utf8(path: Path) -> list[list[str]]:
    if path.read_bytes().startswith(b"\xef\xbb\xbf"):
        raise ValueError(f"{path}: BOM не соответствует проверенному выпуску")
    with path.open("r", encoding="utf-8", newline="") as source:
        return list(csv.reader(source, delimiter=",", quotechar='"', strict=True))


structure_rows = read_csv_utf8(STRUCTURE)
structure_header, *structure_rows = structure_rows
if structure_header != ["field name", "english description", "russian description", "format"]:
    raise ValueError("Неожиданный заголовок structure CSV")
if len({row[0] for row in structure_rows}) != len(structure_rows):
    raise ValueError("Повтор технического кода в структуре")

schema = {
    code: {"description_ru": description_ru, "format": field_format}
    for code, _description_en, description_ru, field_format in structure_rows
}
expected_codes = list(schema)  # порядок именно из structure того же релиза

data_rows = read_csv_utf8(DATA)
data_header, *data_rows = data_rows
if data_header != expected_codes:
    raise ValueError("Заголовок data CSV не совпадает со structure того же релиза")
if any(len(row) != len(data_header) for row in data_rows):
    raise ValueError("Строка data CSV имеет неверное число полей")

with Path("1nds-20260201-long.csv").open("w", encoding="utf-8", newline="") as target:
    writer = csv.DictWriter(
        target,
        fieldnames=[
            "release_version", "GA", "metric_code", "value_integer",
            "format", "unit_as_published", "description_ru",
        ],
    )
    writer.writeheader()
    for row in data_rows:
        record = dict(zip(data_header, row, strict=True))
        for code in data_header:
            if code == "GA":
                continue
            value = record[code]
            field = schema[code]
            if field["format"] != "integer" or not re.fullmatch(r"-?\d+", value):
                raise ValueError(f"{code}: нарушен формат integer: {value!r}")
            description = field["description_ru"]
            unit = (
                "тыс. рублей" if "тыс" in description.lower()
                else "единиц" if (
                    "(единиц)" in description.lower()
                    or "Количество деклараций" in description
                )
                else "не раскрыто в опубликованной структуре"
            )
            writer.writerow({
                "release_version": RELEASE_VERSION,
                "GA": record["GA"],
                "metric_code": code,
                "value_integer": int(value),  # отрицательные значения сохраняются
                "format": field["format"],
                "unit_as_published": unit,
                "description_ru": description,
            })
```

Этот long-формат даёт 17 680 строк для выпуска `20260201` (`85 × 208`). Не агрегируйте его механически: один и тот же экономический смысл может быть одновременно в итоговой и детализирующей графах; показатели по разным ставкам, вычеты и итоги не следует складывать между собой, пока методика ФНС не подтверждает аддитивность. Для межвыпускного анализа сохраняйте отдельную структуру каждого выпуска и сопоставляйте сначала русские описания, а не только технические коды.

## Изменение схемы между `20260101` и `20260201`

Для сравнения использованы предыдущие URL из того же паспорта: `data-20260101-structure-20260101.csv` и парный `structure-20260101.csv`. В `20260101` — 208 полей и 85 строк, в `20260201` — 209 полей и 85 строк.

| Проверка структуры | Результат |
| --- | --- |
| Добавления | Добавлен `G207` |
| Удаления | Нет |
| Перестановка общих кодов | Нет |
| Изменение формата общих полей | Нет: `GA` — `text`, остальные общие поля — `integer` |
| Изменение описаний | Изменились русские и английские описания 85 общих кодов |

Среди изменений описаний есть сдвиг названных отчётных периодов с IV кв. 2024 г. — I–III кв. 2025 г. на IV кв. 2025 г. — I–III кв. 2026 г., обновление дат «на 01.01.2027» и более существенное переназначение описаний в хвосте `G195`–`G206`; новый `G207` завершает последовательность. Поэтому перед каждой загрузкой обязательна проверка заголовка на парную структуру: она отловит добавление/удаление/перестановку. Но одной проверки заголовка недостаточно для сохранения смысла — одинаковый код может получить новое официальное описание; храните и сравнивайте `russian description` у структуры того же выпуска.

## Периодичность обновления

Периодичность в проверенном паспорте не раскрыта. Он показывает датированные текущий и предыдущие выпуски, но из самих дат нельзя выводить неизменный график. Перед каждым новым сбором откройте [паспорт набора](https://www.nalog.gov.ru/opendata/7707329152-1nds/) и получите из него пару текущих ссылок.

## Ограничения и особенности

- Условия использования — **открытая лицензия** ФНС на открытые данные, текст на [странице витрины](https://www.nalog.gov.ru/rn77/opendata/) (в т.ч. требования к указанию источника и неискажению метаданных обновления).
- Это агрегированная территориальная статистика, а не микроданные, декларации или реестр налогоплательщиков.
- Технические коды, единицы и даты берите из `structure-*.csv` того же выпуска; не выводите их из имени столбца, типа `integer` или даты файла.
- Нельзя безопасно делать соединение с внешним классификатором субъектов только по `GA`: это текстовое название, а машиночитаемый код в опубликованной структуре отсутствует.
- Не подменять эту выгрузку другой **агрегированной налоговой статистикой** ФНС — см. [Налоговая статистика ФНС](/data-sources/federal/nalog-statistics) и при необходимости [госрегистрация](/data-sources/federal/nalog-regstats).
