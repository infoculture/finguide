# Сервис «Прозрачный бизнес» ФНС (pb.nalog.ru)

| Признак | Значение |
| :--- | :--- |
| Уровень и охват | Федеральная витрина по юридическим лицам и ИП; отдельные инструменты для сравнения отраслевых показателей |
| Тип доступа | Публичный веб-интерфейс; внутренние JSON/XHR-запросы работают только как часть интерфейса и не являются публичным API |
| Основные форматы | HTML; JSON в ответах внутренних запросов браузера |
| Оператор | [ФНС России](/organizations/fns) |
| Режим доступа | Открытый просмотр; при интенсивных запросах интерфейс запрашивает CAPTCHA и сообщает о лимите |

Коротко: **«Прозрачный бизнес»** — официальный интерфейс ФНС для точечной проверки контрагента по **ИНН**, **ОГРН**/**ОГРНИП**, названию или ФИО. Он показывает сводную карточку организации либо ИП, а также даёт калькулятор налоговой нагрузки и узкоспециализированные отраслевые показатели. Это удобный слой ручной проверки и исследования одного субъекта, но **не** замена файловых наборов [открытых данных ФНС](/data-sources/federal/fns-opendata-index), [выписки ЕГРЮЛ/ЕГРИП](/information-systems/federal/egrul) или [ГИР БО](/data-sources/federal/bo-nalog-gov-ru).

## Описание

Витрина объединяет поиск по реестровым и публикационным признакам, карточки субъектов, сравнение выбранных организаций, калькулятор и отдельный раздел отраслевых показателей. Карточка создаётся для просмотра, а не как постоянная запись: адрес `company.html?token=…` содержит временный интерфейсный токен, а не ИНН и не публичный идентификатор записи.

В объём этой страницы **не** входят личные кабинеты налогоплательщика, АИС «Налог-3» и сторонние агрегаторы. Также не следует считать частью самой витрины сервисы, на которые ведёт её меню: они работают на `service.nalog.ru`, `egrul.nalog.ru`, `lkul.nalog.ru`, `lkip2.nalog.ru` и других доменах.

## Оператор

[Федеральная налоговая служба](/organizations/fns) — оператор и публикатор витрины: https://pb.nalog.ru/

- Карточка ИС: [Сайт ФНС России — статистика и данные](/information-systems/federal/fnssite) — общий публичный контур ФНС; «Прозрачный бизнес» является специализированной витриной на отдельном поддомене.

## Объекты данных и показатели

### Результат поиска и ключи связки

Внутренний ответ общего поиска группирует результаты по типу: `ul` (ЮЛ), `ip` (ИП), `upr` (руководители), `uchr` (учредители), `addr` (адреса), `rdl` (дисквалифицированные лица), `ogrfl`/`ogrul` (ограничения участия), `docul`/`docip` (поданные документы). У групп, содержащих строки, интерфейс использует контейнер вида `{"data": [...], "rowCount": number, "page": number, "pageSize": number, "hasMore": boolean}`.

| Поле результата ЮЛ / ИП | Тип в ответе интерфейса | Смысл, источник и ограничения |
| --- | --- | --- |
| `inn` | строка из цифр | ИНН — основной ключ склейки с карточкой, ГИР БО, ЕИС и наборами ФНС. Не переводите в число: ведущие нули и контрольная длина должны сохраняться. |
| `ogrn` | строка из цифр | Для ЮЛ — ОГРН, для ИП — ОГРНИП; второй ключ проверки идентичности. |
| `namec`, `namep` | строка, возможна пустота | Сокращённое и полное наименование ЮЛ либо ФИО ИП. Это поисковое представление, не надёжный ключ. |
| `dtogrn` | строка даты `ДД.ММ.ГГГГ`, возможна пустота | Дата присвоения ОГРН/ОГРНИП из регистрационных сведений. |
| `sulst_ex`, `sulst_name_ex` | код и строка, возможны пустоты | Состояние ЮЛ; `sulst_ex` — интерфейсный код, `sulst_name_ex` — отображаемое наименование. Не используйте код без словаря сайта. |
| `pr_sipst` | строковый флаг `"0"`/`"1"`, возможна пустота | Состояние ИП: интерфейс отображает «действующий» или «деятельность прекращена». |
| `regionname` | строка, возможна пустота | Регион в поисковом результате ЮЛ; у ИП в кратком результате может не показываться. |
| `okved2main`, `okved2mainname` | строка, возможна пустота | Основной ОКВЭД и его наименование; ключ отраслевой классификации. |
| `invalid`, `predo` | строковый/числовой флаг, возможна пустота | Признак недостоверности сведений и признак участия в ЭДО в представлении интерфейса. Это статусы, не количественные показатели. |
| `token` | временная строка | Только сессионный указатель для перехода к карточке. **Не** ключ склейки, не добавляйте в базу и не публикуйте как ссылку. |

Ключи предметной связки: **ИНН** — предпочтительный ключ между витриной, [ЕИС](/data-sources/federal/procurement), [XML ЕИС](/data-sources/federal/zakupki-xml), ГИР БО и наборами ФНС; **ОГРН/ОГРНИП** — контроль идентичности; **ОКВЭД** — отраслевой разрез. **КБК** в карточках pb.nalog.ru не является полем связи.

У части ЮЛ может быть «нет обязанности» либо нет сведений. |

### Карточка индивидуального предпринимателя

ИП использует иной шаблон (`type: 2`). Общие с ЮЛ смысловые поля — идентификатор, статус, налоговый орган, ОКВЭД, МСП, участие в ЭДО и специальные режимы; наименования полей отличаются.

| Блок и поля | Тип, единицы | Отличие от ЮЛ и пустоты |
| --- | --- | --- |
| **ФИО и идентификаторы:** `Фамилия`, `Имя`, `Отчество`, `ИННФЛ`, `ОГРНИП`, `ДатаОГРНИП`, `ДатаПостУч` | Строки; даты; ИНН/ОГРНИП — строки цифр | Вместо названия ЮЛ и ОГРН. `Отчество` и отдельные даты могут отсутствовать. |
| **Гражданство, статус, учёт:** `ВидГражд`, `НаимСтатусСокр`, `НаимСтатус`, `ДатаСтатус`, `КодНО`/`НаимНО`, `КодРО`/`НаимРО` | Строки, коды, дата | Статус ИП и регистрация по месту жительства; не переносите в модель поля ЮЛ (КПП, уставный капитал, учредители). |
| **Деятельность:** `КодОКВЭД`, `НаимОКВЭД`, `ТипОКВЭД`, `okved2dolyaex`, `okved2ex` | Строки/массивы | Те же два представления ОКВЭД, что в карточке ЮЛ; коды — ключ для отраслевых срезов. |
| **МСП и режим:** `rsmpcategory`, `rsmpdate`, `hastaxmode`, `ausn`, `eshn`, `psn`, `npd`, `usn`, `taxmode_yearcode`, `taxmode_periodcode` | Код, дата, флаги | Для ИП интерфейс перечисляет АУСН, ЕСХН, ПСН, НПД и УСН. Отсутствие значения не доказывает неприменение режима без указанного периода. |

В текущем шаблоне ИП **нет** блоков учредителей/уставного капитала, сводных доходов и расходов бухгалтерской отчётности, уплаченных налогов, задолженности/пеней, правонарушений и среднесписочной численности, которые показываются условно для ЮЛ. Не подставляйте в эти пропуски нули.

### Отраслевые нормативные показатели

Это не универсальный классификатор отраслей. На дату проверки страница фиксирует единственную отрасль «Клининг и техническая эксплуатация», с кодом фильтра `81.1`; описание охватывает основные заявленные ОКВЭД **81.1** и/или **81.2** и только лиц, давших специальное согласие на раскрытие. Интерфейс заявляет ежемесячный расчёт.

| Поле записи | Тип, единица | Смысл, источник и пустоты |
| --- | --- | --- |
| `name`, `inn` | строка; ИНН — строки цифр | Наименование/ФИО и ключ участника. Связка с карточкой — по ИНН. |
| `industry`, `region`, `okved` | строка | Отрасль, субъект РФ и ОКВЭД. По состоянию страницы отрасль фиксирована; регион/ОКВЭД могут быть пусты в ответе. |
| `rsmp`, `sn` | строка | Категория МСП с доходом не более 500 млн руб. и режим налогообложения в представлении сервиса. |
| `year`, `quarter` | число; квартал необязателен | Период записи: если `quarter` отсутствует, интерфейс показывает только год. |
| `dohod` | число, **рубли** | Сумма дохода за период. Ноль выводится как «отсутствуют доходы», поэтому нельзя делить на него и интерпретировать как пропуск. |
| `fsfn` | число, **%** | Фактическая совокупная фискальная нагрузка. При нулевом доходе интерфейс вместо значения показывает текст. |
| `nsfn` | число, **%** | Нормативная совокупная фискальная нагрузка, уменьшенная на 20%. |
| `osfn` | число, **процентные пункты** | Отклонение фактической нагрузки от нормативной, уменьшенной на 20%; может быть отрицательным, положительным или пустым. |

Интерфейс описывает методику: совокупная фискальная нагрузка — отношение исчисленных налогов и взносов к доходам за квартал/год; состав налогов и источники дохода различаются для общего режима и УСН. Нормативные показатели приведены в письме ФНС, на которое ссылается сама страница. Это ориентир риск-анализа, а не доказательство нарушения.

### Калькулятор налоговой нагрузки

Калькулятор предназначен для **организаций на общем режиме налогообложения**. Вводимые деньги — в тысячах рублей, а не в рублях. Периоды, доступные на странице при проверке: 2022–2024.

| Вход / выход | Поля и тип | Методика и ограничения |
| --- | --- | --- |
| Параметры ориентира | `PERIOD` (год), `OKVED` (код ОКВЭД), `REGION` (код/выбор региона), `SCL` (масштаб деятельности, необязателен) | Первые три обязательны. Справочники кодов подгружаются интерфейсом и не задокументированы как внешний API. |
| Фактический ввод | `CALC_DOH`, `CALC_FACT`, `CALC_PRIB`, `CALC_NDS`, `CALC_NDPI`, `CALC_NDFL`, `CALC_SV` — числа, **тыс. руб.** | Доход; общая сумма уплаченных налогов, сборов и взносов; в том числе налог на прибыль, НДС, НДПИ/акцизы, НДФЛ и страховые взносы. Не передавайте форматированные строки с пробелами без проверки интерфейса. |
| Среднеотраслевой ответ | `fisk`, `snr_ndpi`, `snr`, `nr_prib`, `nr_nds`, `ndfl`, `sv`, `rent` — числа, **%**; `srzp` — число, **руб./месяц** | Фискальная нагрузка, совокупная с/без НДПИ и акцизов, нагрузки по видам, рентабельность продаж и средняя зарплата. Пустое значение выводится пустым или `**`; не заменяйте его нулём. |
| Фактический ответ и отклонение | `FACT_FISK`, `FACT_SNR_NDPI`, `FACT_SNR`, `FACT_NR_PRIB`, `FACT_NR_NDS`, `FACT_NDFL`, `FACT_SV` — **%**; соответствующие `DIV_*` — **п.п.** | Сравнение ввода с ориентиром. Интерфейс указывает формулу фискальной нагрузки: сумма уплаченных налогов, сборов и страховых взносов без дивидендов / доходы за год × 100%. |

## Машиночитаемые форматы и каналы доступа

- **HTML — основной поддерживаемый канал.** Он подходит для интерактивного поиска одного субъекта и ручной проверки отображаемой даты/периода.
- **Внутренний JSON/XHR — только техническая деталь интерфейса.** Формы сайта на дату проверки отправляли запросы к `search-proc.json`, `company-proc.json`, `calculator-proc.json?c=search` и `indicator-proc.json`. Это адреса, извлечённые из HTML/JavaScript сайта, а не опубликованная спецификация: они могут измениться без предупреждения, требуют временные `id`/`token` и могут вернуть CAPTCHA либо `pbRateLimit`.
- **Внешние файлы — не на pb.nalog.ru.** `/od.html` не отдаёт CSV/XML/XLSX; за массовыми данными переходите в [каталог открытых данных ФНС](/data-sources/federal/fns-opendata-index) и к конкретным карточкам наборов.
- **Внешние сервисы — отдельные системы.** Ссылки в меню на `service.nalog.ru`, ЕГРЮЛ, личные кабинеты и НПД не делают их эндпоинтами pb.nalog.ru.

Ограничения автоматизации: CAPTCHA, заявленный интерфейсом лимит частоты, асинхронная подготовка результата, временные токены, отсутствие опубликованных схем полей/версии API, изменение HTML и клиентского JavaScript. Не запускайте параллельный или непрерывный опрос и не обходите защитные механизмы.

## Периодичность обновления

Единой периодичности нет. Сайт показывает период у конкретного блока: часть сведений из открытых данных отмечена как ежемесячная или годовая, отраслевые показатели интерфейс называет ежемесячно рассчитываемыми, а реестровые реквизиты обновляются по обработке регистрационных данных. Для повторяемого ряда берите периодичность из паспорта соответствующего набора, а не по дате открытия карточки.

## Ограничения и особенности

- Карточка — снимок представления ФНС, а не доказательство юридического статуса на дату сделки. Для официальной выписки используйте [ЕГРЮЛ/ЕГРИП](/information-systems/federal/egrul).
- «Сведения отсутствуют», «нет обязанности», пустое поле и числовой ноль — разные состояния. Сохраняйте их раздельно вместе с годом/периодом и датой извлечения.
- `token` и `id` из внутренних ответов краткоживущие; не используйте их в качестве ссылки, первичного ключа или идентификатора сущности.
- Карточка не покрывает все организации одинаково: часть налоговых, финансовых и регистрационных блоков зависит от типа субъекта, периода и доступности публикации.
- Отклонение от отраслевого ориентира и результат калькулятора — аналитический сигнал, но не подтверждение нарушения и не готовое решение для контроля.

## Примеры доступа

### 1. Получить карточку через веб-интерфейс — рекомендуемый путь

1. Откройте https://pb.nalog.ru/.
2. Выберите «Организации» либо «Индивидуальный предприниматель (ИП)».
3. Введите ИНН, ОГРН/ОГРНИП или допустимый текстовый запрос и откройте строку результата.
4. Зафиксируйте ИНН, ОГРН/ОГРНИП, дату/период каждого показателя и факт отсутствия/наличия блока; не сохраняйте `token` из URL карточки.

Это единственный канал, который сервис явно предоставляет пользователю для просмотра карточки.

### 2. Однократный внутренний запрос поиска — шаблон, проверьте на актуальной версии сайта

Ниже приведён **не публичный API**, а минимальный технический пример потока, который использовала страница `search.html` при проверке 2026-07-19. Он запрашивает **один** ИНН, ждёт асинхронный результат и прекращает работу при ошибке. CAPTCHA или `pbRateLimit` — нормальный результат защиты; не обходите их и вернитесь к браузеру.

```bash
pb_inn='7707329152' # пример: ИНН ФНС России; замените на один разрешённый запрос

pb_request=$(curl --compressed --silent --show-error --location \
  --referer 'https://pb.nalog.ru/search.html' \
  --data-urlencode 'mode=search-ul' \
  --data-urlencode "queryUl=$pb_inn" \
  --data-urlencode 'page=1' \
  --data-urlencode 'pageSize=10' \
  'https://pb.nalog.ru/search-proc.json')

pb_request_id=$(printf '%s' "$pb_request" | node -e '
  let text = ""; process.stdin.on("data", c => text += c).on("end", () => {
    const value = JSON.parse(text);
    if (!value.id || value.captchaRequired) process.exit(2);
    process.stdout.write(value.id);
  });
') || { printf '%s\n' 'Сайт запросил CAPTCHA или изменил ответ; продолжайте в браузере.' >&2; exit 1; }

for pb_attempt in $(seq 1 20); do
  pb_result=$(curl --compressed --silent --show-error --location \
    --referer 'https://pb.nalog.ru/search.html' \
    --data-urlencode "id=$pb_request_id" \
    --data-urlencode 'method=get-response' \
    'https://pb.nalog.ru/search-proc.json')
  [ "$pb_result" = 'null' ] || break
  sleep 2
done

printf '%s\n' "$pb_result"
```

Ожидаемая форма успешного ответа для ЮЛ — объект `ul` с `data` (массив строк) и `rowCount`; для ИП — `ip`. В ответе нет стабильной схемы и версии, поэтому перед обработкой валидируйте типы и наличие ключей.

### 3. Проверить и нормализовать ответ внутреннего поиска

Этот пример работает с JSON, уже сохранённым из предыдущего однократного запроса в `pb-search-result.json`; он не обращается к сайту и не предполагает несуществующих полей.

```js
// node normalize-pb-search.mjs < pb-search-result.json
let text = '';
for await (const chunk of process.stdin) text += chunk;

const payload = JSON.parse(text);
const group = payload.ul ?? payload.ip;
if (!group || !Array.isArray(group.data)) {
  throw new Error('Ожидался результат ul или ip с массивом data');
}

for (const row of group.data) {
  const inn = String(row.inn ?? '');
  const ogrnOrOgrnip = String(row.ogrn ?? '');
  if (!/^\d{10}(\d{2})?$/.test(inn) || !/^\d{13}(\d{2})?$/.test(ogrnOrOgrnip)) {
    throw new Error(`Неверный идентификатор в строке поиска: ${JSON.stringify(row)}`);
  }

  console.log(JSON.stringify({
    inn,
    ogrn_or_ogrnip: ogrnOrOgrnip,
    name: row.namec || row.namep || null,
    status: row.sulst_name_ex ?? (row.pr_sipst === '0' ? 'Действующий ИП' : null),
    okved: row.okved2main ?? null,
    source: 'pb.nalog.ru internal search response',
  }));
}
```

Поле `token` намеренно не переносится в результат: оно временное. Проверка длины ИНН/ОГРН/ОГРНИП позволяет отбросить ошибочный или изменившийся ответ прежде, чем делать склейку.

### 4. Склейка с внешними наборами по ИНН/ОГРН — псевдокод

В pb.nalog.ru нет экспортируемого файла карточек, поэтому массовую основу берут из внешнего набора, а витрину используют для точечной верификации. Ниже `<…>` обозначают конкретные названия колонок выбранной выгрузки, которые нужно привести к общему псевдониму `inn` при чтении её схемы.

```sql
WITH contracts AS (
  SELECT regexp_replace(<ИНН_поставщика_из_ЕИС>, '[^0-9]', '', 'g') AS inn, *
  FROM <таблица_контрактов_ЕИС>
),
financials AS (
  SELECT regexp_replace(<ИНН_из_ГИР_БО>, '[^0-9]', '', 'g') AS inn, *
  FROM <таблица_ГИР_БО>
),
registry AS (
  SELECT regexp_replace(<ИНН_из_ЕГРЮЛ_ЕГРИП>, '[^0-9]', '', 'g') AS inn, *
  FROM <реестровая_выгрузка>
)
SELECT c.*, f.*, r.*
FROM contracts AS c
LEFT JOIN financials AS f USING (inn)
LEFT JOIN registry AS r USING (inn);
```

После склейки проверяйте ОГРН/ОГРНИП и дату среза. Данные [ЕИС](/data-sources/federal/procurement) отвечают за контракт, [ГИР БО](/data-sources/federal/bo-nalog-gov-ru) — за бухгалтерские формы, а [ЕГРЮЛ/ЕГРИП](/information-systems/federal/egrul) — за реестровую идентичность; pb.nalog.ru не подменяет ни один из этих источников.

### 5. Скачивание открытых данных

Для `/od.html` рабочего `curl`/`wget` примера **нет**, потому что эта страница не публикует файл и не даёт URL загрузки — она ведёт во внешний каталог ФНС. Не подменяйте отсутствующий пример выдуманным URL: выберите нужный набор в [каталоге открытых данных ФНС](/data-sources/federal/fns-opendata-index), затем используйте проверенный URL и схему именно его паспорта.

## Частые вопросы

**В:** Можно ли считать `search-proc.json` или `company-proc.json` API ФНС?<br />
**О:** Нет. Это внутренние адреса, которые использовал веб-интерфейс на дату проверки. Для них нет опубликованной схемы, версии или обещания совместимости; CAPTCHA, лимит и временные токены прямо исключают использование как устойчивого массового API.

**В:** Если в карточке нет блока долга или доходов, это ноль?<br />
**О:** Нет. Это может означать отсутствие сведений, другой период, неприменимость блока или отсутствие обязанности. Сохраняйте исходный статус интерфейса и проверяйте файловый набор/ГИР БО по конкретной задаче.

**В:** Какой ключ использовать для связи с закупками?<br />
**О:** Нормализованный ИНН, с проверкой ОГРН/ОГРНИП и даты среза. Наименование и временный `token` для такой связи не подходят.
