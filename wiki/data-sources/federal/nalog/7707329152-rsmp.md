Коротко: набор **7707329152-rsmp** — машиночитаемый снимок реестра принадлежности к МСП. Для каждой записи он даёт категорию (микро/малое/среднее), идентификаторы ЮЛ или ИП, дату включения, код региона и виды деятельности. Это источник для обогащения контрагентов по ИНН, а не выписка ЕГРЮЛ/ЕГРИП и не реестр получателей мер поддержки.

## Описание

Паспорт идентифицирует набор как **«Единый реестр субъектов малого и среднего предпринимательства»**, идентификатор — `7707329152-rsmp`. Он содержит сведения о юридических лицах и индивидуальных предпринимателях, внесённых в реестр; состав сведений определён законом № 209-ФЗ. В примечании паспорта отдельно указано: сведения о среднесписочной численности с 10 января по 10 июня относятся к календарному году, предшествующему предыдущему, а в срезах с 10 июля по 10 декабря — к предыдущему календарному году.

Каждый ZIP — снимок состояния реестра: все проверенные XML-файлы текущего выпуска имеют в `Документ/@ДатаСост` значение `10.07.2026`. Изменения состава между датами не помечаются отдельными событиями «включён» или «исключён»; их восстанавливают сравнением архивных срезов по ИНН/ОГРН(ИП).

В объём карточки не входят интерактивный поиск на [rmsp.nalog.ru](/data-sources/federal/rmsp-nalog-ru), выписки ЕГРЮЛ/ЕГРИП и реестр **получателей** поддержки МСП — это отдельный набор [7707329152-rsmppp](/data-sources/federal/7707329152-rsmppp).

## Объекты данных и показатели

- **Одна запись** — элемент `Документ`: один субъект МСП на дату `@ДатаСост`; субъект — ЮЛ, ИП или глава КФХ.
- **Ключи субъекта** — ИНН и ОГРН для ЮЛ либо ИНН и ОГРНИП для ИП/главы КФХ. Они передаются строками, чтобы не потерять ведущие нули и не выйти за безопасную точность числовых типов.
- **Классификация** — категория МСП, признак вновь созданного субъекта, социального предприятия, среднесписочная численность для ЮЛ, регион и ОКВЭД.
- **Не публикуются:** КПП, ОКТМО, полный почтовый адрес, выручка/доход, данные об участниках и явный признак исключения из реестра. В XSD объявлен общий тип `КППТип`, но в структуре `Файл/Документ` нет элемента или атрибута `КПП`.

### Структура файла (XML-схема)

- **Кодировка:** XML-декларация фактических файлов — `UTF-8`; текущая XSD также имеет кодировку UTF-8.
- **Корневой элемент:** `Файл`; обязательные атрибуты `ИдФайл`, `ВерсФорм="4.06"`, `ТипИнф="РЕЕСТРМСП"`, `КолДок`; `ВерсПрог` необязателен.
- **Повторяющаяся запись:** `Документ` — один включённый в реестр субъект МСП. В текущем ZIP каждый проверенный XML-файл содержит `КолДок="900"` и 900 элементов `Документ`.
- **Вложенные группы:** один из `ОргВклМСП` или `ИПВклМСП`; обязательный `СведМН`; необязательные ОКВЭД, лицензии, продукция и программы партнёрства.
- **XSD:** [structure-12052026.xsd](https://file.nalog.ru/opendata/7707329152-rsmp/structure-12052026.xsd) — опубликована ФНС, не реконструирована.

Компактный скелет актуального формата:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<Файл ИдФайл="VO_RRMSPSV_0000_9965_20260710_<GUID>"
      ВерсФорм="4.06" ТипИнф="РЕЕСТРМСП" КолДок="900">
  <ИдОтпр>
    <ФИООтв Фамилия="-" Имя="-"/>
  </ИдОтпр>
  <Документ ИдДок="<GUID>" ДатаСост="10.07.2026" ДатаВклМСП="01.08.2016"
            ВидСубМСП="1" КатСубМСП="1" ПризНовМСП="2" СведСоцПред="2" ССЧР="0">
    <ОргВклМСП НаимОрг="..." НаимОргСокр="..." ИННЮЛ="4305070828" ОГРН="1054301505840"/>
    <!-- либо для ИП/главы КФХ: <ИПВклМСП ИННФЛ="..." ОГРНИП="..."><ФИОИП .../></ИПВклМСП> -->
    <СведМН КодРегион="43"><Регион Тип="обл." Наим="Кировская"/></СведМН>
    <СвОКВЭД><СвОКВЭДОсн КодОКВЭД="47.2" НаимОКВЭД="..." ВерсОКВЭД="2014"/></СвОКВЭД>
    <СвЛиценз НомЛиценз="..." ДатаЛиценз="...">...</СвЛиценз>
    <СвПрод КодПрод="69.1" НаимПрод="Услуги юридические" ПрОтнПрод="2"/>
  </Документ>
</Файл>
```

Проверенный ZIP содержит 7 347 XML-членов с именами вида `VO_RRMSPSV_0000_9965_20260710_<GUID>.xml`. Центральный каталог ZIP сообщает суммарный распакованный размер 22 282 684 594 байта (20,75 GiB). По первому, последнему и восьми распределённым XML-файлам `КолДок=900`; ориентир объёма — около **6,6 млн** записей (`7 347 × 900`). Точное число при производственном импорте считайте итерацией, а не только умножением или доверием к атрибуту.

## Периодичность обновления

Паспорт не выводит отдельное текстовое поле «периодичность». Наблюдаемая по списку архивов периодичность — **ежемесячный срез на 10-е число**: `data-10082016`, `data-10092016`, …, `data-10062026`, текущий `data-10072026`. Для текущего выпуска паспорт фиксирует дату последнего изменения `10.07.2026`, содержание «Обновление набора» и дату актуальности `10.08.2026`.

Это наблюдение по архиву, а не гарантированное расписание: перед запуском регулярной загрузки сверяйте паспорт и извлекайте актуальный URL программно.

## Ограничения и особенности

- Не загружайте ZIP целиком в память: один актуальный архив занимает ~2,11 ГБ, XML после распаковки — ~22,28 ГБ. Обрабатывайте XML-члены последовательно и используйте `iterparse`.
- Актуальный ZIP поддерживает HTTP Range (`Accept-Ranges: bytes`); это позволило проверить центральный каталог и отдельные XML-члены без скачивания 2,11 ГБ. Для полного импорта всё равно потребуется место на диске под ZIP либо собственная потоковая загрузка с сохранением архива.
- В наборе нет КПП, ОКТМО и явной даты исключения. Отсутствие субъекта в новом срезе можно трактовать только после сравнения снимков, с учётом возможных реорганизаций и качества ключей.
- Не путайте `КодРегион` со значением ОКТМО: это двухзначный код субъекта РФ, а не муниципальное образование.
- ОКВЭД заявительного и отчётного типов находятся в разных ветках (`СвОКВЭД` и `СвОКВЭДотч`); не объединяйте их без поля-источника.

## Примеры доступа

### 1. Найти актуальный ZIP в паспорте и скачать на диск

Проверено 2026-07-19: запрос к паспорту выбрал `data-10072026-structure-12052026.zip`; его HTTP-заголовок подтвердил размер 2 111 054 656 байт. Сам полный ZIP в рамках проверки не скачивался: ниже код делает это потоково на диск и требует около 2,1 ГБ свободного места.

```python
from pathlib import Path
from urllib.parse import urljoin
from urllib.request import Request, urlopen

from lxml import html

PASSPORT = "https://www.nalog.gov.ru/opendata/7707329152-rsmp"
HEADERS = {"User-Agent": "Mozilla/5.0 (compatible; rsmp-downloader/1.0)"}

with urlopen(Request(PASSPORT, headers=HEADERS), timeout=60) as response:
    page = html.fromstring(response.read())

# Первая ссылка data-*.zip в паспорте — поле «Гиперссылка (URL) на набор».
zip_hrefs = page.xpath(
    "//a[contains(@href, '/opendata/7707329152-rsmp/data-') "
    "and contains(@href, '.zip')]/@href"
)
if not zip_hrefs:
    raise RuntimeError("ФНС не отдала актуальную ссылку data-*.zip в паспорте")

zip_url = urljoin(PASSPORT, zip_hrefs[0])
target = Path(zip_url.rsplit("/", 1)[-1])

with urlopen(Request(zip_url, headers=HEADERS), timeout=600) as response, target.open("wb") as output:
    while chunk := response.read(1024 * 1024):  # 1 MiB; весь ZIP не держится в RAM
        output.write(chunk)

print(f"saved {target} ({target.stat().st_size:,} bytes) from {zip_url}")
```

### 2. Потоково разобрать все XML внутри ZIP

Проверено 2026-07-19 на XML-члене `VO_RRMSPSV_0000_9965_20260710_ff5129a4-bef7-4837-a344-eea98946edb8.xml` актуального ZIP: XML валиден против `structure-12052026.xsd`; `lxml.etree.iterparse` извлёк ИНН, ОГРН(ИП), дату среза, категорию, регион и основной ОКВЭД. Функция ниже не строит DOM всего файла и освобождает обработанные элементы.

```python
import json
from pathlib import Path
from zipfile import ZipFile

from lxml import etree


def get_attr(element, name):
    return element.get(name) if element is not None else None


def flatten_document(document):
    company = document.find("ОргВклМСП")
    entrepreneur = document.find("ИПВклМСП")  # также используется для главы КФХ
    address = document.find("СведМН")
    primary_okved = document.find("./СвОКВЭД/СвОКВЭДОсн")
    fio = entrepreneur.find("ФИОИП") if entrepreneur is not None else None

    return {
        "record_id": document.get("ИдДок"),
        "as_of": document.get("ДатаСост"),
        "included_at": document.get("ДатаВклМСП"),
        "subject_kind": document.get("ВидСубМСП"),
        "msp_category": document.get("КатСубМСП"),
        "is_new": document.get("ПризНовМСП"),
        "is_social_enterprise": document.get("СведСоцПред"),
        "employee_count": document.get("ССЧР"),
        "inn": get_attr(company, "ИННЮЛ") or get_attr(entrepreneur, "ИННФЛ"),
        "ogrn_or_ogrnip": get_attr(company, "ОГРН") or get_attr(entrepreneur, "ОГРНИП"),
        "name": get_attr(company, "НаимОрг") or " ".join(
            part for part in (get_attr(fio, "Фамилия"), get_attr(fio, "Имя"), get_attr(fio, "Отчество")) if part
        ),
        "region_code": get_attr(address, "КодРегион"),
        "okved_primary": get_attr(primary_okved, "КодОКВЭД"),
        "okved_version": get_attr(primary_okved, "ВерсОКВЭД"),
    }


def stream_records(zip_path):
    with ZipFile(zip_path) as archive:
        for member in archive.namelist():
            if not member.endswith(".xml"):
                continue
            with archive.open(member) as source:
                for _, document in etree.iterparse(
                    source, events=("end",), tag="Документ", huge_tree=True
                ):
                    row = flatten_document(document)
                    document.clear()
                    while document.getprevious() is not None:
                        del document.getparent()[0]
                    yield row


for number, row in enumerate(stream_records(Path("data-10072026-structure-12052026.zip")), 1):
    print(json.dumps(row, ensure_ascii=False))
    if number == 10:
        break
```

### 3. Соединить нормализованный срез МСП с внешней таблицей по ИНН

Проверено 2026-07-19: операция `pandas.merge` выполнена на двух строках, извлечённых из проверенного XML-среза, и тестовой внешней таблице. В примере `contracts.csv` — локально подготовленный контракт внешнего набора с колонкой `supplier_inn`; конкретное имя поля в сырой выгрузке ЕИС или другом источнике **ТРЕБУЕТ ПРОВЕРКИ** до нормализации. ИНН нельзя приводить к числу.

```python
import pandas as pd

# rsmp_current.parquet формируется потоковым разбором выше; это один срез @ДатаСост.
msp = pd.read_parquet(
    "rsmp_current.parquet",
    columns=["inn", "as_of", "msp_category", "region_code", "okved_primary"],
)
contracts = pd.read_csv("contracts.csv", dtype={"supplier_inn": "string"})

required = {"supplier_inn"}
missing = required - set(contracts.columns)
if missing:
    raise ValueError(f"Нормализуйте внешний набор: отсутствуют колонки {sorted(missing)}")

msp["inn"] = msp["inn"].astype("string").str.strip()
contracts["supplier_inn"] = contracts["supplier_inn"].astype("string").str.strip()

# Уникальность — ожидаемое свойство одного среза; fail-fast защищает от ошибочной склейки.
if msp["inn"].duplicated().any():
    raise ValueError("В срезе есть повторяющиеся ИНН: разберите дубликаты до join")

joined = contracts.merge(
    msp,
    how="left",
    left_on="supplier_inn",
    right_on="inn",
    validate="m:1",
    indicator=True,
)
joined.to_parquet("contracts_enriched_with_msp.parquet", index=False)
print(joined["_merge"].value_counts(dropna=False))
```

## Частые вопросы

**В:** Можно ли узнать из XML, когда субъект исключён из реестра?

**О:** Нет. В текущей XSD нет даты или флага исключения. Наличие `Документ` означает присутствие в срезе на `@ДатаСост`; исчезновение нужно определять сравнением двух срезов по ИНН и ОГРН/ОГРНИП.

**В:** Есть ли в наборе КПП, ОКТМО и полный адрес?

**О:** Нет. У ЮЛ есть ИНН и ОГРН, у ИП — ИНН и ОГРНИП. Территория ограничена `СведМН/@КодРегион` и необязательными именованными частями адреса; КПП и ОКТМО не передаются.

**В:** Какой ключ выбрать для склейки с закупками?

**О:** Основной ключ — строковый ИНН (`@ИННЮЛ` для ЮЛ, `@ИННФЛ` для ИП/главы КФХ). ОГРН/ОГРНИП полезны как дополнительная проверка, но применимы только к соответствующему виду субъекта. Перед соединением сохраняйте длину ИНН и проверяйте смысл внешнего поля: поставщик, заказчик и получатель — разные роли.
