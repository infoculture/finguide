# ГИС «Торги»: публикации и машиночитаемый доступ

| Признак | Значение |
| :--- | :--- |
| Уровень и охват | Федеральная единая площадка **имущественных и смежных торгов** (государственное и муниципальное имущество, земли из фондов, процедуры банкротства и др.) |
| Тип доступа | Смешанный: **веб-поиск и карточки** для всех; **API и выгрузки для разработчиков** — по документации на портале (версии и URL обновляются) |
| Основные форматы | HTML в интерфейсе; машинные форматы — по **актуальной** спецификации оператора |
| Оператор | [Федеральное казначейство](/organizations/federal-treasury) (инфраструктура площадки; документы размещают **организаторы торгов**) |
| Режим доступа | Открытый просмотр; участие в процедурах может требовать регистрации и ЭП |

Коротко: **официальный след торгов имуществом** — извещения, предмет лота, цены, организатор, протоколы и итоги; для анализа **нестандартных доходов бюджета** и сделок с активами государства. **Не** закупки по **44-ФЗ** / **223-ФЗ** (это **[ЕИС](/information-systems/federal/zakupki)**).

## Описание

Карточка фиксирует **источник данных** в смысле публичных публикаций **ГИС «Торги»**: что можно извлечь для исследований и интеграций, и чем этот контур отличается от **[XML ЕИС](/data-sources/federal/zakupki-xml)** и **[бюджетных наборов](/data-sources/federal/budget-gov-ru-datasets)**.

Сюда **не** входят: внутренние кабинеты участников; **казначейское исполнение** по КБК; **контракты закупок** на `zakupki.gov.ru`.

## Оператор

**Федеральное казначейство** — канонический URL площадки: https://torgi.gov.ru/

- Карточка ИС: **[ГИС «Торги»](/information-systems/federal/torgi-gov-ru)** — границы системы, отличие от ЕИС закупок и типовые ограничения склейки с бюджетом.

## URL и точки доступа

| Тип доступа | URL | Формат |
| --- | --- | --- |
| Веб-портал (поиск и карточки) | https://torgi.gov.ru/ | HTML |
| Раздел для разработчиков / API | уточняйте на **torgi.gov.ru** (поддомены и регламент публикуются оператором) | JSON, XML и др. по документации |

Точные **endpoint’ы**, лимиты и схемы полей **не дублируются** в wiki — переносите из официальной документации при каждой интеграции.

## Объекты данных и показатели

- **Объекты** — извещение о торгах, **лот** (описание, начальная цена, шаг), сроки, **организатор**, статус процедуры, протоколы, **победитель** (по мере публикации); для банкротств — привязка к делу в объёме публичной карточки.
- **Ключи для склейки** — **ИНН**, **ОГРН** сторон (сверка с **[ЕГРЮЛ](/information-systems/federal/egrul)**); прямого стабильного ключа на строку **казначейской отчётности** обычно **нет**.
- **Связь с доходами бюджета** — косвенная: по времени, организации и сумме сделки на фоне **[отчётов Казначейства](/data-sources/federal/roskazna-reports)**.

## Периодичность обновления

Публикации идут **по мере размещения организатором**; массовые выгрузки через API зависят от регламента оператора. Актуальность интерфейса — в разделе **«Ограничения и актуальность»** **[карточки ИС](/information-systems/federal/torgi-gov-ru)**.

## Ограничения и особенности

- **Склейка с бюджетом** — в открытых бюджетных наборах редко есть идентификатор лота; не смешивайте строки **torgi.gov.ru** и **ЕИС** без признака источника.
- **Изменения API** — при обновлении портала перепроверяйте документацию и тестовые запросы.
- **Полнота** — отсутствие записи на площадке не исключает сделку в другом реестре.

## Не путать с

- **[ЕИС в сфере закупок](/information-systems/federal/zakupki)** и **[zakupki XML](/data-sources/federal/zakupki-xml)** — закупки товаров, работ, услуг для нужд заказчика.
- **[Отчёты Казначейства](/data-sources/federal/roskazna-reports)** — кассовое исполнение; не содержат карточек торгов как в ГИС «Торги».

## Частые вопросы

**В:** Где взять актуальную документацию API?

**О:** В разделе для разработчиков на **https://torgi.gov.ru/** — URL и версии меняются; не копируйте endpoint’ы из старых инструкций без проверки.

**В:** Достаточно ли torgi.gov.ru, чтобы собрать доходы консолидированного бюджета региона?

**О:** **Нет** как единственный источник: нужны **[«Электронный бюджет»](/data-sources/federal/budget-gov-ru-datasets)**, **[отчёты Казначейства](/data-sources/federal/roskazna-reports)** и **[региональные обзоры](/data-sources/federal/regionalnye-byudzhety)**; torgi.gov.ru объясняет отдельные **процедуры отчуждения**.
