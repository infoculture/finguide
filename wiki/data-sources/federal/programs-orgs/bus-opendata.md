# Открытые данные bus.gov.ru

| Признак | Значение |
| :--- | :--- |
| Уровень и охват | Федеральный портал **государственных и муниципальных учреждений**; выгрузки по учреждениям всех уровней, размещающим сведения на bus.gov.ru |
| Тип доступа | Витрина паспортов наборов и **файловые выгрузки** (CSV, XML, JSON — по паспорту набора); в клиентской части портала используются запросы к `public-rest/api` (каталог паспортов, списки наборов, прямые ссылки на файлы) |
| Основные форматы | CSV, JSON, XML (конкретный набор форматов и кодировка — в паспорте набора) |
| Оператор | Федеральное казначейство |
| Режим доступа | Открытый (без регистрации для скачивания опубликованных наборов) |

Коротко: **машиночитаемые** срезы по **планам ФХД**, **отчётам об их исполнении**, **государственным (муниципальным) заданиям**, **бухгалтерской отчётности** и смежным сведениям об учреждениях — удобный вход для массовой выгрузки и стыковки с **ИНН/ОГРН**, субсидиями и закупками.

## Описание

Карточка описывает раздел **открытых данных** портала bus.gov.ru: реестр **паспортов наборов**, метаданные, ссылки на файлы и форматы выгрузок. Это **публичный слой** той же предметной области, что и веб-карточки учреждений на портале; **не** входят личные кабинеты, внутренние казначейские регистры и консолидированная бюджетная отчётность распорядителей уровня ГИИС «Электронный бюджет».

Сюда **не** следует относить ожидание полного совпадения с **[казначейским исполнением](/glossary/treasury-execution)** по КБК или с **[реестром контрактов](/information-systems/federal/zakupki)**: bus даёт рамку **учреждения** (план, задание, отчётность), а факт бюджетных выплат и закупок восстанавливают из профильных источников.

## Оператор

**Федеральное казначейство** — канонический вход к витрине: https://bus.gov.ru/opendata

- Карточка ИС: **[bus.gov.ru — портал ГМУ](/information-systems/federal/bus-gov)** — та же система; раздел открытых данных — машиночитаемый выход к данным, описанным в карточке ИС.

## URL и точки доступа

| Тип доступа | URL | Формат |
| --- | --- | --- |
| Витрина открытых данных (каталог паспортов и файлов) | https://bus.gov.ru/opendata | HTML + ссылки на CSV/XML/JSON |
| Основной портал учреждений (контекст и веб-отчётность) | https://bus.gov.ru/ | HTML |
| Реестр госзаданий (веб, для сверки с выгрузками) | https://bus.gov.ru/pub/register-public-tasks | HTML |

В одностраничном приложении портала для списка паспортов и наборов вызываются, в частности, пути вида `/public-rest/api/passports`, `/public-rest/api/dataSets` и прямая выдача файла `/public-rest/api/opendata/<passCode>/<fileName>` (имена параметров берутся из карточки паспорта на витрине). Перед автоматизацией проверяйте **актуальную** сеть запросов в браузере: состав API и ответы сервера менялись вместе с обновлениями портала.

## Объекты данных и показатели

Типовая **строка** или **файл** набора относится к **учреждению** и документу отчётности или реестру; точный перечень полей — только из **паспорта** конкретного датасета.

- **Учреждение** — реквизиты, идентификаторы в контексте портала, **ИНН**, **ОГРН** (для стыковки с **[субсидиями](/data-sources/federal/subsidy)**, **[ЕГРЮЛ](/information-systems/federal/egrul)**, **[ЕИС](/information-systems/federal/zakupki)**).
- **План ФХД и отчёт о его исполнении** — плановые и отчётные суммы и разрезы, в т.ч. с **[КОСГУ](/glossary/kosgu)**; методика и формы — см. **[план ФХД](/reporting/pfhd)**.
- **Государственное (муниципальное) задание** и отчётность по исполнению — объёмы услуг, показатели, финансовое обеспечение; см. **[отчётность по госзаданию](/reporting/state-assignment-reporting)**.
- **Бухгалтерская (финансовая) отчётность** — формы учреждений; обзор — **[бухотчётность учреждений](/reporting/institution-financial-statements)**.
- **Имущество и прочие сведения** — по отдельным паспортам; полнота и периодичность различаются.

Известные **пробелы**: не каждый фрагмент HTML-карточки учреждения дублируется в opendata; версии схем и кодировки **различаются по наборам и годам**.

## Периодичность обновления

У **каждого паспорта набора** на витрине указываются дата актуализации и регламент обновления (смотрите блок метаданных на странице набора). Общая о том, что состав датасетов **эволюционирует** и не все веб-сведения имеют полный машиночитаемый аналог, — в разделе **«Ограничения и актуальность»** карточки **[bus.gov.ru](/information-systems/federal/bus-gov)**.

## Ограничения и особенности

- Условия использования и лицензия — в **паспорте набора** на витрине (типично режим открытых данных с указанием источника).
- **Объёмы файлов** велики; для стабильного пайплайна фиксируйте идентификатор паспорта, **финансовый год** и версию выгрузки из метаданных.
- Прямые URL к `public-rest` при скриптовании могут требовать **тех же заголовков и контекста**, что и браузерный клиент портала; при «серверной ошибке» без сессии ориентируйтесь на скачивание через интерфейс витрины.
- Данные учреждения **не заменяют** **[наборы budget.gov.ru](/data-sources/federal/budget-gov-ru-datasets)** по консолидированному бюджету: сверка — по **ИНН**, периоду и методике показателя.

## Не путать с

- **[Наборы данных ГИИС «Электронный бюджет»](/data-sources/federal/budget-gov-ru-datasets)** — уровень **бюджета и распорядителей**, а не отдельного учреждения; другие показатели и методики агрегирования.

## Частые вопросы

**В:** Почему в CSV по учреждению не сходится сумма с линией исполнения бюджета по КБК?

**О:** Выгрузка bus отражает **учёт и отчётность учреждения** (часто с **КОСГУ**), а не полную картину **казначейских выплат** и **лицевых счетов**. Для бюджетного факта используйте **[отчёты Казначейства](/data-sources/federal/roskazna-reports)** и **[наборы Электронного бюджета](/data-sources/federal/budget-gov-ru-datasets)**, согласуя период и получателя.

**В:** Где взять перечень всех endpoint’ов API?

**О:** Портал **не публикует** в карточке источника отдельный стабильный OpenAPI для всех выгрузок: машиночитаемый вход — **паспорта и файлы** на https://bus.gov.ru/opendata плюс наблюдение за запросами SPA. Детали инфраструктуры — в **[карточке ИС bus.gov.ru](/information-systems/federal/bus-gov)**.
