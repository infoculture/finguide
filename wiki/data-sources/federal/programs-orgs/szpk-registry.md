Коротко: официальный вход к **реестру соглашений о защите и поощрении капиталовложений** (СпИК) на **budget.gov.ru**: удобный **веб-поиск** по разделу «Государственные инвестиции» и **машиночитаемый** слой — набор **`7710168360-SZPK`** в каталоге открытых данных того же портала. Для **субсидий, бюджетных инвестиций и межбюджетных трансфертов** из федерального бюджета используйте **[отдельный реестр соглашений](/data-sources/federal/subsidy)** (`7710568760-REGISTAGREEMENT`), а не этот набор.

## Описание

Карточка описывает **публичный реестр** соглашений в рамках механизма **защиты и поощрения капиталовложений**: сведения о заключённых соглашениях доступны через тематический URL портала и дублируются (для автоматизации) в **каталоге открытых данных** `budget.gov.ru` под постоянным идентификатором **`7710168360-SZPK`**.

Сюда **не** входят: общий **[реестр соглашений о предоставлении субсидий, бюджетных инвестиций и межбюджетных трансфертов](/data-sources/federal/subsidy)**; **контракты закупок** (**[ЕИС](/information-systems/federal/zakupki)**); кассовый **факт** без отдельной сверки с **[отчётностью об исполнении бюджетов](/data-sources/federal/otchety-ob-ispolnenii-byudzhetov)**.

## Оператор

**Федеральное казначейство** указано как **создатель** набора в `meta.json` каталога открытых данных; пользовательский раздел размещён на **https://www.budget.gov.ru/**.

- Карточка ИС: **[ГИИС «Электронный бюджет»](/information-systems/federal/giis-eb)** — единая экосистема портала и каталога наборов.

## Объекты данных и показатели

- **Запись реестра / строка выгрузки** — реквизиты соглашения о защите и поощрении капиталовложений, стороны, сроки и суммы по полям паспорта конкретной версии файла (точный перечень колонок — в актуальном `meta.json` и в структуре CSV).
- **Связка с инвестиционным контуром бюджета** — через коды и ссылки в карточке; сквозная склейка с **[ФАИП](/data-sources/federal/fcp)** и **[бюджетными инвестициями](/glossary/budget-investments)** возможна только после сверки полей периода и методики.
- **Связка с организациями** — ИНН/ОГРН и наименования должны согласовываться со **[сводным реестром участников бюджетного процесса](/data-sources/federal/orgsinfo)** при наличии общих ключей.

**Пробелы:** паспорт набора в выгрузке на дату проверки может содержать **пустые** блоки `data`/`structure` в `meta.json` — перед промышленной загрузкой проверяйте наличие файлов версий и полей в API или в CSV.

## Периодичность обновления

Дата изменения паспорта набора приводится в **`meta.json`** (поле `modified`, на дату проверки карточки: **23.10.2025**). Периодичность выкладки файлов задаётся регламентом оператора; общие предупреждения о смене полей и URL — в разделе **«Ограничения и актуальность»** карточки **[ГИИС «Электронный бюджет»](/information-systems/federal/giis-eb)**.

## Ограничения и особенности

- Идентификатор **`7710168360-`** в каталоге отличается от префикса **`7710568760-`** у ряда бюджетных наборов; при склейке каталогов **не смешивайте** записи разных создателей без проверки паспорта.
- «Красивые» пути на портале **меняются** между релизами; для устойчивой автоматизации опирайтесь на **`list.json`**, **`meta.json`** и параметр `code=` на витрине открытых данных.
- Реестр **СпИК** юридически и по охвату **не тождественен** общему реестру соглашений о субсидиях и трансфертах.

## Примеры доступа

Проверить наличие набора в каталоге и вывести заголовок:

```bash
curl -sS "https://budget.gov.ru/epbs/opendata/list.json" -o opendata-catalog.json
```

```python
import json

with open("opendata-catalog.json", encoding="utf-8") as f:
    catalog = json.load(f)
needle = "7710168360-SZPK"
for item in catalog.get("meta", []):
    if item.get("identifier") == needle:
        print(item["identifier"], "-", item.get("title"))
        break
else:
    print("dataset not found — refresh catalog")
```

Метаданные набора:

```bash
curl -sS "https://budget.gov.ru/epbs/opendata/7710168360-SZPK/meta.json" | python3 -m json.tool | head -40
```

Практические шаги по REST API портала — в **[руководстве по API budget.gov.ru](/howto/access/budget-gov-api)**.

## Частые вопросы

**В:** Почему в каталоге два разных «реестра соглашений»?

**О:** **СпИК** (`7710168360-SZPK`) — отдельный **институт** и отдельный набор. Общий реестр субсидий и трансфертов — **`7710568760-REGISTAGREEMENT`**, см. **[отдельную карточку](/data-sources/federal/subsidy)**.

**В:** Где актуальная схема полей?

**О:** В **`meta.json`** набора и в файлах конкретной **версии** в каталоге; при смене версии колонки CSV **могут измениться**.
