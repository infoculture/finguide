# Сведения о государственных и муниципальных организациях (реестр участников бюджетного процесса)

| Признак | Значение |
| :--- | :--- |
| Уровень и охват | **Федеральный портал** с реестром, охватывающим участников бюджетного процесса **всех уровней** бюджетной системы (в т.ч. органы власти, учреждения и иные юрлица в ролях, предусмотренных учётом в ГИИС) — конкретный состав записей проверяйте по карточке в интерфейсе и по метаданным выгрузки |
| Тип доступа | Смешанный: поиск и просмотр в веб-интерфейсе; машиночитаемый слой — через **каталог открытых данных** портала и связанные наборы (наличие файлов по записи в каталоге нужно проверять по `meta.json`) |
| Основные форматы | HTML (реестр на сайте); JSON (каталог и метаданные наборов); CSV/прочие — **если** паспорт набора в каталоге указывает файловую выгрузку |
| Оператор | Минфин России / Федеральное казначейство (публичный контур **[ГИИС «Электронный бюджет»](/information-systems/federal/giis-eb)**) |
| Режим доступа | Открытый для опубликованных разделов и наборов |

Коротко: **официальный вход** к **сводному реестру участников и неучастников бюджетного процесса** — справочнику организаций в бюджетном контуре, к которому в данных привязываются, в частности, **распределители и получатели** по **[реестру соглашений](/data-sources/federal/subsidy)**. Для планов и отчётности **отдельного учреждения** используйте **[портал ГМУ](/data-sources/federal/bus-opendata)**, а не этот реестр как полную замену.

## Описание

В публичной части единого портала бюджетной системы сведения о **государственных и муниципальных организациях** в смысле **бюджетного процесса** сосредоточены прежде всего в **реестре участников и неучастников бюджетного процесса**: там задаются роли и реквизиты, используемые при планировании и исполнении бюджета, ведении соглашений и смежных реестров ГИИС.

Сюда **не** входят: полнота **регистрационных** сведений как в **[ЕГРЮЛ](/information-systems/federal/egrul)**; **план ФХД, госзадание и бухгалтерская отчётность учреждения** на **[bus.gov.ru](/information-systems/federal/bus-gov)**; **контракты закупок** в **[ЕИС](/information-systems/federal/zakupki)** (хотя идентификаторы вроде ИКО/ИКУ могут пересекаться с закупочным контуром в отдельных наборах открытых данных).

## Оператор

**Минфин России** и **Федеральное казначейство** — операторы публичного контура **ГИИС «Электронный бюджет»**; канонический вход на портал: https://budget.gov.ru/

- Карточка ИС: **[ГИИС «Электронный бюджет»](/information-systems/federal/giis-eb)** — реестр участников и НСИ как часть единого бюджетного контура; границы относительно **bus.gov.ru** и **ЕИС** раскрыты в карточке ИС.

## URL и точки доступа

Часть URL на портале использует **читаемые кириллические пути**; при сохранении ссылок в скриптах при необходимости применяйте URL-кодирование.

| --- | --- | --- |
| Реестр участников и неучастников бюджетного процесса (публичный интерфейс) | https://budget.gov.ru/бюджет/расходы/реестр-участников-и-неучастников-бюджетного-процесса | HTML |
| Раздел «Участники бюджетного процесса» (контекст и смежные материалы) | https://budget.gov.ru/бюджетная-система/участники-бюджетного-процесса | HTML |
| Каталог открытых данных портала (поиск набора по реестру) | https://budget.gov.ru/epbs/opendata/list.json | JSON |

Связанные **открытые наборы** с идентификаторами заказчиков (полезно при стыковке с закупками): в каталоге `list.json` встречаются записи с идентификаторами вида `7710568760-IKO` и `7710568760-IKY` — состав полей и файлы только по паспорту набора.

## Объекты данных и показатели

- **Запись участника / неучастника бюджетного процесса** в реестре — набор реквизитов организации и статуса в бюджетном контуре (точный перечень полей зависит от интерфейса и выгрузки на дату снимка).
- **Связка с субсидиями и соглашениями** — стороны соглашения в **[реестре на портале](/data-sources/federal/subsidy)** должны быть согласуемы с реестром участников по правилам ГИИС; для машинной обработки опирайтесь на **[ИНН](/glossary/inn)**, коды участника и **[КБК](/glossary/kbk)** в файлах набора.
- **Разделение уровней бюджета** — в одной логике реестра могут соседствовать федеральные, региональные и муниципальные участники; уточняйте уровень по полям записи, а не по предположению «только федеральный».

## Периодичность обновления

Обновление **реестра и связанных открытых наборов** задаётся регламентами ГИИС и указывается в **метаданных** соответствующего набора в каталоге; общие предупреждения о смене полей и URL — в разделе **«Ограничения и актуальность»** карточки **[ГИИС «Электронный бюджет»](/information-systems/federal/giis-eb)**.

## Ограничения и особенности

- **Каталог `list.json` и `meta.json`** для набора `7710568760-RUBPNUBP` может не содержать готовых ссылок на файлы выгрузки на дату проверки; основной гарантированный канал — **веб-реестр** и выборка из других опубликованных наборов по задаче (см. **[бюджетные организации: обзор](/data-sources/federal/budorgs)**).
- Структура меню и «красивые» URL портала **менялась** в ходе обновлений Liferay; при 404 проверяйте вход с главной страницы `budget.gov.ru` в разделы **«Бюджет»** и **«Бюджетная система»**.
- Сведения реестра **не заменяют** юридически значимую выписку из **[ЕГРЮЛ](/information-systems/federal/egrul)** и не отражают полноту **портала учреждений** **[bus.gov.ru](/information-systems/federal/bus-gov)**.

## Примеры доступа

Скачать метаданные набора реестра в каталоге открытых данных (ответ `200` без авторизации):

```bash
curl -sS "https://budget.gov.ru/epbs/opendata/7710568760-RUBPNUBP/meta.json" -o rubpnubp-meta.json
```

Найти запись набора в общем каталоге по идентификатору:

```bash
curl -sS "https://budget.gov.ru/epbs/opendata/list.json" | python3 -c "import json,sys; c=json.load(sys.stdin); print([m for m in c.get('meta',[]) if m.get('identifier')=='7710568760-RUBPNUBP'])"
```

## Не путать с

- **[Портал ГМУ (bus.gov.ru)](/data-sources/federal/bus-opendata)** — сведения о **деятельности учреждения** и его отчётности, а не сводный реестр участников бюджетного процесса в смысле ГИИС.
- **[ЕИС в сфере закупок](/information-systems/federal/zakupki)** — контракты и процедуры; заказчик сопоставляется с бюджетным контуром по **[ИНН](/glossary/inn)** и справочникам, но это **другой** реестр.

## Частые вопросы

**В:** Почему в `meta.json` набора `7710568760-RUBPNUBP` пустой блок `data`?

**О:** Каталог фиксирует **метаданные** набора; фактические файлы или API для массовой выгрузки реестра целиком могут отсутствовать или публиковаться иначе на дату снимка. Для аналитики комбинируйте **веб-поиск** по реестру и **другие** открытые наборы участников в `list.json` (см. **[budorgs](/data-sources/federal/budorgs)**).

**В:** Достаточно ли реестра, чтобы идентифицировать получателя субсидии в CSV?

**О:** Реквизиты в выгрузке соглашения и запись реестра должны быть **согласованы по правилам ГИИС**; на практике удобно сшивать по **[ИНН](/glossary/inn)** и проверять расхождения наименований по **[ЕГРЮЛ](/information-systems/federal/egrul)**.
