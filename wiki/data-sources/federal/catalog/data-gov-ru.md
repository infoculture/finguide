# Федеральный портал открытых данных (data.gov.ru)

| Признак | Значение |
| :--- | :--- |
| Уровень и охват | Федеральный **каталог-каталог** открытых наборов; поставщиками остаются **размещающие** органы власти |
| Тип доступа | Портал: поиск по темам и организациям, карточки наборов, ссылки на файлы; при наличии — **API каталога** по документации оператора |
| Основные форматы | Метаданные — HTML и при машинном доступе JSON и др. по спецификации; **содержимое** наборов — CSV, JSON, XLSX, XML и др. в зависимости от поставщика |
| Оператор | [Минэкономразвития России](/organizations/minekonomiki) — оператор ГИС «Портал открытых данных РФ» |
| Режим доступа | Открытый для просмотра каталога и типовых загрузок; расширенные сценарии — по правилам портала |

Коротко: **официальная витрина метаданных** открытых наборов многих ведомств в **единой модели** — удобный **второй вход** после ведомственных сайтов для поиска наборов по теме **госфинансов**, не заменяя **операционные** системы **[ГИИС](/information-systems/federal/giis-eb)** и **[ЕИС](/information-systems/federal/zakupki)**.

## Описание

Карточка описывает **источник данных** в смысле **федерального каталога** `data.gov.ru`: какие объекты на уровне портала (наборы, поставщики, ресурсы) и как согласовать запись каталога с **первичной** публикацией на сайте ведомства.

Сюда **не** входят: **внутренние учётные** системы; **полнота** всех строк бюджета — каталог отражает только **размещённые** открытые наборы.

## Оператор

**Минэкономразвития России** (оператор портала) — https://data.gov.ru/

- Карточка ИС: **[Федеральный портал открытых данных](/information-systems/federal/data-gov-ru)** — подсистемы, ограничения дублей с ведомственными витринами и актуальность API.

## URL и точки доступа

| Тип доступа | URL | Формат |
| --- | --- | --- |
| Веб-каталог | https://data.gov.ru/ | HTML |
| Машинный доступ | разделы документации на **data.gov.ru** | JSON и др. по актуальной спецификации |

Точные **endpoint’ы** API и схемы полей **не фиксируются** в wiki — переносите из официальной документации при изменении версии портала.

## Объекты данных и показатели

- **Объект каталога** — **набор данных**: идентификатор, наименование, **поставщик**, тематические метки, **периодичность обновления**, лицензия, ссылки на **ресурсы** (файлы).
- **Содержимое таблиц** внутри файлов **не унифицировано** порталом — задаётся автором набора (Минфин, Росстат, казначейство и др.).
- **Связка с бюджетом** — через фильтр по поставщику и теме («бюджет», «казначейство», «закупки» и т.д.), затем сверка с **[budget.gov.ru](/data-sources/federal/budget-gov-ru-datasets)** и **[minfin.gov.ru](/data-sources/federal/minfin-opendata)**.

## Периодичность обновления

У **каждого набора** — по паспорту на `data.gov.ru`; глобальные изменения интерфейса и API — см. **«Ограничения и актуальность»** в **[карточке ИС](/information-systems/federal/data-gov-ru)**.

## Ограничения и особенности

- **Дубли и версии** — один показатель может быть **и** на ведомственном сайте, **и** в каталоге с разной датой обновления; для воспроизводимости фиксируйте **URL ресурса** и дату.
- **Полнота** — отсутствие набора в каталоге **не** означает отсутствие данных на **[ЕИС](/information-systems/federal/zakupki)** или **[казначейском портале](/information-systems/federal/roskazna-portal)**.
- **Юридическая опора** — при расхождении метаданных с ведомством уточняйте **первоисточник** у поставщика набора.

## Примеры доступа

Проверка доступности главной страницы каталога:

```bash
curl -sS -o /dev/null -w "%{http_code}\n" "https://data.gov.ru/"
```

Для выборки наборов через API используйте параметры из **актуальной** документации на портале.

## Частые вопросы

**В:** Достаточно ли одного data.gov.ru для всех открытых бюджетных данных?

**О:** **Нет** как гарантии полноты: часть наборов существует **только** на ведомственных порталах или в **СОИ ЕИС**; федеральный каталог — удобная **навигация**, а не единый реестр всех строк учёта.

**В:** Если даты обновления на data.gov.ru и на сайте Минфина различаются, что брать?

**О:** Для согласования с ведомством ориентируйтесь на **первоисточник** набора (часто **Минфин**); data.gov.ru — для **поиска** и единой модели метаданных.
