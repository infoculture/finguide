## Описание

Источник — **не** каталог открытых данных с `meta.json`, а **презентационный файл**, который Росстат вешает в блоке **«Государственные финансы»** на странице https://rosstat.gov.ru/statistics/finance. Имя файла на mediabank фиксировано (`gos_fin_byudget.xlsx`), а **отчётный год** и текст заголовков задаются **внутри книги** (при обновлении Росстат перезаписывает файл).

Сюда **не** входят: помесячное/поквартальное исполнение по КБК из **[ГИИС «Электронный бюджет»](/data-sources/federal/budget-gov-ru-datasets)**; операционные **[отчёты Казначейства](/data-sources/federal/roskazna-reports)**; региональные детализации в разрезе мер социальной поддержки — для них см. **[отдельные файлы раздела](/data-sources/federal/rosstat-gosfin-regional-social-support-expenditures)**.

## Объекты данных и показатели

- **Единица наблюдения** — **строка сводной таблицы** (статья дохода или итоговый блок) × **три числовых столбца**: по смыслу заголовков в книге это **итого по консолидированному бюджету РФ и ГВФ**, **федеральный бюджет**, **консолидированные бюджеты субъектов РФ** (точные подписи колонок берите из **строки шапки** листа `1` — при обновлениях верстка может слегка съезжать).
- **Лист `Содержание`** — навигация по листам и служебные реквизиты (ответственный исполнитель, дата обновления).
- **Лист `1`** — доходы по крупным налоговым и неналоговым группам, строка **«Расходы — всего»**, **«Дефицит(-)/Профицит»**; внизу — **сноски** (в проверенной версии указано, что информация по доходам — **по данным Федерального казначейства**, а **направления расходов не подлежат публикации** в этой форме).
- **Лист `2`** — диаграмма **профицита/дефицита** консолидированного и федерального бюджетов **в процентах к ВВП**. В проверенном файле значения живут в кеше диаграммы и external link, а не в таблице листа; полноценно прочитать годовой ряд из одного XLSX нельзя (см. ограничение ниже).
- **Связка с другими источниками:** для сопоставления с **[открытыми наборами Минфина/ГИИС](/data-sources/federal/budget-gov-ru-datasets)** нужна явная сверка **определений** «консолидированный бюджет» и состава фондов — см. **[глоссарий: консолидированный бюджет](/glossary/consolidated-budget)**.

## Проверенная структура XLSX и устойчивый парсинг

В снимке от 2026-07-18 книга содержит листы `Содержание` (`A1:N34`), `1` (`A1:M31`) и `2` (`A1:O33`). Указанные границы — `max_row`/`max_col` книги, поэтому включают пустые оформительские ячейки.

### Лист `1`: табличный блок

Заголовок с отчётным годом находится в `A2`: «Консолидированный бюджет Российской Федерации и бюджетов государственных внебюджетных фондов в **2023** г.1)». Единица в объединённой ячейке `A3:D3` — **млрд рублей**.

| Роль | Координата в проверенном файле | Как находить при импорте |
| --- | --- | --- |
| Строка «Доходы - всего» | `A8` | Нормализовать NBSP и тире, найти единственное вхождение подписи. |
| Итог по консолидированному бюджету РФ и ГВФ | `B8` | Первый непрерывный числовой столбец справа от доходного якоря; заголовок `B4:B7`. |
| Федеральный бюджет | `C8` | Следующий числовой столбец; заголовок `C6:C7`. |
| Бюджеты субъектов РФ | `D8` | Следующий числовой столбец; заголовок `D6:D7`. |
| «Расходы - всего» | `A28:D28` | Искать по отдельному единственному якорю. |
| «Дефицит(-)/Профицит» | `A29:D29` | Искать по отдельному единственному якорю. |
| Методическая сноска | `A31:D31` | По данным Федерального казначейства; направления расходов не публикуются. |

Ячейки `B:D` хранят числа с форматом `#,##0.0`; прочерк `-` — это строковое отсутствие значения, а не ноль. В проверенной версии он встречается в `C12`, `D12`, `C20` и `D22`.

Для чтения шапки нужно учитывать объединения `B4:B7`, `C4:D5`, `C6:C7` и `D6:D7`: виртуально распространить значение верхней левой ячейки **только по области шапки**, затем сопоставить подпись с уровнем бюджета. Не распространяйте значения merged-cells по строкам данных.

Стабильные машинные ключи для итогов: `revenue_total`, `expense_total`, `deficit_surplus`; уровень бюджета: `consolidated_budget_rf_and_state_extra_budgetary_funds`, `federal_budget`, `consolidated_budgets_of_subjects_rf`. Остальные доходные строки следует хранить как отдельные показатели, а не складывать их без подтверждённой методики в общий итог.

### Лист `2`: диаграмма, а не готовый ряд

На листе есть только объединённые заголовки `A2:O2` и `A3:H3` («в % к ВВП») и одна диаграмма. Её серии ссылаются на внешние формулы `[1]Данные!$C$4:$C$14` и `[1]Данные!$D$4:$D$14`; в кеше доступны по 11 значений. Однако категория года закеширована лишь для первой точки: `Данные!B4 = 2005`, тогда как `B5:B14` отсутствуют. Несмотря на заголовок «2005-2023 гг.», годы остальных точек в книге — **UNKNOWN**.

Поэтому не формируйте tidy-записи `(год, показатель, значение)` для листа `2`, пока не получен первичный табличный источник или новый XLSX с полным кешем категорий. Корректное поведение парсера — остановить импорт с диагностикой неполного кеша, а не присваивать годы по позиции.

## Периодичность обновления

Файл обновляется **эпизодически**, когда Росстат выпускает очередную редакцию свода; точную дату смотрите на **карточке файла** на https://rosstat.gov.ru/statistics/finance (поле вида «обновлено …») и в **листе `Содержание`**. Отдельного регламентного API с версионированием нет: URL файла стабилен, **содержимое** — подменяемое.

## Ограничения и особенности

- **Не «сырые» бюджетные строки:** нет КБК, нет помесячной кассы, нет муниципального среза в этом файле.
- **Расходы агрегированы:** в проверенной версии книги присутствует только **итог расходов** и **сальдо**; детализация расходов в этой выгрузке **намеренно не раскрывается** (см. сноску на листе `1`).
- **Верстка Excel:** многослойные шапки, объединённые ячейки, нечисловые прочерки `'-'` — для автоматического импорта обычно нужен **этап нормализации** (поиск строки с подписью колонок, приведение типов).
- **Лист `2` не является надёжным табличным рядом:** внешняя диаграмма содержит значения без соответствующих лет в кеше. Это ограничение проверенной книги, а не основание достраивать годы из заголовка.
- **HTTP-доступ:** для скриптов используйте **браузероподобный `User-Agent`** и умеренные паузы; «голый» curl иногда получает **403** (см. **[общую карточку Росстата](/data-sources/federal/rosstat-opendata)**).

## Примеры доступа

Проверка ответа сервера для прямого скачивания (код HTTP):

```bash
curl -sSfkL \
  -A "Mozilla/5.0 (compatible; research/1.0)" \
  -D /tmp/rosstat-gos_fin_byudget.headers \
  -o /tmp/rosstat-gos_fin_byudget.xlsx \
  "https://rosstat.gov.ru/storage/mediabank/gos_fin_byudget.xlsx"
file /tmp/rosstat-gos_fin_byudget.xlsx
```

Опция `-k` нужна только если в среде загрузчика сломана цепочка доверенных TLS-сертификатов; в штатной инфраструктуре проверку сертификата отключать не следует. Для кеширования передавайте сохранённые `ETag` и `Last-Modified` как `If-None-Match` и `If-Modified-Since`.

## Чек-лист перед импортом

- [ ] HTTP-ответ — `200` или `304`; сохранены `ETag`, `Last-Modified`, размер и дата скачивания.
- [ ] Файл начинается с `PK 03 04` и открывается как XLSX.
- [ ] Найдены листы `Содержание`, `1`, `2`; из заголовка листа `1` извлечён единственный год.
- [ ] Каждый из якорей «Доходы - всего», «Расходы - всего», «Дефицит(-)/Профицит» найден ровно один раз.
- [ ] Справа от строки доходов найдены три непрерывных числовых столбца и все три семантических уровня бюджета.
- [ ] Прочерки преобразованы в `null`, а не в нули; в итогах нет `NaN`, `Infinity` и `null`.
- [ ] Для каждого уровня выполняется тождество «доходы − расходы = дефицит/профицит» в установленном допуске.
- [ ] Для листа `2` число кешированных годов совпадает с числом значений в каждой серии; при несовпадении импорт ряда прекращён с диагностикой.

## Частые вопросы

**В:** Почему суммы доходов здесь и в открытых наборах `budget.gov.ru` не сходятся построчно?

**О:** Различаются **определение консолидированного бюджета**, **момент учёта**, **классификация статей** и **охват фондов**. Используйте этот файл как **опорный статистический свод Росстата**, а для юридически значимой отчётности — первоисточники **ГИИС/Казначейства** с явной методической сверкой.

**В:** Можно ли стабильно парсить лист `1` без правок кода при каждом обновлении?

**О:** Рискованно: меняются **год в заголовке**, состав строк и иногда **раскладка шапки**. Надёжнее искать **якорные подписи** («Доходы - всего», «Расходы - всего») и читать **динамический диапазон** колонок.

**В:** Можно ли использовать лист `2` как готовый многолетний набор?

**О:** Не в проверенной версии файла. Значения двух серий закешированы в диаграмме, но год задан только для первой точки (`2005`); присваивание оставшимся точкам годов из заголовка «2005-2023» было бы выдумыванием данных.
