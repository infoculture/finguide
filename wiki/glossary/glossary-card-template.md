# Шаблон карточки глоссария

Служебная страница для авторов и агентов: при добавлении или правке термина копируйте **блок ниже** (содержимое fenced-блока) в файл `wiki/glossary/<подпапка>/<slug>.md` (подпапка по теме — см. [оглавление глоссария](/glossary/)), замените плейсхолдеры и удалите неприменимые разделы целиком (без пустых заголовков). Нормативные требования к полноте текста, кодам, происхождению термина и ответственности органов — в OpenSpec **glossary-card-content**: `openspec/changes/update-glossary-editorial-depth/specs/glossary-card-content/spec.md` ([GitHub](https://github.com/infoculture/finguide/blob/master/openspec/changes/update-glossary-editorial-depth/specs/glossary-card-content/spec.md)); после архивации change — `openspec/specs/glossary-card-content/spec.md` в репозитории. Поле `glossary_kind` задаёт, какие блоки обязательны (см. ту спеку): для `classification` нужны расшифровка формата кода и разделение «ведение перечня» / «применение в сфере»; для `concept` без кода блок про структуру кода удалите, усилив контекст в «Определении» и «Происхождении». Раздел **«Примеры»** удаляйте только если иллюстрации не добавляют содержания к «Определение» и «Где встречается» — и укажите в пулл-реквесте одну фразу-обоснование. В frontmatter обязательно укажите `slug: /glossary/<slug>` (совпадает с именем файла без `.md`), чтобы URL не зависел от вложенности пути. У шаблона `draft: false` — в production-сборке страница не публикуется; в `npm start` файл всё равно компилируется.

### Поля frontmatter

| Поле | Обязательно | Пример |
|------|-------------|--------|
| `title` | да | `"КБК — Код бюджетной классификации"` |
| `sidebar_label` | да | `"КБК"` |
| `tags` | да | `[glossary, classification]` — вторая метка: тема (`budget`, `classification`, `procurement`, `transfers`, …) |
| `last_updated` | да | `2026-05-10` |
| `slug` | да для новых карточек | `/glossary/<slug>` — тот же `<slug>`, что имя файла |
| `content_type` | да | `concept` (по умолчанию для терминов) |
| `entity_type` | да | `glossary` |
| `relationships` | опционально | типизированные связи `{type, target}`; типы — в `scripts/relationship-types.json` |
| `glossary_kind` | да | `concept` \| `classification` \| `law` \| `process` \| `organization` — переключает обязательные блоки тела карточки (см. OpenSpec `glossary-card-content`) |
| `source_url` | по возможности | прямая ссылка на НПА или страницу Минфина |
| `abbreviations` | при наличии | `["КБК"]` |
