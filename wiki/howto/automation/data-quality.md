# Контроль качества данных

## Зачем

Базовые проверки: согласованность КБК, корректность сумм по уровням, отсутствие дублей, валидация дат.

## Шаги

1. Описать набор проверок (контрактов, экспектаций).
2. Запустить проверки на новой выгрузке.
3. Зафиксировать отчёт о качестве.

## Подводные камни

- Кодировки и формат дат могут различаться между источниками.
- Ограничения по rate limit — проверяйте документацию.
- Обновления могут приходить с задержкой по отношению к публикации.
