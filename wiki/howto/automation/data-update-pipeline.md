# Пайплайн автоматического обновления данных

## Зачем

Регулярная актуализация данных через GitHub Actions: расписание, скачивание, нормализация, коммит результатов.

## Шаги

1. Настроить cron в GitHub Actions.
2. Скачать данные и нормализовать.
3. Проверить контрольные суммы и закоммитить.

## Подводные камни

- Кодировки и формат дат могут различаться между источниками.
- Ограничения по rate limit — проверяйте документацию.
- Обновления могут приходить с задержкой по отношению к публикации.
