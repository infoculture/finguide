# Межбюджетные трансферты в открытых данных

## Зачем и для кого

Инструкция для **аналитиков бюджета**, которым нужно пройти путь от **агрегатов отчётности** до **микроданных соглашений** и обратно, не смешивая **федеральный** и **региональный** контур в одной таблице без методики.

## Входные данные

- **Финансовый год** и **уровень бюджета** (кто передатчик, кто получатель по смыслу строки).
- Публичные **отчёты об исполнении** с разрезом трансфертов — прежде всего связка [0503317](/reporting/0503317) и [0503387](/reporting/0503387).
- При необходимости микроуровня — выгрузки [реестра соглашений](/data-sources/federal/subsidy) с фильтром по виду документа и территории.
- Нормативный контекст — [глоссарий: межбюджетные трансферты](/glossary/transfers) и [БК РФ](/legal/budget-code) (гл. 16).

## Инструменты

- Браузер и скачивание **XLSX/ZIP** с портала [Федерального казначейства](/data-sources/federal/roskazna-reports) (разделы исполнения бюджетов).
- **Python 3** с `requests` и `csv`/`pandas` для сверки идентификаторов и сумм на одном уровне бюджета.
- Справочники **ОКТМО** и **КБК** на год из [открытых данных Минфина](/data-sources/federal/minfin-opendata).

## Шаги

1. Зафиксируйте **один** уровень отчёта (например консолидированный бюджет субъекта) и скачайте **0503387** к нему за выбранный год.
2. Сопоставьте **виды трансфертов** из формы с классификацией [трансфертов](/glossary/transfers) (дотации, субвенции, межбюджетные субсидии, иные).
3. Для проверки «кто кому» на уровне договоров откройте [реестр соглашений](/data-sources/federal/subsidy) и отфильтруйте **межбюджетные** виды; не подмешивайте **субсидии юрлицам** по ст. 78 БК РФ без отдельной задачи.
4. При сводке «федеральный + региональный» учитывайте **двойной счёт**: одна и та же сумма может входить в доходы получателя и расходы передатчика — исключайте дубли в модели.

## Воспроизводимый пример

Проверка доступности **сводного каталога** Минфина (часто нужен для справочников КБК при разборе трансфертных строк):

```bash
curl -fsSL -A "finguide-transfers-check/1.0" \
  "https://minfin.gov.ru/opendata/list.csv" | head -n 5
```

Ожидаемо: первые строки CSV с заголовком и служебной строкой `standardversion`; при HTML вместо CSV — смените `User-Agent` или скачайте вручную (см. [how-to по Минфину](/howto/access/minfin-api)).

## Проверка результата

- Суммы по **видам** в 0503387 **согласованы** с итоговыми блоками 0503317 на тот же период (в пределах методики округления).
- В модели нет **удвоения** трансферта между уровнями без строки «перевод между счетами».
- Для каждой цепочки к микроданным есть **дата выгрузки** и версия справочника КБК.

## Ограничения и типовые ошибки

- Публичные файлы **не** всегда содержат контрагента в машиночитаемом виде на уровне строки 0503387.
- **Субсидия** в разговорной речи ≠ **межбюджетная субсидия** в смысле гл. 16 БК РФ — сверяйте вид документа в реестре.
- Не сравнивайте **кассу** казначейства с **начислением** в корпоративных отчётах сторонних организаций.
