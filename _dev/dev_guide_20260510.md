# База знаний «Открытые государственные финансы России»: структура и контентный план

## Обзор концепции

Предлагаемая база знаний предназначена для обучения ИИ-ассистента и поддержки специалистов, работающих с бюджетными данными и данными государственных финансов России. В основе лежит репозиторий [opengovfinancesbook](https://github.com/infoculture/opengovfinancesbook), который уже содержит черновик книги. Финальная архитектура строится на трёх принципах: пригодность для RAG-индексации (семантически чёткие чанки, плоская структура заголовков), человекочитаемость через Docusaurus, и инкрементальное пополнение как вручную, так и автоматически.[^1][^2][^3]

***

## Принципы структурирования для RAG

Поскольку база знаний — это прежде всего источник для языковой модели, структура файлов должна соответствовать требованиям RAG-систем:[^3][^4]

- **Один файл — одна концепция или объект.** Не нужно помещать несколько независимых терминов в один большой файл.
- **Заголовки H1/H2 — смысловые якоря.** RAG-парсеры используют заголовки для семантического фрагментирования.[^5]
- **Избегать сложных таблиц.** Таблицы плохо читаются LLM при индексации — заменять многоуровневыми списками или плоской структурой.[^3]
- **Метаданные frontmatter в каждом файле.** Поля `id`, `title`, `category`, `tags`, `last_updated`, `source_url` улучшают точность поиска.[^6]
- **Q&A-секции** в конце каждого файла повышают качество ответов ассистента для типичных запросов.[^4]

## Автоматизированное наполнение

Следующие разделы хорошо поддаются автоматизации:

- **`budget-classification/`** — коды КБК, КОСГУ, КВР ежегодно публикуются Минфином в машиночитаемом формате. Скрипт конвертации: JSON/XML → Markdown.[^16][^10]
- **`data-sources/`** — метаданные датасетов из datacatalogs.ru и hubofdata.ru доступны через API.[^19][^22]
- **`reporting/treasury-reports.md`** — URL шаблоны отчётов Казначейства генерируются по расписанию.
- **`reference/links.md`** — автоматическая проверка доступности URL (link checker в CI/CD).
- **`reference/update-log.md`** — автоматическая фиксация коммитов с описанием изменений.

Рекомендуемый инструментарий: GitHub Actions для автоматических обновлений, Python-скрипты в папке `scripts/`, frontmatter-поле `auto_generated: true` для разметки автоматически созданных файлов.[^6]
