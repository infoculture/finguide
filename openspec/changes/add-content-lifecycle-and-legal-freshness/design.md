## Context

`last_updated` отвечает на вопрос о последней правке Markdown, но не о дате набора, из которого создана карточка, и не о дате проверки статуса организации. Для массовых исторических импортов нужны отдельные временные признаки. Проверка ссылок также должна различать детерминированно ошибочный URL и временную сетевую недоступность.

## Goals / Non-Goals

- Goals: отделить редакционную дату от даты снимка и статуса; обновить ключевой правовой контур местного самоуправления; блокировать синтаксически испорченные URL.
- Non-Goals: универсальный реестр правопреемства всех организаций; автоматическое толкование НПА; падение каждого PR из-за внешнего timeout/5xx.

## Decisions

- Decision: `as_of` — дата состояния исходного snapshot; `status_as_of` — дата последней проверки жизненного цикла; `last_updated` — дата изменения страницы.
- Decision: `lifecycle_status` использует контролируемые значения `current`, `renamed`, `reorganized`, `liquidated`, `unknown`.
- Decision: `successor_slug` задаётся только при существующей канонической карточке преемника и проходит ссылочную проверку. На карточке преемника MAY использоваться typed relationship `replaces` к прежней организации.
- Decision: юридическая актуальность подтверждается официальной публикацией и актуальной редакцией на дату реализации; proposal не фиксирует неизменяемую будущую дату перехода.
- Decision: синтаксически некорректные URL падают в обычном CI. Сетевые статусы остаются в отдельной scheduled-проверке с собственной политикой исключений.

## Risks / Trade-offs

- Статус части организаций может остаться неизвестным → `unknown` является допустимым честным результатом с `status_as_of` и пояснением.
- Новые поля увеличивают редакционную нагрузку → они обязательны только для snapshot/import семейств и существенно обновляемых карточек с временным риском.
- Правовая дата может измениться после реализации → карточки содержат дату проверки и официальный источник, а отчёт свежести выявляет просрочку.

## Migration Plan

1. Добавить словарь полей и проверки без изменения существующих карточек.
2. Исправить синтаксически испорченные URL и включить гейт.
3. Создать/обновить три правовые карточки и проверить переходные ссылки.
4. Прогнать GovNGO-аудит, затем мигрировать карточки волнами с сохранением исходного snapshot.
5. Включить отчёт свежести и документировать остаток `unknown`.

## Open Questions

- Порог просрочки для разных типов нормативных документов и организаций устанавливается редакционной политикой при реализации, но MUST быть явным в отчёте.
