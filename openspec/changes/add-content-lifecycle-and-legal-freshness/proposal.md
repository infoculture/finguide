# Change: Добавить управление жизненным циклом и актуальностью правовой базы

## Why

Массовые карточки организаций, созданные по снимку GovNGO 2020, имеют свежую дату редактирования, но не отличают дату исходного снимка от текущего статуса организации. Одновременно карточка 131-ФЗ не отражает новый контур 33-ФЗ и переходное регулирование, а часть импортированных внешних ссылок содержит закодированные редакционные примечания внутри URL. Эти дефекты создают ложное ощущение актуальности.

## What Changes

- Ввести для snapshot-карточек организаций поля `as_of`, `lifecycle_status`, `status_as_of` и опциональный `successor_slug`.
- Провести аудит семейства GovNGO 2020 и явно отметить действующие, переименованные, реорганизованные, ликвидированные и не подтверждённые записи.
- Добавить канонические карточки 33-ФЗ и 414-ФЗ, актуализировать карточку 131-ФЗ и связать переходный контур без дублей одного акта.
- Добавить синтаксическую проверку внешних URL, запрещающую встроенные пробелы и закодированные редакционные примечания.
- Добавить воспроизводимый отчёт свежести для нормативных карточек и snapshot-семейств.

## Impact

- **Затронутые capabilities:** новый `content-freshness-governance`.
- **Затронутый контент:** `wiki/organizations/federal-gos-nko/`, `wiki/legal/`, шаблоны организаций и нормативных документов, README и `sidebars.ts`.
- **Затронутый код:** frontmatter lint, проверка внешних URL, возможный отчёт свежести и импортёр GovNGO.
- **Совместимость:** `last_updated` сохраняет значение даты редакционной правки; новые поля аддитивны и описывают дату фактов отдельно.
- **Не входит в change:** автоматическое юридическое заключение о правопреемстве без официального источника и сетевой CI-гейт на любой временно недоступный сайт.
