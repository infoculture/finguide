## ADDED Requirements

### Requirement: Pilot regional information system cards

Проект SHALL содержать от 5 до 10 индивидуальных карточек региональных информационных систем в [`wiki/information-systems/regional/`](../../../../../wiki/information-systems/regional/). Выборка MUST включать несколько развитых публичных витрин пилотных субъектов и минимум один документированный контрастный пример, где публичный доступ ограничен или нестабилен.

Каждая карточка MUST указывать оператора, назначение, объекты данных, режимы доступа и 2–3 конкретных источника данных. Если публичных наборов нет, карточка MUST явно описывать отсутствие и перечислять доступные публичные производные.

#### Scenario: Reader moves from system to source

- **GIVEN** пользователь открывает карточку региональной ИС
- **WHEN** он читает раздел «Данные: объекты и связь с источниками»
- **THEN** он получает конкретные источники либо явное и проверяемое описание ограничения доступа

### Requirement: Entity-type separation in pilot matrix

Матрица пилотных субъектов SHALL разделять ссылки на карточку организации-владельца, карточку ИС и карточки источников данных. Ссылка на источник или общий портал MUST NOT использоваться как значение колонки «организация».

Если сущность ещё не покрыта, ячейка MUST содержать явную отметку «карточки нет на дату проверки», а не предполагаемый slug несуществующей страницы.

#### Scenario: Matrix links preserve entity meaning

- **GIVEN** строка субъекта содержит ссылки на владельца и портал
- **WHEN** пользователь открывает каждую ссылку
- **THEN** ссылка владельца ведёт на `entity_type: organization`, а ссылка портала — на ИС или источник соответствующего типа

### Requirement: Pilot source-family gap matrix

Для каждого из десяти пилотных субъектов проект SHALL фиксировать состояние покрытия семейств: государственный долг, государственные программы, закупки, внешний финансовый контроль, региональный портал открытых данных и муниципальные бюджетные публикации.

Для каждого семейства MUST использоваться одно из состояний: «есть проверенная карточка», «публичный след найден, карточка запланирована», «не найдено на дату проверки», «неприменимо». Состояние «есть» MUST ссылаться на конкретную карточку источника.

#### Scenario: Editor can prioritize regional gaps

- **GIVEN** редактор сравнивает два пилотных субъекта
- **WHEN** он смотрит семейство государственного долга или контроля
- **THEN** он видит одинаковые состояния покрытия и конкретные ссылки для заполненных ячеек

## MODIFIED Requirements

### Requirement: Regional finance authority cards

Для всех десяти пилотных субъектов проект SHALL содержать карточки региональных органов исполнительной власти, ответственных за бюджетное раскрытие и данные (региональные Минфины / департаменты финансов), в [`wiki/organizations/`](../../../../../wiki/organizations/) по шаблону [`wiki/organizations/organizations-card-template.md`](../../../../../wiki/organizations/organizations-card-template.md).

Матрица MUST ссылаться на существующий канонический slug каждой организации. Карточка MUST содержать официальное название и статус на дату проверки, витрины, связанные ИС и конкретные источники данных; общий раздел каталога не заменяет эти ссылки.

#### Scenario: Analyst finds data owner

- **GIVEN** пользователь читает матрицу по региону
- **WHEN** он переходит на карточку организации
- **THEN** он видит витрины, ИС и связанные источники данных

#### Scenario: Every pilot row has a real owner card

- **GIVEN** матрица содержит десять строк пилотных субъектов
- **WHEN** выполняется редакционная проверка ссылок владельцев
- **THEN** каждая строка ведёт на существующую карточку организации с актуальной датой проверки
