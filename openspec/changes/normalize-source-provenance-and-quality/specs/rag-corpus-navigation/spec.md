## ADDED Requirements

### Requirement: Mass-import family reachability

Каждое массово импортированное семейство опубликованных карточек SHALL иметь канонический хаб или иерархию хабов, достижимую из README соответствующего раздела. Каждая карточка семейства MUST быть достижима по внутренним Markdown-ссылкам из такого хаба либо из детерминированной обзорной страницы семейства.

Карточки MAY сохранять `rag_priority: low`, но низкий приоритет MUST NOT служить заменой читательской навигации.

#### Scenario: Imported card has an inbound route

- **GIVEN** опубликована карточка из массового импорта ФНС или «Электронного бюджета»
- **WHEN** строится граф внутренних ссылок
- **THEN** существует маршрут от README раздела через хаб семейства к этой карточке

### Requirement: Navigation coverage audit

Проект SHALL генерировать детерминированный отчёт по опубликованным карточкам без входящих ссылок с группировкой по семейству и типу сущности. Согласованные исключения MUST быть перечислены явно и не зависеть от абсолютного пути рабочей копии.

#### Scenario: New orphan is visible in review

- **GIVEN** pull request добавляет опубликованную карточку, не включённую ни в один хаб
- **WHEN** выполняется аудит навигации
- **THEN** путь и slug карточки появляются в отчёте или проверка завершается ошибкой по принятой политике CI
