# Change: Добавить приоритетные федеральные домены данных

## Why

Федеральный раздел велик по числу импортированных карточек, но в нём отсутствуют несколько аналитически важных контуров: государственное имущество, ГИС ГМП, открытые данные социальных фондов, отдельные семейства долга и фискальных рисков, а также разложение ЕИС по реестрам. Из-за этого пользователь видит много однотипных записей, но не получает маршрута «владелец → система → конкретный набор → ограничение доступа» для ключевых задач.

## What Changes

- Ввести тематическую группу `wiki/data-sources/federal/property-assets/` при сохранении плоских публичных slug федеральных источников.
- Создать связанный пакет по государственному имуществу: Росимущество, публичный контур ФГИАС ЕУСГИ / «Федеральное имущество онлайн», приватизация, пакеты акций и доходы от имущества.
- Создать карточку ГИС ГМП и отделить закрытый транзакционный контур от публичных агрегатов и производных публикаций.
- Добавить карточки каталога открытых данных СФР, публикаций ФФОМС и методику сверки с Казначейством и «Электронным бюджетом».
- Покрыть отдельными источниками федеральный долг, гарантии и фискальные риски, публичные инвестиции, результаты финансового контроля и четыре ключевых реестровых семейства ЕИС.
- Добавить карточки недостающих федеральных владельцев данных и углубить покрытие конкретных форм отчётности, для которых подтверждён публичный след.

## Impact

- **Затронутые capabilities:** новый `federal-fiscal-source-coverage`; содержательно связан с `reporting-landscape` и `reproducible-analytics`.
- **Затронутый контент:** `wiki/data-sources/federal/`, `wiki/information-systems/federal/`, `wiki/organizations/`, `wiki/reporting/`, `wiki/howto/`, федеральные README и `sidebars.ts`.
- **Правила репозитория:** `AGENTS.md` MUST разрешить новую подпапку `property-assets/`; публичные URL федеральных карточек остаются плоскими.
- **Машинные артефакты:** после реализации регенерируются все файлы `exports/` и дополняется QA-корпус.
- **Не входит в change:** зеркалирование закрытых реестров, публикация персональных начислений/платежей и создание не подтверждённых официальными источниками карточек форм.
