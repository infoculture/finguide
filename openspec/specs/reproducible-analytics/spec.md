# Capability: reproducible-analytics

Полные аналитические маршруты от вопроса до проверяемого результата: каталог сценариев, целевые how-to, мини-набор кода и аудит воспроизводимости существующих инструкций. Источник целей: [_dev/content_plan_20260514.md](../../../_dev/content_plan_20260514.md), разд. 5.4, 6, 7.

## Requirement: Scenario catalog

Проект SHALL поддерживать явный каталог сценариев (отдельная страница в [`wiki/intro/`](../../../wiki/intro/) или [`wiki/howto/`](../../../wiki/howto/) или раздел README how-to), перечисляющий как минимум маршруты:

1. Оценка исполнения бюджета региона (региональные источники, Казначейство, форма 0503317, ОКТМО, КБК).
2. Поиск расходов на государственную программу (КБК, целевые статьи, госпрограммы, отчётность, `budget.gov.ru`).
3. Анализ закупок учреждения (`bus.gov.ru`, ЕИС, ЕГРЮЛ, КБК/КОСГУ при наличии).
4. Проверка межбюджетных трансфертов (формы отчётности, КБК, законы о бюджете, региональные порталы).
5. Сравнение регионов (ОКТМО, консолидированные бюджеты, население, дефляторы или иной согласованный нормализатор).

Каждый пункт каталога MUST ссылаться на опубликованные страницы корпуса (источники, отчётность, how-to, глоссарий) с одной строкой пояснения к каждой ссылке.

### Scenario: Catalog entry is complete path

- **GIVEN** читатель открывает каталог сценариев
- **WHEN** он выбирает сценарий
- **THEN** он может пройти цепочку ссылок до карточки данных и инструкции без тупиковых заглушек

## Requirement: Interbudgetary transfers how-to

Проект SHALL содержать опубликованный how-to [`wiki/howto/analysis/interbudgetary-transfers.md`](../../../wiki/howto/analysis/interbudgetary-transfers.md) (или согласованный путь под `wiki/howto/`), соответствующий [`howto-playbook`](../howto-playbook/spec.md): входные данные, инструменты, шаги, воспроизводимый пример, проверка результата, ограничения, связанные страницы.

### Scenario: How-to passes structure lint

- **GIVEN** страница опубликована без `draft: true`
- **WHEN** выполняется `npm run lint:howto`
- **THEN** страница удовлетворяет эвристике обязательных разделов

## Requirement: Regional budget execution case

Проект SHALL содержать опубликованный how-to полного пути анализа бюджета субъекта РФ: [`wiki/howto/analysis/regional-budget-execution-case.md`](../../../wiki/howto/analysis/regional-budget-execution-case.md) (или согласованный путь), с явной связкой региональных источников, форм отчётности и проверок полноты.

### Scenario: Case links reporting and regional sources

- **GIVEN** читатель открывает кейс
- **WHEN** он следует разделу «Связанные страницы»
- **THEN** он находит карточку регионального источника и форму отчётности из корпуса

## Requirement: Minimal reproducible code bundle

Проект SHOULD содержать мини-набор скриптов или notebooks в каталоге **`examples/analytics/`** в корне репозитория (создать при первом сценарии).

Каждый пример MUST включать в README или в шапке скрипта:

- источник входных данных (URL или путь к согласованному небольшому фикстурному подмножеству);
- шаги обработки;
- проверки полноты или согласованности;
- ожидаемый артефакт (таблица, CSV, график);
- ограничения и лицензия использования данных.

### Scenario: Contributor runs example from clean clone

- **GIVEN** разработчик клонировал репозиторий
- **WHEN** он следует README в `examples/analytics/`
- **THEN** он может воспроизвести минимальный результат с задокументированными зависимостями

## Requirement: How-to reproducibility audit

Проект SHALL выполнить аудит существующих опубликованных how-to в [`wiki/howto/`](../../../wiki/howto/) на воспроизводимость и зафиксировать результат в артефакте редакции: таблица в [`wiki/intro/`](../../../wiki/intro/) или Issue/Milestone (ссылка из [`wiki/intro/wiki-filling-plan.md`](../../../wiki/intro/wiki-filling-plan.md) или отдельная страница «аудит how-to»).

Аудит MUST классифицировать каждую страницу как минимум по признакам: есть ли рабочий пример кода или команды; указаны ли входные данные; есть ли проверка результата.

### Scenario: Audit lists gaps

- **GIVEN** редакция открывает артефакт аудита
- **WHEN** она фильтрует страницы без воспроизводимого примера
- **THEN** получает приоритизированный список доработок
