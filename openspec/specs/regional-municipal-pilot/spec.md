# Capability: regional-municipal-pilot

Пилотное регионально-муниципальное покрытие: сопоставимая матрица раскрытия по субъектам РФ, крупные муниципалитеты, карточки региональных финорганов и навигация к источникам и how-to. Источник целей: [_dev/content_plan_20260514.md](../../../_dev/content_plan_20260514.md), разд. 5.2, 6, 7.

**Канон путей:** материалы уровня «источник данных» для муниципальных и региональных витрин размещаются под [`wiki/data-sources/regional/`](../../../wiki/data-sources/regional/). Каталог `wiki/data-sources/municipal/` не используется без изменения [`AGENTS.md`](../../../AGENTS.md).

**Файловая иерархия и публичный URL:** карточки источников MAY размещаться во вложенных папках `wiki/data-sources/regional/<fed_okrug_slug>/<subject_slug>/` при сохранении публичного `slug: /data-sources/regional/<имя-файла-без-md>` (без сегментов округа и субъекта в URL). Матрица, README и прочие перекрёстные ссылки в корпусе MUST продолжать указывать цели как `/data-sources/regional/<slug>`. Детали, сводные страницы субъектов и импорт из таблицы исследования — в capability [`regional-sources-hierarchy`](../regional-sources-hierarchy/spec.md).

## Requirement: Pilot ten-subject disclosure matrix

Проект SHALL содержать страницу-матрицу раскрытия для **десяти** пилотных субъектов РФ (канонический список из отчёта: Москва; Санкт-Петербург; Республика Татарстан; Краснодарский край; Свердловская область; Республика Саха (Якутия); Республика Дагестан; Кемеровская область; Ханты-Мансийский автономный округ — Югра; Калининградская область).

Матрица MUST фиксировать для каждого субъекта сопоставимые признаки (да/нет/краткое пояснение):

- открытый бюджет / «бюджет для граждан» (или эквивалентная публичная витрина);
- машиночитаемые открытые данные (портал наборами);
- API (стабильный или ad hoc — указать);
- архив законов о бюджете;
- отчёты об исполнении бюджета в открытом доступе;
- удобство поиска (краткая качественная оценка с критериями в тексте страницы).

Файл размещается в [`wiki/data-sources/regional/`](../../../wiki/data-sources/regional/) (например `regional-disclosure-matrix.md`) с `slug: /data-sources/regional/<имя-файла-без-md>`.

### Scenario: Reader compares two regions

- **GIVEN** пользователь открывает матрицу
- **WHEN** он сравнивает два субъекта по одной строке признака
- **THEN** формулировки сопоставимы (одинаковая семантика колонок)

### Scenario: Matrix links to sources

- **GIVEN** для субъекта указан портал или набор
- **WHEN** в корпусе есть карточка источника
- **THEN** матрица содержит ссылку на `/data-sources/regional/<slug>` с пояснением строки

## Requirement: Regional finance authority cards

Для пилотных субъектов проект SHOULD содержать карточки региональных органов исполнительной власти, ответственных за бюджетное раскрытие и данные (региональные Минфины / департаменты финансов), в [`wiki/organizations/`](../../../wiki/organizations/) по шаблону [`wiki/organizations/organizations-card-template.md`](../../../wiki/organizations/organizations-card-template.md).

Если карточка откладывается, в матрице MUST быть явная отсылка к планируемому slug или причина исключения на дату обновления страницы.

### Scenario: Analyst finds data owner

- **GIVEN** пользователь читает матрицу по региону
- **WHEN** он переходит на карточку организации
- **THEN** он видит витрины, ИС и связанные источники данных

## Requirement: Large municipality coverage

Проект SHOULD содержать не менее **десяти** карточек или агрегированных записей по крупным муниципалитетам (города-миллионники и/или административные центры) с **единой** сеткой полей, согласованной с матрицей субъектов (те же признаки, где применимо).

Размещение: [`wiki/data-sources/regional/`](../../../wiki/data-sources/regional/) (например обзор `large-city-budgets.md` со ссылками на отдельные карточки при росте корпуса).

### Scenario: Municipal row links to federal context

- **GIVEN** читатель изучает муниципальную строку
- **WHEN** нужен федеральный контекст (например `budget.gov.ru`)
- **THEN** страница объясняет ограничение федерального входа и отсылает к проверке сайта МО

## Requirement: Interbudgetary context links

Матрица и связанные страницы SHOULD перекрёстно ссылаться на:

- how-to по межбюджетным трансфертам ([`reproducible-analytics`](../reproducible-analytics/spec.md));
- релевантные формы отчётности и классификацию ([`reporting-landscape`](../reporting-landscape/spec.md), [`budget-classification-temporal`](../budget-classification-temporal/spec.md)).

### Scenario: Transfer analysis path

- **GIVEN** аналитик изучает регион в матрице
- **WHEN** его интересуют дотации и субвенции
- **THEN** с матрицы есть путь к how-to или карточкам отчётности без поиска по всему сайту
