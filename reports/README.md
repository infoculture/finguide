# Отчёты управления корпусом

`corpus-governance-audit.json` генерируется командой `npm run audit:corpus` и не редактируется вручную.

Политика проверки хранится в `scripts/data/corpus-governance-policy.json`:

- `audit_as_of` — единая дата расчёта, чтобы отчёт был воспроизводимым;
- нормативная карточка считается просроченной через 180 дней после `last_verified` либо при отсутствии даты;
- карточка snapshot-семейства считается просроченной через 365 дней после `status_as_of` либо при отсутствии даты;
- `orphan_exceptions` — явный список исключений из проверки навигационной достижимости; сейчас исключений нет.
- `quality_exceptions` — семейные исключения из первой волны миграции качества: для 67 legacy-карточек curated-федерального семейства и пяти гражданских источников `last_verified` не выводится из `last_updated`, а остаётся пустым до индивидуальной сетевой проверки. Остальные обязательные поля качества этих семейств заполнены.

Раздел `lifecycle.inventory` перечисляет 58 карточек импорта GovNGO 2020. `snapshot_name` и `official_site` относятся к исходной карточке; `current_name` остаётся `null`, пока текущий статус не подтверждён. `possible_successor_slug` заполняется только проверенной канонической ссылкой, без автоматического предположения о правопреемстве.

Раздел `navigation` группирует карточки без входящих ссылок по `entity_type`, а источники — дополнительно по семейству импорта. Достижимость семейств считается от корней из политики; согласованные исключения публикуются в отчёте даже при пустом списке. Раздел `freshness.groups` отдельно показывает нормативные карточки и snapshot-семейство GovNGO, включая долю честного статуса `unknown`.

Региональное семейство не входило в обязательную первую волну заполнения всех полей качества: импорт сохраняет неизвестные `archive_depth`, `legal_significance`, `license_or_terms` и `update_lag` пустыми. Их численность остаётся видимой в `source_families.regional.quality`, чтобы следующая редакционная волна не подменялась автоматически придуманными значениями.
