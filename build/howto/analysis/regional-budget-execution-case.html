Кейс: исполнение бюджета субъекта РФ по открытым контурам

Зачем и для кого​

Инструкция для аналитика, которому нужен один последовательный маршрут от официальных публикаций субъекта до машиночитаемой годовой отчётности 0503317 и обратно — с явными проверками, что сравнивается консолидированный бюджет субъекта, а не федеральный бюджет РФ.

Входные данные​

  • Субъект РФ (для поиска на портале) и финансовый год исполнения.
  • Ссылка на закон о бюджете субъекта на официальном портале (или архив) — для смыслового якоря плановых цифр и примечаний.
  • Каталог годовых файлов 0503317 в разделе консолидированные бюджеты субъектов и/или региональная витрина.
  • При необходимости стыковки территорий — справочник ОКТМО и обзор классификации для КБК.

Инструменты​

  • Браузер для скачивания XLSX с сайта субъекта и с портала Казначейства.
  • Python 3 или Excel — для выборки итогов по листам; детальный разбор вёрстки Excel см. в анализе региональных бюджетов.

Шаги​

  1. Найдите официальную публикацию исполнения на сайте субъекта (часто раздел «Бюджет» / «Исполнение») по методике как искать региональные данные; зафиксируйте дату публикации и формат (PDF, XLSX).
  2. Скачайте годовой файл 0503317 для того же года из федерального зеркала (см. консолидированные бюджеты субъектов); убедитесь, что имя файла относится к нужному субъекту (в архивах Казначейства код региона часто зашит в имя файла).
  3. Сопоставьте периметр: в тексте закона и в пояснениях к 0503317 должно быть консолидированное исполнение субъекта, а не только «республиканская часть» без муниципалитетов — при расхождении остановитесь и перечитайте консолидированный бюджет.
  4. Сверьте итог по расходам (или доходам) между региональной витриной и агрегатом из 0503317 в допусках округления и методики; расхождения фиксируйте как гипотезу (разные даты среза, разные версии справочника КБК, неполный консолидированный периметр).
  5. Для разреза по экономике расходов используйте поля отчёта и связку с КВР; для построчной расшифровки кодов — расшифровку КБК.

Воспроизводимый пример​

Проверка доступности раздела исполнения на портале Федерального казначейства (ожидаемый HTTP-код 200):

curl -fsSL -A "finguide-regional-case/1.0" -o /dev/null -w "%{http_code}\n" \
  "https://roskazna.gov.ru/ispolnenie-byudzhetov/"

Для выборки конкретного XLSX по году используйте навигацию из карточки отчёты Казначейства и 0503317; автоматизацию загрузки архивов не фиксируйте как «вечный» URL без проверки Last-Modified.

Проверка результата​

  • Совпадают год и субъект во всех трёх слоях: закон/витрина региона, имя файла 0503317, заголовки таблицы.
  • Итоговые расходы/доходы согласованы между региональной публикацией и 0503317 или задокументировано объяснение расхождения.
  • Для дальнейшей панели по годам учтены смены макета Excel (см. анализ региональных бюджетов).

Ограничения и типовые ошибки​

  • Смешение федерального бюджета РФ и консолидированного бюджета субъекта в одной таблице без пометки периметра.
  • Использование месячных форм там, где нужен годовой итог (и наоборот).
  • Подмена плана законом о бюджете фактом исполнения без отдельного столбца.
