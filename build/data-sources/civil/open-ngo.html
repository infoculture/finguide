OpenNGO.ru — финансирование НКО

ПризнакЗначение
Уровень и охватГражданский агрегатор; НКО по всей РФ, сквозное объединение субсидий, грантов, госконтрактов и справки по организациям
Тип доступаПубличный веб-портал; отдельная страница открытых данных — дампы и описание JSON API у оператора (не федеральный реестр)
Основные форматыHTML (поиск и тематические разделы); JSON Lines внутри ZIP (дампы профилей НКО); JSON (ответы HTTP API по документации проекта)
ОператорНП «Информационная культура»
Режим доступаОткрытый просмотр без регистрации; условия переработки — в материалах оператора на сайте

Коротко: одна точка входа в интерфейсе, чтобы по ИНН или названию НКО увидеть рядом бюджетную поддержку, конкурсные гранты (в т. ч. контур ФПГ) и контракты; машиночитаемый канал на openngo.ru/opendata/ даёт прежде всего агрегированные профили организаций (JSON Lines в ZIP и JSON API), со схемой полей и условиями — у оператора.

Описание​

Карточка описывает публичную витрину OpenNGO.ru: сводные разделы по субсидиям и грантам НКО, государственным и муниципальным контрактам с НКО и справочному контуру по НКО (включая опору на публичные реестры). Отдельно у оператора опубликована страница Открытые данные: сведения о российских НКО в виде «профилей организаций» (база 700 тыс.+ записей); большие дампы режутся на части (типично до ~20 тыс. записей на файл), внутри архивов — JSON Lines. Для полей, лицензии и API см. описание данных и параметры API в репозитории проекта. Сервис не является юридически значимым реестром и не подменяет учётные форматы ГИИС «Электронный бюджет», ЕИС или портала ФПГ.

Сюда не входят закрытые кабинеты без публичной документации. Сквозную сверку субсидий, контрактов и грантов по суммам и статусам для продакшена по-прежнему закладывайте на открытые данные первичных операторов (см. раздел «Связанные страницы wiki»); дампы OpenNGO ориентированы на массовую загрузку профилей НКО и связанных с ними полей по схеме оператора, а не на полную замену экранов «финансирование» в интерфейсе.

Оператор​

НП «Информационная культура» — каноническая витрина: https://openngo.ru/

  • Карточка ИС: OpenNGO.ru — назначение агрегатора, типовые задачи и оговорки относительно первичных систем.

URL и точки доступа​

Тип доступаURLФормат
Публичный веб-порталhttps://openngo.ru/HTML
Открытые данные (метаданные набора, ссылки на дампы ZIP, условия использования)https://openngo.ru/opendata/HTML + ссылки на ZIP (JSON Lines внутри); условия — terms
Размещение файлов дампов (каталог на ngodata.ru; список файлов дублирует страницу открытых данных)См. прямые ссылки на ngo_dump_*.zip на https://openngo.ru/opendata/ZIP → JSON Lines

Объекты данных и показатели​

В веб-интерфейсе на пересечении стоят получатель (НКО), источник или программа поддержки, сумма и период, документ соглашения или контракта (в объёме, подтянутом из первичных витрин), идентификаторы организации для связки с реестрами. Глубина истории и полный перечень полей зависят от интеграций оператора.

В открытых дампах и по API публикуется прежде всего профиль НКО (одна строка JSON Lines на организацию в выгрузке); конкретный набор полей, справочники и вложенные структуры — в wiki проекта. Не ожидайте, что каждый показатель из HTML-разделов про субсидии и контракты воспроизведён в том же виде в строке дампа без сверки со схемой.

  • Типичные ключи связи: ИНН, ОГРН (где отображаются), реестровые номера контрактов и соглашений — для перехода в ЕИС, наборы budget.gov.ru и портал президентских грантов.
  • Пробелы: агрегатор может отставать, нормализовать наименования иначе, чем первичный источник; для публикации выводов сверяйте суммы и статусы в официальных витринах.
  • ЕГРЮЛ и реестр НКО Минюста обновляются независимо; дополнительные поля реестра НКО не дублируются автоматически в выписке ЕГРЮЛ и наоборот. Для цепочки учредителя и классификации государственных и подведомственных НКО опирайтесь на ЕГРЮЛ и методику исследования, а не на один только интерфейс агрегатора.

Периодичность обновления​

На странице открытых данных для набора указаны ежедневное обновление и дата последней публикации (проверяйте актуальное значение на витрине). Фиксированного SLA как у государственного регистра нет; файлы дампов на ngodata.ru могут иметь иную отметку времени, чем живой API или интерфейс. Задержки и расхождения с budget.gov.ru, ЕИС и pgrants возможны из‑за расписания обновления и нормализации на стороне оператора. Дополнительные оговорки — в разделе «Ограничения и актуальность» карточки OpenNGO.ru (ИС).

Ограничения и особенности​

  • Не первичный реестр: для юридически значимой и журналистской проверки используйте документ в системе-источнике (реестр соглашений, карточка контракта в ЕИС, материалы конкурса на портале ФПГ).
  • Лицензия и атрибуция: условия повторного использования — terms и сопроводительные тексты на сайте оператора; при цитировании указывайте источник по их правилам.
  • Дампы и API vs тематические разделы сайта: официальные ZIP с JSON Lines и JSON API закрывают сценарий машинной работы с профилями НКО по документации проекта; для регулярной сверки сумм и статусов по субсидиям, грантам и контрактам по-прежнему используйте открытые данные первичных операторов ниже. Парсинг HTML без отдельной договорённости с оператором не заменяет эти каналы.
  • Объём и разбиение файлов: полная база — сотни тысяч записей; выгрузки разбиты на несколько ZIP; закладывайте хранение и пайплайн по частям.

Примеры доступа​

Список актуальных архивов и их размеров — на openngo.ru/opendata/ (прямые ссылки ведут на каталог ngodata.ru). В каждом ZIP лежит один текстовый файл JSON Lines (расширение в архиве обычно .json, одна JSON-запись на строку); точное имя файла смотрите в unzip -l.

Скачать первый фрагмент дампа и вывести первые две строки без распаковки на диск:

curl -LO "https://ngodata.ru/dataset/19b2e25f-89a6-4c05-b1c9-9317ccbfd6b2/resource/d5de6317-b7a1-4324-a6d9-f0f5bf41412a/download/ngo_dump_01.zip"
unzip -l ngo_dump_01.zip
unzip -p ngo_dump_01.zip ngo_dump_01.json | head -n 2

Для остальных частей подставьте соответствующие имена архива и файла из таблицы на странице открытых данных.

Связанные страницы wiki​

  • OpenNGO.ru (ИС) — границы сервиса, сценарии пользователей и связь с первичными системами.
  • Президентские гранты (источник данных) — официальные файлы и витрина ФПГ; один из контуров, который отражается в OpenNGO рядом с другими видами поддержки.
  • Наборы данных ГИИС «Электронный бюджет» — открытые реестры и наборы для сверки бюджетных соглашений с НКО.
  • Соглашения (субсидии и субвенции) — федеральный контур реестра соглашений; точка сверки для строк «бюджетной» поддержки.
  • Сведения о закупках и контрактах — машиночитаемые потоки и справка по контрактному следу, который в OpenNGO показывается обобщённо.
  • Государственные и подведомственные НКО — как в данных отличить аналитическую категорию НКО с публичным учредителем от типового учреждения.
  • Гранты и Субсидии (ст. 78 и 78.1 БК РФ) — как различать конкурсную поддержку и безвозмездную поддержку из бюджета в учёте.

Не путать с​

  • Президентские гранты (pgrants) — официальный оператор и открытые файлы только по конкурсному контуру ФПГ; OpenNGO смешивает этот контур с другими ведомствами и типами поддержки.
  • Соглашения о субсидиях и наборы budget.gov.ru — первичный бюджетный контур для соглашений; OpenNGO даёт обзор, но не заменяет выгрузку из ГИИС «Электронный бюджет».
