ЕИС Закупки: статистика на главной (госзакупки и контракты)

ПризнакЗначение
Уровень и охватФедеральный; агрегированные показатели по закупкам и контрактам, размещаемым в ЕИС, с разделением 44-ФЗ и 223-ФЗ
Тип доступаСмешанный: интерактивный блок на главной странице; JSON для графиков и карты (относительно contextPath портала); отдельный файл методики в PDF
Основные форматыHTML (оболочка и вкладки), JSON (серии для слайдера и карты), PDF (методика расчёта)
ОператорФедеральное казначейство
Режим доступаПубличный просмотр; для HTTP-запросов к JSON на практике может потребоваться типовой User-Agent и заголовок Referer с главной страницы портала

Коротко: официальная сводная статистика ЕИС на главной странице: экономия, планирование, извещения, контракты, способы закупки и оперативные показатели за текущий день — с картой по регионам и переключением 44-ФЗ / 223-ФЗ. Это не замена построчных данных и XML-выгрузок, а слой обзорной аналитики и сверки с методикой оператора.

Описание​

Блок «Статистика» (#statAnchor) на https://zakupki.gov.ru/epz/main/public/home.html содержит:

  • вкладку «Общая» — кнопки режима 44-ФЗ / 223-ФЗ и срезы «Экономия», «Планирование», «Извещения», «Контракты»; отображение «Карта» или «Цифры» (соответствуют параметрам запросов к JSON);
  • вкладку «Способы закупки» — распределение по способам (данные также подгружаются с сервера);
  • вкладку «Показатели за сегодня» — оперативные счётчики по ЕИС;
  • ссылку «Методика расчета показателей» на PDF с официальным описанием показателей.

Интерактив реализован скриптом портала (statistics.js): значения не зашиты в статический HTML, а приходят после загрузки страницы.

Оператор​

Федеральное казначейство — оператор ЕИС в сфере закупок; канонический вход на главную: https://zakupki.gov.ru/epz/main/public/home.html

  • Карточка ИС: ЕИС в сфере закупок — границы системы, СОИ и машиночитаемые комплекты документов.

URL и точки доступа​

Тип доступаURLФормат
Главная страница, якорь раздела «Статистика»https://zakupki.gov.ru/epz/main/public/home.html#statAnchorHTML
Методика расчёта показателей (файл с портала)https://zakupki.gov.ru/epz/main/public/documents/Calculation_methods.pdfPDF
JSON для блока «Общая» (пример среза; mode и tabType совпадают с переключателями в UI)https://zakupki.gov.ru/epz/main/public/{mode}/{tabType}.json с параметрами запроса (см. пример ниже)JSON

На странице задаётся contextPath = "/epz/main/public" — JSON-запросы строятся относительно этого префикса (как в клиентском коде портала).

Объекты данных и показатели​

  • Агрегаты по годам и регионам (карта, легенда, слайдер «Цифры»): объёмы и доли в логике методики PDF; идентификаторы срезов в JSON (например, ключи вида 44{год}) нужно сопоставлять с подписями в ответе и в интерфейсе.
  • Способы закупки — сводные счётчики/объёмы по способам для выбранного правового режима и параметров фильтра.
  • Показатели за сегодня — краткие оперативные величины по состоянию на момент обновления сервера.

Типичные ключи для связи с микроданными: код региона (КЛАДР/ФИАС) в параметрах карты, разделение FZ44 / FZ223 (placeOfSearch в запросах), год отчёта. Прямой реестровый номер контракта в этих JSON обычно не является строкой набора — для контрактного следа используйте ЕИС и XML.

Периодичность обновления​

«Показатели за сегодня» и часть общих рядов обновляются в течение дня (на клиенте может быть таймер обновления). Исторические годы в «Общей» статистике появляются по мере закрытия отчётного периода и пересчёта на стороне оператора — точные сроки смотрите в методике и по датам в JSON (updateDateTime в ответах вкладки «за сегодня»).

Ограничения и особенности​

  • Не документированный как публичный API контракт JSON: пути, набор параметров (placeOfSearch, year, kladr, contractStatus и др.) и ответы могут меняться с обновлением портала; для воспроизводимых пайплайнов приоритет — XML СОИ.
  • Запросы к JSON без типичного User-Agent или без Referer с главной страницы иногда приводят к ошибкам шлюза; ориентируйтесь на поведение, близкое к браузеру.
  • 44-ФЗ и 223-ФЗ в одном блоке — смешивать ряды без фильтра режима нельзя (см. 44-ФЗ и 223-ФЗ).
  • Цифры блока — витринная аналитика ЕИС; сверка с агрегатами Казначейства или отчётами Минфина требует учёта разных методик.

Примеры доступа​

Скачать методику показателей:

curl -sS -A "Mozilla/5.0" -L -o Calculation_methods.pdf \
  "https://zakupki.gov.ru/epz/main/public/documents/Calculation_methods.pdf"

Пример JSON для вкладки «Общая», режим «Цифры», срез «Экономия», с фильтром 44-ФЗ (заголовок Referer снижает риск отказа шлюза):

curl -sS -A "Mozilla/5.0" \
  -e "https://zakupki.gov.ru/epz/main/public/home.html" \
  "https://zakupki.gov.ru/epz/main/public/all-stats/common-economy.json?placeOfSearch=FZ44"

Связанные страницы wiki​

  • Сведения о закупках и контрактах (ЕИС) — основной вход в документы и реестры закупок и контрактов.
  • XML-выгрузки ЕИС Закупки — машиночитаемый построчный контур для анализа и воспроизводимых выборок.
  • ЕИС в сфере закупок — оператор, СОИ и ограничения доступа.
  • Витрины данных Казначейства (datamarts) — иные агрегированные срезы по закупкам; пересечение по смыслу возможно, методики различаются.
  • Минфин: мониторинг закупок (отчёты PDF) — ведомственная сводная отчётность по мониторингу, не заменяет ни ЕИС, ни блок статистики на главной.

Не путать с​

  • Сведения о закупках и контрактах (ЕИС) и XML-выгрузки — оперативные и построчные данные; блок «Статистика» даёт своды и визуализацию по тем же правовым режимам.
  • Минфин: мониторинг закупок — PDF-отчёты Минфина, другой оператор и состав показателей.
